===========
Quick Start
===========

.. meta::
   :description: Create your first Atlas cluster using Kubernetes configuration files with the Atlas Kubernetes Operator.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

You can use |ak8so| to manage resources in |service| without leaving
|k8s|. This tutorial demonstrates how to create your first {+cluster+}
in |service| from |k8s| configuration files with |ak8so|.

.. warning:: Multi-Tenancy Security Guidance

   Do not share a single |ak8so| installation across multiple
   tenants, even if they use different namespaces. Tenants that
   share an operator can use cross-namespace references to access
   resources in other namespaces, including secrets that provide
   access to other tenants' |service| configurations or other
   cloud resources. To isolate tenants, deploy a separate |ak8so|
   instance for each tenant.

.. note:: **Would you prefer to start with Helm?**

   To create your first {+cluster+} in |service| from Helm Charts with 
   |ak8so|, see :ref:`ak8so-quick-start-helm-ref`.

Prerequisites
-------------

This tutorial requires:

- An |service| account. To create one, see
  :ref:`create-atlas-account`.
- An |service| organization and project. To create them, see
  :ref:`create-project`.
- A running |k8s| cluster with nodes running processors with the
  x86-64, AMD64, or ARM64 architecture.
- ``kubectl`` installed and configured to communicate with your
  |k8s| cluster.
- ``jq`` 1.6 or later.
- Access to the |ak8so| project on GitHub: `https://github.com/mongodb/mongodb-atlas-kubernetes
  <https://github.com/mongodb/mongodb-atlas-kubernetes>`_

**Networking requirements:**

- |k8s| nodes must reach |service| on port 443 (HTTPS) for |ak8so|
  to create and manage resources.
- Port 27017 (default) must be open between all |k8s| pods if you
  deploy a multi-node replica set.

Verify the Deployment
---------------------

After the procedure completes, confirm your cluster is ready before
connecting.

.. list-table::
   :header-rows: 1
   :widths: 40 60

   * - Checkpoint
     - Verification Step

   * - Custom resource ready
     - Run ``kubectl get atlasdeployment -n <your-namespace>`` and
       confirm the ``AtlasDeployment`` resource shows the ``READY``
       condition as ``True``.

   * - Cluster healthy in |service|
     - In the |service| UI, confirm the cluster appears in your
       project with a status of ``Active``.

   * - Connection succeeds
     - Use the connection string from |service| to connect with
       {+mongosh+} and confirm the cluster is reachable.

.. _ak8so-quick-start-next-steps:

Next Steps
----------

After you verify your cluster, you can:

- **Configure access control**: Set up database users and network
  access. To learn how, see :ref:`ak8so-getting-started`.

- **Set up backup**: Configure Cloud Backup for your cluster.
  To learn how, see :ref:`ak8so-configure-backup`.

- **Create an Atlas Search index**: Enable full-text search on your
  data. To learn how, see :ref:`ak8so-atlas-search`.

- **Try the Helm-based quickstart**: Deploy a cluster from Helm
  Charts instead of configuration files. To learn how, see
  :ref:`ak8so-quick-start-helm-ref`.
