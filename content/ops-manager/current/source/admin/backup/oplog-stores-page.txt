===================
Manage Oplog Stores
===================

.. meta::
   :description: Configure and manage local and S3 oplog stores used by Ops Manager to store oplog slices for continuous backup and point-in-time restore of replica sets.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

Oplog stores are backing replica sets that store
:term:`oplog slices <oplog slice>`. |onprem| uses oplog
slices to provide continuous backups for replica sets, allowing
you to restore to any point in time. This page configures
the oplog stores that |onprem| uses for this purpose.

From this page, you can:

- :doc:`Manage local oplog stores </tutorial/manage-oplog-storage>`

- :doc:`Manage S3 oplog stores </tutorial/manage-s3-oplog-storage>`
