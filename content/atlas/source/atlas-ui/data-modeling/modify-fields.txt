Modify Fields in Your |service| |erd-title|
===========================================

   :values: tutorial

.. meta::

.. contents:: On this page

.. include:: /atlas-ui/includes/shared/data-modeling-modify.rst

In the :guilabel:`Data Modeling` tab, you can:

- Add or remove fields
- Change field data types
- Rename fields

.. include:: /atlas-ui/includes/shared/fact-diagram-modification-data.rst

Procedure
---------

The following steps describe how you can add, edit, and remove fields 
in your |erd|:

.. procedure:: 
   :style: normal

   .. include:: /includes/nav/steps-data-explorer.rst 

   .. include:: /atlas-ui/includes/shared/steps-data-modeling-open.rst

   .. step:: Add a new field. 

      To add a new field, click the plus :icon-lg:`Plus` icon in the upper-right 
      corner of one of your collection nodes. A new field appears in the 
      collection node.

      Once you add a field, you can edit the field's properties, including: 

      - Renaming the field.
      - Changing the field's data type.
      - Adding relationships.
    
   .. include:: /atlas-ui/includes/shared/steps-field-side-panel.rst

   .. step:: *(Optional)* Rename a field.

      Under the :guilabel:`Name` section, specify a new name for your selected 
      field. 

   .. step:: *(Optional)* Change a field's data type.

      In the :guilabel:`Datatype` drop-down menu, select the datatypes that you 
      want to assign to the field. Because data in MongoDB has a flexible schema 
      model, you can specify multiple data types for a field. If a field 
      contains multiple data types, |service| labels the field as 
      :guilabel:`(mixed)` in the diagram.

   .. step:: *(Optional)* Delete a field.

      In the side panel, click the trash :icon-lg:`Trash` icon.
