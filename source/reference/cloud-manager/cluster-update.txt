.. _mcli-cm-cluster-update-command:

=====================================
mongocli cloud-manager cluster update
=====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster update`` command updates the cluster in the
specified |cloud|  project. You can also update a cluster through the 
|cloud-short| :cloudmgr:`UI <tutorial/edit-replica-set/>` 
or the :cloudmgr:`Automation Configuration API 
<reference/api/automation-config/>`. 

Using the ``cluster update`` command, you can add new members to 
the replica set, replace or remove a member, and modify the priority 
of a member or the number of votes for a member during an election. 
You cannot modify the port number or file paths of the :manual:`mongod 
</reference/program/mongod/#bin.mongod>` processes.

.. _cm-cluster-update-syntax:

Syntax 
------

.. code-block:: text 
   :copyable: false 

   mongocli cloud-manager|cm cluster update 
        --file|-f <cm-settings-file> 
        [ --profile|-p <profile-name> ] 
        [ --projectId <project-ID> ]

.. _cm-cluster-update-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--file``, ``-f`` 
     - string 
     - Path to the file that contains the modified settings. 
       To learn more about the format and the configuration 
       properties to specify in the configuration file, see 
       :ref:`mms-cluster-settings-file`.
     - yes

   * - ``--profile``, ``-p``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable <mcli-env-var>`.
     - no

.. _cm-cluster-update-command-output:

Output 
------

The command prints output similar to the following to the 
terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` for 
recommended solutions.

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://cloud.mongodb.com/v2/<projectId>#deployment/topology for status

where ``projectId`` is the unique identifier of the project.

.. _cm-cluster-update-examples:

Examples 
--------

The following examples show two ways to run the command.

.. _cm-cluster-update-example1:

Example 1
~~~~~~~~~

The following ``mongocli cloud-manager cluster update`` command 
updates the cluster named ``myCluster`` in the specified project using: 

- A file named ``cm-cluster.yaml``
- A custom profile named ``egCMprofile``

.. code-block:: sh 
   :copyable: false

   ~ mongocli cloud-manager cluster update --file cm-cluster.yaml --projectId 5e2f06127a3e5a5700de8d8d --profile egCMprofile

.. _cm-cluster-update-example2:

Example 2
~~~~~~~~~

The following ``mongocli cm cluster update`` command updates the 
cluster named ``myCluster`` in the specified project using: 

- A file named ``cm-cluster.json``
- The ``default`` profile

.. code-block:: sh 
   :copyable: false

   ~ mongocli cm cluster update --file cm-cluster.json --projectId 5e2f06127a3e5a5700de8d8d

Output
~~~~~~

The :ref:`cm-cluster-update-example1` and :ref:`cm-cluster-update-example2` 
commands print the following to the terminal. To learn 
more about these fields, see :ref:`cm-cluster-update-command-output`.

.. code-block:: json
   :copyable: false 

   Changes are being applied, please check https://cloud.mongodb.com/v2/5e2f06127a3e5a5700de8d8d#deployment/topology for status
