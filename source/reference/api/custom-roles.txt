====================
Custom MongoDB Roles
====================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: twocols

.. include:: /includes/fact-projects-groups-synonyms.rst

.. include:: /includes/api-base-url.rst

The ``customDBRoles`` resource lets you retrieve, create and modify the
custom MongoDB roles in your cluster. Use custom MongoDB roles to
specify custom sets of actions which cannot be described by the
built-in |service| :ref:`database user privileges
<atlas-user-privileges>`.

.. note::

   The available :manual:`privilege actions
   </reference/privilege-actions/>` for custom MongoDB roles support a
   subset of MongoDB commands. See
   :ref:`paid-tier-command-limitations` for more information.

.. include:: /includes/fact-role-auditing.rst

The ``customDBRoles`` resource requires your :ref:`group-id`.

.. list-table::
   :header-rows: 1
   :widths: 10 60 25

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/customDBRoles/roles </reference/api/custom-roles-get-all-roles>`

     - Get all custom MongoDB roles in the project.

   * - ``GET``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/customDBRoles/roles/{ROLE-NAME} </reference/api/custom-roles-get-single-role>`

     - Get the custom MongoDB role named {ROLE-NAME}.

   * - ``POST``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/customDBRoles/roles </reference/api/custom-roles-create-a-role>`

     - Create a new custom MongoDB role in the project.

   * - ``PATCH``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/customDBRoles/roles/{ROLE-NAME} </reference/api/custom-roles-update-a-role>`

     - Update a custom MongoDB role in the project.

   * - ``DELETE``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/customDBRoles/{ROLE-NAME} </reference/api/custom-roles-delete-a-role>`

     - Delete a custom MongoDB role from the project.
