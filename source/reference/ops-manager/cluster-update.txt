
.. _mcli-om-cluster-update-command:

===============================
mcli ops-manager cluster update
===============================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster update`` command updates a cluster in the specified 
|mms| project. You can also update a cluster through the |mms| 
:opsmgr:`UI </tutorial/edit-replica-set/>` or :opsmgr:`Automation 
Configuration API </reference/api/automation-config/>`.

Using the ``cluster update`` command, you can add new members to 
the replica set, replace or remove a member, and modify the priority 
of a member or the number of votes for a member during an election. 
You cannot modify the port number or file paths of the :manual:`mongod 
</reference/program/mongod/#bin.mongod>` processes.

.. _om-cluster-update-syntax:

Syntax 
------

.. code-block:: text 
   :copyable: false 

   mcli ops-manager|om cluster update 
        -f|--file <om-settings-file> 
        [ -p|--profile <profile-name> ] 
        [ --projectId <project-ID> ]

.. _om-cluster-update-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``-f``, ``--file`` 
     - string 
     - The path to the file that contains the modified settings. 
       To learn more about the format and the configuration 
       properties to specify in the configuration file, see 
       :ref:`mms-cluster-settings-file`.
     - yes

   * - ``-p``, ``--profile``
     - string
     - The name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`configure-mcli`.
     - no

   * - ``--projectId``
     - string
     - The unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable <mcli-env-var>`.
     - no

.. _om-cluster-update-output:

Output 
------

The command prints output similar to the following to the 
terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` for 
recommended solutions.

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://<hostname>:<port>/v2/<projectId>#deployment/topology for status

where: 

- ``<hostname>:<port>`` is the |mms| base URL
- ``projectId`` is the unique identifier of the project 

.. _om-cluster-update-examples:

Examples 
--------

The following examples show two ways to run the command.

.. _om-cluster-update-example1:

Example 1
~~~~~~~~~

The following ``mcli ops-manager cluster update`` command 
updates the cluster named ``myCluster`` in the specified project using: 

- A file named ``om-cluster.yaml``
- A custom profile named ``egOMprofile``

.. code-block:: sh 
   :copyable: false

   mcli ops-manager cluster update --file om-cluster.yaml --profile egOMprofile --projectId 5e2dec2131cd7c007880b05e

.. _om-cluster-update-example2:

Example 2
~~~~~~~~~

The following ``mcli om cluster update`` command updates the 
cluster named ``myCluster`` in the specified project using: 

- A file named ``om-cluster.json``
- The ``default`` profile

.. code-block:: sh 
   :copyable: false

   mcli om cluster update --file om-cluster.json --projectId 5e2dec2131cd7c007880b05e

Output
~~~~~~

The :ref:`om-cluster-update-example1` and :ref:`om-cluster-update-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://localhost:30700/v2/5e2dec2131cd7c007880b05e#deployment/topology for status
