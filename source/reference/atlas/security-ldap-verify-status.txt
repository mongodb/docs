.. _mcli-atlas-security-ldap-verify-status-cmd:

==========================================
mongocli atlas security ldap verify status
==========================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``security ldap verify status`` command retrieves the 
status of the most recent verification request submitted 
using the :ref:`mcli-atlas-security-ldap-verify-cmd` command 
or the |service| :atlas:`API 
</reference/api/ldaps-configuration-request-verification/>`. 
You can also retrieve the status of a verification request 
using the |service| :atlas:`API 
</reference/api/ldaps-configuration-verification-status/>`.

.. note:: 

   |service| returns a |http| 404 error if you request the 
   status of any request other than the most recent. 

.. _mcli-atlas-security-ldap-verify-status-syntax: 

Syntax 
------

.. code-block:: shell 
   
   mongocli atlas security ldap verify <request-ID>
        [ --output|-o <output-format> ]
        [ --profile|-P <name-of-profile> ]
        [ --projectId <id-of-project> ]

.. include:: /includes/fact-command-line-help.rst

.. _mcli-atlas-security-ldap-verify-status-args:

Arguments 
---------

.. list-table::
   :header-rows: 1
   :widths: 25 10 55 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - <request-ID>
     - string
     - Unique identifier of the most recent request to verify 
       the |ldap| over |tls|/|ssl| configuration. Run the 
       :ref:`mcli-atlas-security-ldap-verify-cmd` command 
       for the request ID.
     - yes

.. _mcli-atlas-security-ldap-verify-status-options:

Options 
-------

.. list-table::
   :header-rows: 1
   :widths: 25 10 55 10

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string 
     - Name of the profile to use for accessing the |service| 
       project. If omitted, uses the {+default-profile+}.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the |service| project. 
     - yes

.. _mcli-atlas-security-ldap-verify-status-output: 

Output 
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh 
   :copyable: false 

   REQUEST ID                 PROJECT ID           STATUS
   <verification-request-ID>  <atlas-project-ID>   <verification-status>

The default output contains a subset of the fields returned by this 
command. For the complete list of |json| fields returned by the 
command, see the :atlas:`API reference 
</reference/api/ldaps-configuration-verification-status/#response-elements>`.

.. _mcli-atlas-security-ldap-verify-status-egs:

Example
-------

The following example uses the ``mongocli atlas security ldap verify 
status`` command to retrieve the status of the most recent request for 
verification of an LDAP over TLS configuration. The command uses the 
{+default-profile+} to access the |service| project.

   .. tab:: Default Output 
      :tabid: default

      .. code-block:: shell
         
         mongocli atlas security ldap verify status 5fa1afa40bcb85015ae356ec

      .. code-block:: shell 
         :copyable: false 

         REQUEST ID                 PROJECT ID                 STATUS
         5fa1afa40bcb85015ae356ec   5e2211c17a3e5a48f5497de3   SUCCESS
