==================================================
Install the Automation Agent with ``deb`` Packages
==================================================

.. default-domain:: mongodb

Overview
--------

.. include:: /includes/fact-installation-automation-needs-automation-agent.rst

Use this procedures to install the Automation Agent on Ubuntu with ``deb``
packages. For Debian systems, use the
:doc:`/tutorial/install-automation-agent-from-archive` procedure.

To install to a server that has an existing MongoDB
deployment that was **not** installed with a ``deb`` package, use the
:doc:`/tutorial/install-automation-agent-from-archive` procedure.

Prerequisites
-------------

64-Bit Architecture
~~~~~~~~~~~~~~~~~~~

The Automation Agent can run only on a 64-bit architecture.

.. include:: /includes/prerequisites-automation-agent-servers.rst

.. include:: /includes/prerequisites-automation-agent-monitored-deployment.rst

.. include:: /includes/prerequisites-automation-agent-new-deployment.rst

Ubuntu 12.04
~~~~~~~~~~~~

If you install the Automation Agent to an Ubuntu 12.04 machine, and if you will
deploy MongoDB instances that use the `MongoDB Enterprise
<http://www.mongodb.com/products/mongodb-enterprise>`_ ``PLAIN``
authentication mechanism, then you must either:

- manually change the primary group of the ``mongodb`` user to ``sasl``, or

- :doc:`install the agent from the tar.gz archive </tutorial/install-automation-agent-from-archive>`.

Alternatively, you can use Ubuntu 14.04.

.. include:: /includes/prerequisites-automation-agent-servers.rst

Root Access
~~~~~~~~~~~

To install the Automation Agent using a ``deb`` package, you must have
root access.
