==========================================================================================
{+fts+} and {+avs+} Local Development Quickstart
==========================================================================================

.. meta::
   :description: Learn how to deploy mongot for local development of MongoDB Search and Vector Search on a single-node Atlas deployment using Docker Compose and the atlas-local image.

.. facet::
   :name: genre
   :values: tutorial

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

You can use the ``mongodb/mongodb-atlas-local`` Docker image to create a
single node replica set with |fts| and {+avs+}. The
``mongodb/mongodb-atlas-local`` Docker image is a self-contained MongoDB
deployment that includes ``mongot`` and is meant only for local
development and testing purposes. The single node replica set deployment
runs on port 27017 and doesn't support multiple ``mongot`` processes.
This deployment is not suitable for production use.

This tutorial shows you how to deploy ``mongot`` with a single node
replica set using Docker Compose. 

Prerequisites
-------------

Before you begin, you must have the following installed:

- Docker
- Docker Compose
- ``mongosh`` locally or in Docker

Deploy {+fts+} and {+avs+} with Docker Compose
-------------------------------------------------------------------

.. procedure::
   :style: normal

   .. step:: Pull the latest ``mongodb/mongodb-atlas-local`` Docker image.

      To pull the ``mongodb/mongodb-atlas-local`` Docker image, run the
      following command:

      .. code-block:: shell

         docker pull mongodb/mongodb-atlas-local:preview

      .. note::

         You must use the ``preview`` tag to get the latest version of
         ``mongot``. The ``preview`` tag bundles the Community build of
         ``mongot``, which supports automated embedding. Images without
         the ``preview`` tag don't include the Community build of
         ``mongot`` or support automated embedding. Use the ``latest``
         tag only if you have existing tooling or scripts pinned to
         ``latest`` and you do not need automated embedding.

      To create a reproducible environment, pin the image to a specific
      version tag instead of a moving tag like ``preview`` or
      ``latest``. Replace ``<version>`` with the version tag you want:

      .. code-block:: shell

         docker pull mongodb/mongodb-atlas-local:<version>

   .. step:: Create a ``docker-compose.yaml`` file.

      Create the ``docker-compose.yaml`` file. For a complete and
      detailed example, see :ref:`atlas-cli-docker-compose`.

      .. code-block:: yaml
         :linenos:

         services:
            mongodb:
               image: mongodb/mongodb-atlas-local:preview
               hostname: mongodb
               environment:
                  - MONGODB_INITDB_ROOT_USERNAME=<username>
                  - MONGODB_INITDB_ROOT_PASSWORD=<password>
               ports:
                  - 27017:27017
               volumes:
                  - ./init:/docker-entrypoint-initdb.d
                  - db:/data/db
                  - configdb:/data/configdb
                  - mongot:/data/mongot
         volumes:
            db:
            configdb:
            mongot:

   .. step:: Replace the placeholder values and save the file.

      Replace the ``<username>`` and ``<password>`` placeholder values
      with your own values and save the file. You can also set the
      username and password through a file for an extra layer of
      security. Use the ``MONGODB_INITDB_ROOT_USERNAME_FILE`` and
      ``MONGODB_INITDB_ROOT_PASSWORD_FILE`` environment variables to
      specify the path to the files that contain the username and
      password.

      To use Automated Embedding, provide your |voyage| API key
      (created from the {+service+} UI) through the environment variable
      ``VOYAGE_API_KEY``. By default, the embedding provider endpoint is
      ``https://ai.mongodb.com/v1/embeddings`` (for keys created from
      the {+service+} UI).

      .. note::

         If you created your API key directly from |voyage|, you must
         also provide the embedding provider endpoint through the
         environment variable ``VOYAGE_EMBEDDING_PROVIDER_ENDPOINT``.
         The value for this variable is
         ``https://api.voyageai.com/v1/embeddings``.

         To learn more, see :ref:`mongot-with-auto-embed`.

      To automatically load the MongoDB sample dataset during
      initialization, set the ``MONGODB_LOAD_SAMPLE_DATA`` environment
      variable to ``true``.

      .. code-block:: yaml
         :emphasize-lines: 6-9
         :linenos:
         :caption: docker-compose.yaml Settings for Automated Embedding and Sample Data

         services:
            mongodb:
               image: mongodb/mongodb-atlas-local:preview
               hostname: mongodb
               environment:
                  - MONGODB_INITDB_ROOT_USERNAME_FILE=<path-to-username-file>
                  - MONGODB_INITDB_ROOT_PASSWORD_FILE=<path-to-password-file>
                  - MONGODB_LOAD_SAMPLE_DATA=true
                  - VOYAGE_API_KEY=<voyage-api-key>

   .. step:: Run Docker Compose.

      Run the following command to create a single node replica set
      with |fts| and {+avs+}.

      .. code-block:: shell

         docker compose up -d

   .. step:: Verify the deployment.

      To verify the deployment, run the following command:

      .. code-block:: shell

         docker ps

      The output should resemble the following:

      .. code-block:: sh

         CONTAINER ID   IMAGE                                 COMMAND                  CREATED       STATUS                 PORTS                                             NAMES
         f68ee6df9907   mongodb/mongodb-atlas-local:preview   "/usr/local/bin/runn…"   4 hours ago   Up 4 hours (healthy)   0.0.0.0:27017->27017/tcp, [::]:27017->27017/tcp   search-in-community-mongodb-1

Connect and Use {+fts+} and {+avs+} with ``mongosh``
-------------------------------------------------------------------------

.. procedure::
   :style: normal

   .. step:: Connect to the single node replica set.

      To connect to the single node replica set, run the following
      command after replacing ``<username>`` with the username you
      specified in the ``docker-compose.yaml`` file:

      .. code-block:: shell

         mongosh "mongodb://localhost:27017" -u <username>

   .. step:: (Optional). If you loaded the sample dataset, create an index.

      a. To verify that the same data is loaded, run the following
         command:

         .. io-code-block::
            :copyable: true

            .. input::
               :language: shell

               show dbs

            .. output::
               :language: shell
               :emphasize-lines: 1

               __mdb_internal_search   56.00 KiB
               admin                  252.00 KiB
               config                 172.00 KiB
               local                  193.25 MiB
               sample_airbnb           52.52 MiB
               sample_analytics         9.48 MiB
               sample_geospatial        1.15 MiB
               sample_guides           40.00 KiB
               sample_mflix            96.16 MiB
               sample_restaurants       6.43 MiB
               sample_supplies          1.04 MiB
               sample_training         46.91 MiB
               sample_weatherdata       2.58 MiB

      #. To create indexes, switch to the ``sample_mflix`` database.

         .. code-block:: shell

            use sample_mflix

      #. To create a {+fts+} index named ``movie-title-index`` on the
         ``sample_mflix.movies`` collection, run the following command:

         .. include:: /includes/local-dev-quick-start/facts/fts-index-intro.rst

         .. literalinclude:: /includes/local-dev-quick-start/code-snippets/index/fts-index.js
            :linenos:
            :copyable: true
            :language: javascript

      #. To create a {+avs+} index named ``movie-vector-index`` on the
         ``sample_mflix.embedded_movies`` collection, run the following
         command:

         .. include:: /includes/local-dev-quick-start/facts/vs-index-intro.rst

         .. literalinclude:: /includes/local-dev-quick-start/code-snippets/index/vs-index.js
            :linenos:
            :copyable: true
            :language: javascript

      #. To verify that the indexes were created, run the following
         command:

         .. io-code-block::
            :copyable: true

            .. input::
               :language: shell

               db.movies.getSearchIndexes().forEach(printjson)

            .. output:: /includes/local-dev-quick-start/code-snippets/output/get-search-indexes.js
               :language: javascript
               :visible: false

   .. step:: (Optional). If you created the index, run a query on the index.

      a. To run a {+fts+} query on the ``movie-index`` index, run
         the following command:

         .. include:: /includes/local-dev-quick-start/facts/fts-query-intro.rst

         .. io-code-block::

               :language: json

      #. To run a {+avs+} query on the ``movie-vector-index`` index,
         perform the following steps:

         i. Create a file named ``query-embeddings.js``:

            .. code-block:: shell

               touch query-embeddings.js

         #. Copy and paste the following embeddings for the phrase
            ``courtroom lawyer`` into the ``query-embeddings.js`` file:

            .. literalinclude:: /includes/local-dev-quick-start/code-snippets/query-embeddings.js
               :language: javascript
               :copyable: true

         #. Load the ``query-embeddings.js`` file into ``mongosh``:

            .. code-block:: shell

               load('/<path-to-file>/query-embeddings.js')
