.. _atlasorgsettings-custom-resource:

====================================
``AtlasOrgSettings`` Custom Resource
====================================

.. meta::
   :description: The AtlasOrgSettings custom resource allows you to modify the specific organization settings.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

The ``AtlasOrgSettings`` custom resource allows you to modify settings for a specific organization.

.. include:: /includes/fact-ak8so-crds.rst

Example
-------

The following example shows an ``AtlasOrgSettings`` custom resource:

.. code-block::

   apiVersion: atlas.mongodb.com/v1
   kind: AtlasOrgSettings
   metadata:
     name: my-org-settings
     namespace: mongodb-atlas-system
   spec:
     orgID: "455aff84aca055313451234c"
     connectionSecretRef:
       name: my-org-connection-secret
     apiAccessListRequired: true
     genAIFeaturesEnabled: true
     maxServiceAccountSecretValidityInHours: 42
     multiFactorAuthRequired: true
     restrictEmployeeAccess": true
     securityContact": "test@example.com"
     streamsCrossGroupEnabled": true

Parameters
----------

This section describes the ``AtlasOrgSettings`` custom
resource parameters available.

.. setting:: spec.orgId 

   *Type*: string

   *Required*

   Unique 24-hexadecimal digit string that identifies the organization 
   that contains your projects. Format should match the following 
   pattern: ``^([a-f0-9]{24})$``.

.. _atlasorg-spec-connectionSecretRef-name:

``spec.connectionSecretRef.name``

*Type*: string

*Optional*

You can use the ``spec.connectionSecretRef.name`` parameter to set 
the connection secret for the ``AtlasOrgSettings`` custom resource. 
This parameter overrides the default global connection secret.

.. note::

   The credentials for the ``AtlasOrgSettings`` must have the 
   :authrole:`Organization Owner` role to modify the organization settings.

.. setting:: spec.apiAccessListRequired 

   *Type*: boolean

   *Optional*

   Flag that indicates whether to require API operations to originate 
   from an IP Address added to the API access list for the specified organization.

.. setting:: spec.genAIFeaturesEnabled 

   *Type*: boolean

   *Optional*

   *Default*: true

   Flag that indicates whether this organization has access to generative AI features. 
   This setting is enabled by default, and once it is turned on, Project Owners may be able 
   to enable or disable individual AI features at the project level.

   .. note::

      This setting is not applicable to |service| for Government.

.. setting:: spec.maxServiceAccountSecretValidityInHours 

   *Type*: integer(int32)

   *Optional*

   *Minimum*: 8

   *Maximum*: 8760

   The maximum period before expiry in hours for new {+atlas-admin-api+} 
   Service Account secrets within the specified organization.

.. setting:: spec.multiFactorAuthRequired 

   *Type*: boolean

   *Optional*

   Flag that indicates whether to require users to set up |mfa| 
   before accessing the specified organization.

   To learn more, see :ref:`Manage Your Multi-Factor Authentication Options <atlas-enable-mfa>`.

.. setting:: spec.restrictEmployeeAccess 

   *Type*: boolean

   *Optional*

   Flag that indicates whether to block MongoDB Support from accessing 
   |service| infrastructure and {+cluster+} logs for any {+deployment+} in the 
   specified organization without explicit permission. 
   Once this setting is turned on, you can grant MongoDB Support a 
   24-hour bypass access to the |service| {+deployment+} to resolve support issues.

   To learn more, see :ref:`Configure MongoDB Support Access to Atlas Backend Infrastructure<restrict-access>`.

.. setting:: spec.securityContact 

   *Type*: string

   *Optional*

   String that specifies a single email address for the specified organization 
   to receive security-related notifications. Specifying a security contact 
   does not grant them authorization or access to |service| for security 
   decisions or approvals. An empty string is valid and clears the existing 
   security contact, if one is defined.

.. setting:: spec.orgstreamsCrossGroupEnabledId 

   *Type*: boolean

   *Optional*

   Flag that indicates whether a project's {+atlas-sp+} workspaces 
   in this organization can create connections to other projects' {+clusters+} 
   in the same organization.
