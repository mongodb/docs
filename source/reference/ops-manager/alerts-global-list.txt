.. _mcli-om-alert-global-list-cmd:

=======================================
mongocli ops-manager alerts global list
=======================================


.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol


The ``alerts global list`` command retrieves all the global alerts
for the specified |onprem| project. You can also retrieve all the
global alerts through the |onprem| :opsmgr:`UI 
</tutorial/manage-global-alerts/#view-global-alerts>` and :opsmgr:`API
</reference/api/global-alerts/>`.

.. _mcli-om-alert-global-list-syntax:

Syntax 
------

.. code-block:: json 

   mongocli ops-manager|om alerts global list 
        [ --limit <number> ]
        [ --page <number> ]
        [ --profile|-P <profile-name> ]
        [ --projectId projectId ]

.. include:: /includes/fact-command-line-help.rst

.. _mcli-om-alert-global-list-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--limit`` 
     - string 
     - The number of alert records to return.
     - no

   * - ``--page`` 
     - string 
     - The page to return. For example, suppose there are 200 
       alerts. If ``limit`` is set to ``100`` and ``page`` is set to ``2``, 
       the command will return records 101 to 200.
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable <mcli-env-var>`.
     - no

.. _mcli-om-alert-global-list-output: 

Output 
------

The command prints the following fields in |json| format to 
the terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` for 
recommended solutions.

.. list-table:: 
   :header-rows: 1
   :widths: 30 10 60 

   * - Field
     - Type
     - Description

   * - ``alertConfigId``
     - string
     - Unique identifier of the global alert configuration that triggered 
       this alert.

   * - ``created``
     - date
     - When the alert was opened.

   * - ``eventTypeName``
     - string
     - The :opsmgr:`name of the event </reference/alert-types/>` that triggered 
       the alert.

   * - ``groupId``
     - string
     - Unique identifier of the project that this alert was opened for.

   * - ``id``
     - string
     - Unique identifier.

   * - ``lastNotified``
     - date
     - When the last notification was sent for this alert. Only present if
       notifications have been sent.

   * - ``links``
     - document array
     - One or more links to sub-resources and/or related resources.
       The relations between URLs are explained in the `Web Linking
       Specification <http://tools.ietf.org/html/rfc5988>`_.

   * - ``tags``
     - array of strings
     - The tags associated with the alert.
   * - ``updated``
     - date
     - When the alert was last updated.

.. _mcli-om-alert-global-list-egs:

Examples 
--------

The following example uses the ``mongocli om alerts global list``
command to retrieve the list of global alerts for the specified
project. It uses the default profile to access the
|onprem| project.

.. code-block:: text

   mongocli om alerts global list --projectId 5e84b2edf949e9007a7beac3
