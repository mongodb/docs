.. _mcli-cm-cluster-create-command:

=====================================
mongocli cloud-manager cluster create
=====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster create`` command creates a cluster in the
specified |cloud| project. You can also create a cluster through the
|cloud-short| :cloudmgr:`console </tutorial/deploy-replica-set/>` or
:cloudmgr:`Automation Configuration API </reference/api/automation-config/>`.

.. _cm-cluster-create-syntax:

Syntax
------

.. code-block:: sh
   :copyable: false

   mongocli cloud-manager|cm cluster create
            --file|-f <cm-settings-file>
            [ --profile|-P <profile-name> ]
            [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _cm-cluster-create-options:

Options
-------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type
     - Description
     - Required?

   * - ``--file``, ``-f``
     - string
     - Path to the file that contains the settings for creating the
       cluster. To learn more about the configuration properties to
       specify in the file, see :ref:`mms-cluster-settings-file`.
     - yes

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private keys for the
       project are saved. If omitted, uses the ``default`` profile. To
       learn more about creating a profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the cluster to
       update. If omitted, uses the project ID in the profile or the
       :ref:`environment variable <mcli-env-var>`.
     - no

.. _cm-cluster-create-command-output:

Output
------

The command prints output similar to the following to the terminal if
the command succeeds. If the command returns errors, see
:ref:`Troubleshooting <troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   Changes are being applied, please check http://cloud.mongodb.com/v2/<projectId>#deployment/topology for status

where ``projectId`` is the unique identifier of the project.

.. _cm-cluster-create-examples:

Examples
--------

The following examples show two ways to run the command.

.. _cm-cluster-create-example1:

Example 1
~~~~~~~~~

The following ``mongocli cm cluster create`` command creates a 
cluster named ``myCluster`` in the specified project using: 

- A file named ``cm-cluster.json``, which contains the settings in the 
  :ref:`sample file <mms-sample-config-file>` in ``.json`` format
- The {+default-profile+}

.. code-block:: sh
   :copyable: false

   mongocli cloud-manager cluster create \
            --file cm-cluster.json \
            --projectId 5e2f06127a3e5a5700de8d8d

.. _cm-cluster-create-example2:

Example 2
~~~~~~~~~

The following ``mongocli cloud-manager cluster create`` command 
creates a cluster named ``myCluster`` in the specified project using: 

- A file named ``cm-cluster.yaml``, which contains the settings in the 
  :ref:`sample file <mms-sample-config-file>` in ``.yaml`` format
- A custom profile named ``egCMprofile``

.. code-block:: sh
   :copyable: false

   mongocli cloud-manager cluster create \
            --file cm-cluster.yaml \
            --projectId 5e2f06127a3e5a5700de8d8d \
            --profile egCMprofile

Output
~~~~~~

The :ref:`cm-cluster-create-example1` and :ref:`cm-cluster-create-example2`
commands print the following to the terminal:

.. code-block:: json
   :copyable: false

   Changes are being applied, please check https://cloud.mongodb.com/v2/<projectId>#deployment/topology for status
