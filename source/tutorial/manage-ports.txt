=====================
Manage |onprem| Ports
=====================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

Overview
--------

|onprem| uses the ports and health-check endpoints described in
:doc:`/reference/firewall-configuration`. The endpoints are disabled by
default. You can change |onprem|'s default port and enable the endpoints.

Procedures
----------

Change the Default Port on Unix-based Systems
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

To change the ports on Unix-based operating systems, you must edit
the port settings in ``<install-directory>/conf/mms.conf``, update the 
|onprem| URL in :guilabel:`Ops Manager Config` settings,
and restart |onprem|.

.. include:: /includes/steps/change-ports-unix.rst

Change the Default Port on a Windows System
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

To change the ports on Windows, you need to edit the Windows registry,
update the |onprem| URL in :guilabel:`Ops Manager Config` settings,
and restart |onprem|.

.. |ssl-port-reg| replace:: ``HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Apache Software Foundation\Procrun 2.0\MMS\Parameters\Java\Options``

.. include:: /includes/steps/change-ports-windows.rst

Enable the Health Check Endpoint
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

|onprem| provides an HTTP health-check endpoint on port 8090 that is
disabled by default. Before you enable the endpoint, ensure that port 8090
is available for use.
