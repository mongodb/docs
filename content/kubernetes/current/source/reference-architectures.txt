.. _reference-architectures:

=======================
Reference Architectures
=======================

.. meta::
   :description: Reference architectures for deploying MongoDB resources on Kubernetes, including multi-cluster deployments with and without a service mesh.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can use reference architectures to quickly and efficiently build
out your MongoDB |k8s-op-short| infrastructure. These resources are
opinionated and meant to serve as a practical reference and starting
point as you work through the process of deploying or upgrading your
MongoDB resources running in Kubernetes.

:ref:`reference-architectures-how-to`
  Learn how to use reference architectures, including how to clone
  the source code and set up the required environment variables.

:ref:`reference-architectures-multi-cluster`
  Deploy MongoDB resources across multiple |k8s| clusters using an
  Istio service mesh for cluster-to-cluster networking.

:ref:`reference-architectures-multi-cluster-no-mesh`
  Deploy MongoDB resources across multiple |k8s| clusters without a
  service mesh, using External DNS for cluster-to-cluster networking.
