      Check Index Status 
      ~~~~~~~~~~~~~~~~~~

      To check the status of your {+avs+} index, use the {+atlas-ui+} to 
      navigate to your cluster and do the following:

      .. procedure:: 
         :style: normal

         .. include:: /includes/nav/steps-search-simple.rst

         .. step:: Check the status of the index. 

            a. In the :guilabel:`Search & Vector Search` page, locate the index 
               to check.
            
            #. Check the :guilabel:`Status` field to verify the status of the 
               index.

   .. selected-content::
      :selections: self, None, None, None

      Check Index Status 
      ~~~~~~~~~~~~~~~~~~

      To check the status of your {+avs+} index, run the following command 
      after replacing the following placeholders: 

      .. list-table:: 
         :widths: 25 75 
         :stub-columns: 1

         * - ``<connection-string>`` 
           - Your cluster connection string.

         * - ``<database-name>``
           - The name of the database the collection.

         * - ``<collection-name>``
           - The name of the collection for which you created the index.

      .. code-block:: javascript

         mongosh <connection-string>/<database-name> --eval '
         db.<collection-name>.getSearchIndexes().forEach(printjson);
         '

      The preceding command connects to your cluster and for the specified 
      collection in the specified database, iterates through each index 
      on the collection and returns the index definition in |json| format. 

      .. io-code-block:: 
         :copyable: true 
         :caption: Sample Command to Retrieve Indexes on the movies Collection
         
         .. input:: 
            :language: javascript 

            mongosh "mongodb://localhost:27017/sample_mflix" --eval '
            db.movies.getSearchIndexes().forEach(printjson);
            '

         .. output:: 
            :language: javascript 
            :visible: false

            {
               id: '698d005f8bf34e5e6624d086',
               name: 'vector_index',
               type: 'vectorSearch',
               status: 'READY',

      Verify Embeddings Were Generated 
      ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

      To verify that {+avs+} has generated embeddings for your indexed text 
      data, run the following command after replacing the ``<connection-string>`` 
      placeholder with your cluster connection string: 

      .. code-block:: javascript 

         mongosh "<connection-string>" --apiVersion 1 --eval '
            print("Embeddings: " +
               db.getSiblingDB("__mdb_internal_search")
                  .getCollectionNames()
                  .filter(c => /^[0-9a-f]{24}-[0-9a-f]{32}-\d+-\d+$/.test(c))
                  .map(c => db.getSiblingDB("__mdb_internal_search").getCollection(c).countDocuments())
                  .reduce((a, b) => a + b, 0)
            );
            '

      The preceding command connects to your cluster and returns the total 
      number of vector embeddings stored across all collections in your 
      deployment.

   Learning Summary
   ----------------

   This quick start focused on retrieving documents from your cluster
   that contain text that is semantically related to a provided query. However,
   you can create a vector search index on embeddings that represent any type of
   data that you might write to your cluster, such as images or videos.

   .. _vector-search-quickstart-vector-index-definition:

   Vector Index Definition
   ~~~~~~~~~~~~~~~~~~~~~~~~

   .. selected-content::
      :selections: atlas, byo, None, None

      An `index <https://www.mongodb.com/docs/manual/indexes/>`__ is a data structure
      that holds a subset of data from a collection's documents that improves database
      performance for specific queries. A :ref:`vector search index <avs-types-vector-search>`
      points to the fields that contain your vector embeddings and includes the
      dimensions of your vectors as well as the function used to measure similarity
      between vectors of queries and vectors stored in the database.

      Because the ``voyage-3-large`` embedding model used in this quick start
      converts data into vector embeddings with 2048 dimensions and supports the
      ``cosine`` function, this vector search index specifies the same number of
      vector dimensions and similarity function.

   .. selected-content::
      :selections: atlas, auto, None, None

      The ``autoEmbed`` type index configures Automated Embedding for the 
      ``fullplot`` field using the ``voyage-4`` embedding model in the 
      index definition. If you don't specify an embedding model in the query, 
      {+avs+} uses the embedding model specified in the index definition for 
      generating embeddings for the query text.

   Vector Search Query
   ~~~~~~~~~~~~~~~~~~~~

   The query you ran in this quick start is an `aggregation pipeline 
   <https://www.mongodb.com/docs/manual/core/aggregation-pipeline>`__,
   in which the ``$vectorSearch`` stage performs an :ref:`Approximate 
   Nearest Neighbor (ANN) <vectorSearch-ann>` search followed by a 
   ``$project`` stage that refines the results. To see all the options 
   for a semantic search query, including using :ref:`Exact Nearest 
   Neighbor (ENN) <vectorSearch-enn>` or how to narrow the scope of 
   your semantic search with the ``filter`` option, see :ref:`Run Vector 
   Search Queries <return-vector-search-results>`.
