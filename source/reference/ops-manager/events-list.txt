================================
mongocli ops-manager events list
================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/styles/corrections.rst

The ``events list`` command retrieves information about events for a
given |mms| project or organization. You must specify either a project or 
an organization in the command.

Syntax 
------

.. code-block:: text 

   mongocli ops-manager events list|ls
        [ --output|-o <output-format> ] 
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]
        [ --orgId <organization-ID> ]
        [ --limit <items-per-page> ]
        [ --maxDate <max-date> ]
        [ --minDate <min-date> ]
        [ --page <page-num> ]

.. include:: /includes/fact-command-line-help.rst

Options 
-------

Specify either a project with ``--projectId`` or an organization with
``--orgId``.

.. include:: /includes/list-table-events-list-options.rst

.. _mcli-om-events-list-output:

Output 
------ 

.. include:: /includes/command-output-intro.rst

``events list`` returns an array of documents, each representing one
event within your |mms| project or organization.

.. note::

   A subset of the following fields may appear, depending on the event type.

.. include:: /includes/list-table-events-response-document.rst

Example
-------

The following example uses the ``mongocli ops-manager events list`` command 
to retrieve information about events on the project with project ID
``5dd5a6b6f10fab1d71a58495``. It uses credentials stored in the {+default-profile+}.

.. code-block:: none

   mongocli ops-manager events list --projectId 5dd5a6b6f10fab1d71a58495 --output json
