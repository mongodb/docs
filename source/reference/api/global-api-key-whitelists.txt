======================================
Global Programmatic API Key Whitelists
======================================

.. default-domain:: mongodb

.. include:: /includes/api/base-api-uri.rst

.. include:: /includes/styles/corrections.rst

Use the ``/admin/whitelist`` resource to view, create, or delete
Global Whitelist Entries, used in conjunction with :ref:`Global Programmatic API Keys <mms-prog-api-key>`.

.. seealso::

   To learn more about Global Whitelist Entries, see
   :ref:`Global Whitelists <whitelist-page-admin-ui>`.

Permissions
-----------

To use these endpoints, grant the following roles to an Project |api|
Key or user in your Organization:

.. list-table::
   :header-rows: 1
   :widths: 30 70

   * - Role
     - Possible Action

   * - Any Global role
     - Retrieve Global Whitelist Entries
   * - :authrole:`Global Owner` or :authrole:`Global User Admin`
     - Create, update, or delete a Global Whitelist Entry

Endpoints
---------

.. list-table::
   :header-rows: 1
   :stub-columns: 1
   :widths: 10 40 50

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/admin/whitelist </reference/api/api-keys/global/get-all-global-whitelist>`

     - Get all global whitelist entries.

   * - ``GET``

     - :doc:`/admin/whitelist/{WHITELIST-ID} </reference/api/api-keys/global/get-one-global-whitelist>`

     - Get one global whitelist entry.

   * - ``POST``

     - :doc:`/admin/whitelist </reference/api/api-keys/global/create-one-global-whitelist>`

     - Create one global whitelist entry.

   * - ``PATCH``

     - :doc:`/admin/whitelist/{WHITELIST-ID} </reference/api/api-keys/global/update-one-global-whitelist>`

     - Update the global whitelist entry specified by ``{WHITELIST-ID}``.

   * - ``DELETE``

     - :doc:`/admin/whitelist/{WHITELIST-ID} </reference/api/api-keys/global/delete-one-global-whitelist>`

     - Delete the global whitelist entry specified by ``{WHITELIST-ID}``.
