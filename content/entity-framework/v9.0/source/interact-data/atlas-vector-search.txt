Overview
--------

You can use {+vector-search+} with the {+provider-short+} to perform
semantic search on data stored in Atlas. Vector search queries your
data based on semantic meaning rather than keyword matches, allowing
you to retrieve more relevant results. You can use {+vector-search+} to
support use cases such as semantic search, hybrid search, and
Retrieval-Augmented Generation (RAG).

To learn more about {+vector-search+}, see the :atlas:`{+vector-search+}
Overview </atlas-vector-search/vector-search-overview/>` in the Atlas
documentation.

.. _entity-framework-vector-embedding-types:

Supported Vector Embedding Types
--------------------------------

The {+provider-short+} supports vector embeddings of the
following types:

Array Representations
~~~~~~~~~~~~~~~~~~~~~

The {+provider-short+} supports the following array types as vector
embeddings:

- ``float[]``
- ``double[]``
- ``Memory<float>`` and ``ReadOnlyMemory<float>``
- ``Memory<double>`` and ``ReadOnlyMemory<double>``

Binary Representations
~~~~~~~~~~~~~~~~~~~~~~

The {+provider-short+} supports the following binary vector types, which
reduce storage and improve query performance compared to array types:

- ``BinaryVectorFloat32`` — full-precision 32-bit floats
- ``BinaryVectorInt8`` — 8-bit quantized integers
- ``BinaryVectorPackedBit`` — 1-bit packed binary values

The ``MongoDB.Driver`` namespace of the {+csharp-driver-short+} contains
the ``VectorQuantization`` and ``VectorSimilarity`` enums used to
configure binary vector indexes.
