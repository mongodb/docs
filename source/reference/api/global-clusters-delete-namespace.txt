==========================
Delete a Managed Namespace
==========================

.. default-domain:: mongodb

.. meta::

Delete a managed namespace for a :doc:`Global Cluster </global-clusters>`.
Deleting a managed namespace does not delete the collection associated
with the namespace. Use the :guilabel:`Data Explorer` to delete both the
collection and the managed namespace. Alternatively, use this endpoint
to delete the managed namespace and {+mongosh+} to delete the
collection.

.. note::

   Deleting a managed namespace does not unshard the associated collection.

For more information about managed namespaces, see 
:doc:`Global Clusters </reference/api/global-clusters>`.

Syntax
------

.. code-block:: none

   DELETE /groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/globalWrites/managedNamespaces

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1
   :widths: 15 10 75

   * - Path Element
     - Required/Optional
     - Description

   * - ``GROUP-ID``

     - Required.

     - Unique identifier for the :ref:`project <group-id>` that
       contains the |global-write-cluster|.

   * - ``CLUSTER-NAME``

     - Required.

     - Name of the |global-write-cluster|.

Request Query Parameters
~~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/base-query-list-parameters.rst

.. list-table::
   :widths: 20 14 11 45 10
   :stub-columns: 1

   * - collection
     - string
     - Optional
     - Name of the collection associated with the managed namespace.
     -

   * - db
     - string
     - Optional
     - Name of the database containing the collection.
     -
