.. _mcli-om-admin-backup-s3-delete-cmd:

===========================================
mongocli ops-manager admin backup s3 delete
===========================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/admonitions/admin-requires-global-owner.rst

The ``admin backup s3 delete`` command deletes the specified backup 
:opsmgr:`S3 blockstore </reference/glossary/#term-s3-snapshot-store>` 
configuration. You can also delete an |s3| configuration using 
the :opsmgr:`UI </tutorial/manage-s3-blockstore-storage/#delete-one-s3-snapshot-store>` 
and the :opsmgr:`API </reference/api/admin/backup/snapshot/s3Configs/delete-one-s3-blockstore-configuration/>`.

.. _mcli-om-admin-backup-s3-delete-syntax: 

Syntax 
------

.. code-block:: shell 

   mongocli ops-manager|om admin backup s3 delete|rm <s3-name>
        [ --force ]
        [ --output|-o <output-format> ]
        [ --profile|-p <profile-name> ]

.. _mcli-om-admin-backup-s3-delete-args:

Arguments 
---------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``<s3-name>``
     - string 
     - Unique identifier of the |s3| blockstore configuration that you
       want to delete. You can retrieve a list of |s3| names for 
       a deployment with the 
       :ref:`List Backup S3 Configurations <mcli-om-admin-backup-s3-list-cmd>` command.
     - yes

.. _mcli-om-admin-backup-s3-delete-options:

Options 
-------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--force`` 
     - 
     - Flag indicating that the alert configuration can 
       be deleted without requiring confirmation.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-p``
     - string
     - Name of the profile where the public and private 
       keys for accessing |onprem| are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

.. _mcli-om-admin-backup-s3-delete-output: 

Output 
------

If successful, the command returns the following message:

.. code-block:: shell 
   :copyable: false 

   s3 configuration '<s3-name>' deleted.

.. _mcli-om-admin-backup-s3-delete-egs:

Example 
-------

The following example uses the ``mongocli om admin backup s3 
delete`` command to delete the |s3| blockstore configuration 
``s3BackupExample``. The command uses the ``--force`` option to 
bypass the confirmation prompt and the {+default-profile+}, which 
contains :authrole:`Global Owner` |mms| credentials.

.. code-block:: sh 
   
   mongocli om admin backup s3 delete s3BackupExample --force

The previous command returns the following: 

.. code-block:: shell
   :copyable: false

   s3 configuration 's3BackupExample' deleted.
