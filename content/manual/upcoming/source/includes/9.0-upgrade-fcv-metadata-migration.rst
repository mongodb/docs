.. note::

   When you upgrade the {+fcv+} to 9.0, MongoDB migrates metadata from
   the config server to the shards. This one time migration can make
   the upgrade take longer on clusters with many databases,
   collections, or chunks.

   While metadata for a database or tracked collection is being cloned,
   operations that modify, rename, reshard, or drop that database or
   collection must wait until cloning completes.
