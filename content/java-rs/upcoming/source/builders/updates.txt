Overview
--------

In this guide, you can learn how to specify **updates** by using
:ref:`builders <java-rs-builders>` in the {+driver-short+}.

The ``Updates`` builder provides helper methods to simplify the following
tasks:

- :ref:`java-rs-field-updates`: Updating or removing the value of a particular field
- :ref:`java-rs-array-updates`: Updating values in an array-valued field
- :ref:`java-rs-multiple-updates`: Performing multiple updates at once, such as setting or changing more than one field

Some methods that expect update documents are:

- ``updateOne()``
- ``updateMany()``
- ``bulkWrite()``

The ``Updates`` class provides static factory methods for all the MongoDB update
operators. Each method returns an instance of the
:driver:`BSON </java/sync/current/data-formats/document-data-format-bson/>`
type, which you can pass to any method that expects an update argument.

.. tip::

   For brevity, you may choose to import the methods of the `Updates
   <{+core-api+}/client/model/Updates.html>`__ and `Filters
   <{+core-api+}/client/model/Filters.html>`__ class statically:

   .. literalinclude:: /includes/builders/updates.java
      :language: java
      :dedent:
      :start-after: begin-static-import
      :end-before: end-static-import

   The following examples assume this static import.

The examples in this guide use the following document:

.. code-block:: json

   {
       "_id": 1,
       "color": "red",
       "qty": 5,
       "vendor": [ "A", "D", "M" ],
       "lastModified": { "$date": "2021-03-05T05:00:00Z" }
   }

.. literalinclude:: /includes/builders/updates.java
   :language: java
   :dedent:
   :start-after: begin-bitwise-or-update
   :end-before: end-bitwise-or-update

.. code-block:: none
   :copyable: false

   :copyable: false

   }
