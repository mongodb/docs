Update the Automation Configuration

   :local:

   :class: singlecol

Overview
--------

A project's :doc:`automation configuration </reference/cluster-configuration>`
determines the goal state of its MongoDB processes and agents. The Automation
Agents build the deployment according to the goals specified.

To update the configuration, retrieve the current configuration and make
changes as needed, changing only those items you want modified. Then replace
the entire configuration using ``PUT`` with the :doc:`automationConfig
</reference/api/automation-config>` endpoint. You must use ``PUT``.
**Do not** use ``PATCH``.

.. note::

   Some attributes of the `backupVersions` and `monitoringVersions` objects
   cannot be updated through the ``automationConfig`` endpoint.
   For details, see :ref:`update-monitoring-or-backup-agent-via-api`.

Prerequisites
-------------

You must have access to the Public API. For more information, see
:doc:`/tutorial/configure-public-api-access`.
