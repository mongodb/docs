===================================
Set Up a Network Peering Connection
===================================

.. default-domain:: mongodb

.. facet::
   :name: genre
   :values: tutorial

.. dismissible-skills-card::
   :skill: Networking Security: Atlas
   :url: https://learn.mongodb.com/skills?openTab=security

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

Use this page to set up, view, and remove network peering connections
between your |service| project and your cloud provider network. For
an overview of network peering, including limitations and best
practices, see :ref:`vpc-peering`.


.. note::

   .. sharedinclude:: cet/atlas/shared-responsibility-model-intro.rst

.. composable-tutorial::
   :options: interface, cloud-provider
   :defaults: atlas-ui, aws

   Required Access
   ---------------

   To set up a network peering connection, you must have
   :authrole:`Organization Owner`, :authrole:`Project Owner`, or
   :authrole:`Project Network Access Manager` access to the project.
