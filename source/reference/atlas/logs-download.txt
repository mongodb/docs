.. _mcli-atlas-logs-download-cmd:

============================
mongocli atlas logs download
============================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``logs download`` command downloads the logs for a specified 
|service| host to a ``.gz`` file. You can also download 
logs through the |service| :atlas:`UI </mongodb-logs/>` or 
:atlas:`API </reference/api/logs/>`.

.. _logs-download-syntax:

Syntax
------

.. code-block:: text
   :copyable: false

   mongocli atlas logs|log download <hostname> <log-name>
        [ --start <start-date> ]
        [ --end <end-date> ]
        [ --out <file-destination>]
        [ --profile|-P <profile-name> ]
        [ --projectId <id-of-project> ]
	    

.. include:: /includes/fact-command-line-help.rst

.. _logs-download-options:

Options 
-------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type
     - Description  
     - Required?

   * - ``--end``
     - string
     - |Epoch-time| that specifies the end point for the range of log 
       messages to retrieve. Default is current timestamp.
     - no

   * - ``<hostname>``
     - string
     - Name of the MongoDB host for which to download logs.
       
       To find the ``hostnames`` for a |service| project, you can use 
       the :ref:`process list <mcli-atlas-process-list-cmd>` 
       ``mongocli`` command. You can also find ``hostnames`` through 
       the |service| :atlas:`API </reference/api/processes-get-all/>` 
       or UI :guilabel:`Overview` tab for a cluster.
     - yes

   * - ``<log-name>``
     - string
     - The name of the log file to retrieve. Valid values are:

       - :manual:`mongodb.gz </reference/program/mongod/>`

       - :manual:`mongos.gz </reference/program/mongos/>`

       - ``mongodb-audit-log.gz``

       - ``mongos-audit-log.gz``

       .. note::
	   
		  Audit logs are only available if you have enabled 
		  :atlas:`Database Auditing </database-auditing/>` for an 
		  |service| project.
     - yes

   * - ``--out``
     - string
     - Output filename or path to filename. Value must be in the 
       following format: 
       
       .. code-block:: text 
          :copyable: false 

          <file-name>.<extension>.gz

       If omitted, {+mcli+} downloads logs to your current directory as 
       ``<log-name>.log.gz``.
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no

   * - ``--start``
     - string
     - |Epoch-time| that specifies starting point for the range of log 
       messages to retrieve. Default is 24 hours prior to the current 
       timestamp.
     - no

.. _logs-download-output:

Output 
------

The command downloads the logs of the specified ``<hostname>`` to:

- The current directory as a file named ``<log-name>.gz``, or 

- A directory and filename you specify with the ``--out`` option.

If the command 
returns errors, see :ref:`Troubleshooting <mcli-troubleshooting>` for 
recommended solutions.

.. _logs-download-examples:

Examples 
--------

The following example uses the ``mongocli atlas logs download`` command 
to download the ``mongodb`` logs from host 
``atlas-123abc-shard-00-00.111xx.mongodb.net``. It uses the {+default-profile+} to access the project.

.. code-block:: text
   :copyable: false

   mongocli atlas logs download atlas-123abc-shard-00-00.111xx.mongodb.net mongodb.gz

The previous command downloads the log file ``mongodb.gz`` to the 
current directory.
