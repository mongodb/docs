Overview
--------

This page reproduces patterns that previously triggered false positive
comments from the AI style reviewer.

.. _ai-regression-pattern-1:

Pattern 1: Sentence with colon before a code block
---------------------------------------------------

A sentence ending with "the following prototype form:" is a valid
list introduction and must not be flagged.

The ``$iceberg`` pipeline stage has the following prototype form:

.. code-block:: json

   {
     "$iceberg": {
       "connectionName": "<registered-connection>",
       "bucket": "<target-bucket>",
       "databaseName": "<database>",
       "tableName": "<string>",
       "mode": "insert" | "cdc"
     }
   }

.. _ai-regression-pattern-2:

Pattern 2: Wrapped sentence with colon on the continuation line
---------------------------------------------------------------

The colon ending this introduction lands on the second RST line
because the first line fills 72 characters. It must not be flagged.

To sort each document into a distinct Iceberg table, you can write
the following ``$iceberg`` stage:

.. code-block:: json

   {
     "$iceberg": {
       "connectionName": "iceberg1",
       "tableName": "$customerStatus"
     }
   }

.. _ai-regression-pattern-3:

Pattern 3: Code-only bullet items require no terminal punctuation
-----------------------------------------------------------------

Items that consist solely of inline code must not be flagged for
missing periods. The introduction below is valid.

The ``mode`` field must be one of the following:

- ``"identity"``
- ``"year"``
- ``"month"``
- ``"day"``
- ``"hour"``
- ``{ truncate: int }``
- ``{ bucket: int }``

.. _ai-regression-pattern-4:

Pattern 4: list-table description cells must not be mixed with bullet lists
---------------------------------------------------------------------------

The description cells in the following list-table end with periods.
The bullet list below them also uses periods. Neither should trigger
a punctuation consistency violation because they are separate structures.

.. list-table::
   :header-rows: 1
   :widths: 20 10 70

   * - Field
     - Type
     - Description

   * - ``connectionName``
     - string
     - Name of the connection to use for reads and writes.

   * - ``bucket``
     - string
     - Name of the bucket containing the target database.

   * - ``mode``
     - string
     - Strategy for determining which operation to perform per document.

The following list uses periods and is consistent:

- Install the driver.
- Copy your connection string.
- Run the sample application.

.. _ai-regression-pattern-5:

Pattern 5: Mid-sentence colon followed by more prose
----------------------------------------------------

A colon that appears mid-sentence followed by more prose is not a
list introduction and must not be flagged.

The following tutorial uses two files: an application file and a
helpers file. You can find links to the complete versions of both
files in the Full Application Code section.

.. _ai-regression-pattern-6:

Pattern 6: Wrapped sentence ending with colon before a list
-----------------------------------------------------------

The colon here lands on the continuation line after a 72-character
first line. The introduction is valid and must not be flagged.

The code samples in this tutorial use the following variables to
perform the queryable encryption workflow:

- ``kms_provider_name``
- ``uri``
- ``key_vault_database_name``
- ``key_vault_collection_name``

.. _ai-regression-pattern-7:

Pattern 7: Past date in a release notes heading
-----------------------------------------------

June 10, 2026 is in the past relative to the current date and must
not be flagged as a future date.

.. _regression-test-release-notes:

1.0.0 - June 10, 2026
~~~~~~~~~~~~~~~~~~~~~

Issues fixed:

- issue 1
- issue 2
- issue 3

.. _ai-regression-pattern-8:

Pattern 8: Prose paragraphs ending with periods
------------------------------------------------

The prose example has sentences that end with periods. The content
isn't a list and must not trigger the list item punctuation
consistency rule.

Specifying a schema for a collection in MongoDB isn't required.
Although it is common for documents in a collection to have a
homogeneous structure, it isn't a requirement. Documents in a single
collection can have different fields. The data type for a document
field can differ across documents in a collection.
