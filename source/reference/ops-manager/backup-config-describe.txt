.. _mcli-om-backup-config-describe-cmd:

===========================================
mongocli ops-manager backup config describe
===========================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``backup config describe`` command retrieves the backup 
configuration details for the specified cluster. You can also retrieve 
the backup configuration details for a cluster using the :opsmgr:`API 
</reference/api/backup/get-one-backup-config-by-cluster-id/>`.

.. _mcli-om-backup-config-describe-syntax:

Syntax 
------

.. code-block:: shell 

   mongocli ops-manager|om backup config describe|get <cluster-ID>
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. _mcli-om-backup-config-describe-args: 

Arguments
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required? 

   * - ``<cluser-ID>``
     - string
     - Unique identifier of the cluster whose backup 
       configuration you want to retrieve. You can use the 
       :ref:`mcli-om-cluster-list-command` to retrieve the 
       ID of a cluster.
     - yes


.. _mcli-om-backup-config-describe-options:

Options 
-------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option 
     - Type 
     - Description 
     - Required? 

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the project ID and the |svc-api-key|\s 
       for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a profile, see 
       :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - .. include:: /includes/extracts/fact-basic-options-project-id.rst
     - no

.. _mcli-om-backup-config-describe-output: 

Output 
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see :ref:`Troubleshooting 
<mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh 
   :copyable: false 

   PROJECT ID     CLUSTER ID     STATUS     STORAGE ENGINE
   <project-ID>   <cluster-ID>   <status>   <storage-engine>

The default output contains a subset of the fields returned by this 
command. For the complete list of |json| fields returned by the 
command, see the :atlas:`API reference 
</reference/api/backup/get-one-backup-config-by-cluster-id/#response>`.

.. _mcli-om-backup-config-describe-egs:

Example 
-------

The following example uses the ``mongocli ops-manager backup config 
describe`` command to retrieve the backup configuration details for the 
specified cluster. The command uses the {+default-profile+} to access 
the project.

.. tabs:: 

   .. tab:: Default Output 
      :tabid: default

      .. code-block:: shell

         mongocli ops-manager backup config describe 5fabae1d98bd2e1eddb875b3

      The previous command returns the following: 

      .. code-block:: shell
         :copyable: false 

         PROJECT ID                 CLUSTER ID                 STATUS     STORAGE ENGINE
         5faba4b381b378146d110f67   5fabae1d98bd2e1eddb875b3   STARTED    WIRED_TIGER

   .. tab:: JSON Output 
      :tabid: json

      .. code-block:: shell

         mongocli ops-manager backup config describe 5fabae1d98bd2e1eddb875b3 --output json

      The previous command returns the following:

      .. code-block:: shell
         :copyable: false 

         {
           "groupId": "5faba4b381b378146d110f67",
           "clusterId": "5fabae1d98bd2e1eddb875b3",
           "statusName": "STARTED",
           "storageEngineName": "WIRED_TIGER",
           "authMechanismName": "NONE",
           "encryptionEnabled": false,
           "sslEnabled": false
         }
