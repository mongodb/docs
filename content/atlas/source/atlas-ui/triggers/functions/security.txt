========
Security
========

.. meta::
   :description: Triggers make calls to external resources from one of the listed IP addresses.

.. facet::
   :name: genre
   :values: reference

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol


Encryption
----------

Function calls to Atlas, such as queries and database operations, are secured
with |tls| encryption. For more information, see :ref:`atlas-faq-security`.

IP Access
---------

If you create a function that makes outgoing HTTP requests, you might need to
configure the external service's firewall IP access list to allow these calls.
This is not a configuration change you make in |service|, but rather on the
servers that host the external endpoint. Depending on which cloud provider
you use, refer to one of the following IP address lists. The lists are also available as
downloadable JSON and CSV files, which are linked below for each
provider.

.. note::

   The IP addresses in the lists are subject to change. For the most
   up-to-date list of outbound IP addresses to the Atlas control plane, you can use the `Admin API <https://www.mongodb.com/docs/api/doc/atlas-admin-api-v2/operation/operation-listcontrolplaneipaddresses>`__.
