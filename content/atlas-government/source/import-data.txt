.. _import-data:

========================================
Migrate or Import Data into Your Cluster
========================================

.. meta::
   :description: Learn how to migrate or import data into your MongoDB Atlas for Government cluster using the Atlas Live Migration Service with minimal application downtime.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The Atlas Live Migration Service can migrate your data from your
existing environment to |cloudgov| with minimal impact to your
app. Live Migration Service procedures for |service| are also
applicable to |cloudgov-short|, with the following exceptions:

- To migrate from commercial |service| to |cloudgov-short|, navigate to the
  `Migration Hub <https://cloud.mongodb.com/go?l=https%3A%2F%2Fcloud.mongodb.com%2Fv2%2F%3Cproject%3E%23%2Fmigration>`__
  and use the :guilabel:`Migrate from Self-Managed` option, not the
  :guilabel:`Migrate from Atlas` option. When providing the source
  connection string, append ``?tls=true``.
- To migrate sharded clusters running MongoDB 8.0 or later, provide
  the hostname and port of the ``mongos`` routers in your connection
  string instead of the individual ``mongod`` hostnames.
- The Live Migration Service doesn't support migrating using private
  endpoints or |vpc| peering. If your migration requires these network
  configurations, use `Mongosync
  <https://www.mongodb.com/docs/mongosync/current/>`__ instead.

For an overview of the service and tutorials for common migrations, see
`Atlas Live Migration <https://www.mongodb.com/cloud/atlas/migrate/>`__.

.. tip:: |service| documentation

   For detailed information on importing data, see 
   :atlas:`Migrate or Import Data into Your Cluster </import/>`.
