==============
Database Users
==============

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: twocols

.. include:: /includes/api-base-url.rst

The ``databaseUsers`` resource lets you retrieve, create and modify the
MongoDB users in your cluster. Each user has a set of roles that provide
access to the group's databases. A user's roles apply to all the clusters in
the group: if two clusters have a ``products`` database and a user has a role
granting ``read`` access on the ``products`` database, the user has that
access on both clusters.

Each user also has a username, password, and :ref:`authentication database
<authentication-database>` used to log into MongoDB. |service| deployments of
MongoDB use ``admin`` as the authentication database for all users. The
authentication database does not determine a user's roles.

The ``databaseUsers`` resource requires your :ref:`group-id`.

     - Description

   * - ``GET``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/databaseUsers </reference/api/database-users-get-all-users>`

     - Get all users in the group

   * - ``GET``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/databaseUsers/admin/{USERNAME} </reference/api/database-users-get-single-user>`

     - Get a single user in the group

   * - ``POST``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/databaseUsers </reference/api/database-users-create-a-user>`

     - Create a user for the group

   * - ``PATCH``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/databaseUsers/admin/{USERNAME} </reference/api/database-users-create-a-user>`

     - Update a user for the group.

   * - ``DELETE``

     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/databaseUsers/admin/{USERNAME} </reference/api/database-users-delete-a-user>`

     - Delete a user for the group.
