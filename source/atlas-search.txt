======================
What is MongoDB |fts|?
======================

.. default-domain:: mongodb

.. introduction:: 

   |fts| is an embedded full-text search in |service-fullname| that 
   gives you a seamless, scalable experience for building 
   relevance-based app features. Built on Apache Lucene, |fts| 
   eliminates the need to run a separate search system alongside your 
   database.

   .. button:: Get Started
      :uri: /atlas-search/tutorial

.. image:: /images/atlas-search-plp/atlas-search-plp-hero.svg 
   :alt: MongoDB Atlas Search landing page hero image

.. kicker:: What You Can Do

Integrated Search in |service-fullname|
---------------------------------------

Easily build search on top of your data with an integrated, fully 
managed search engine that automatically syncs to your database.

.. tabs::

   .. tab:: Get Started With |fts|
      :tabid: atlas-search-get-started

      .. procedure::

         .. step:: Create Search Indexes

            Create index definitions for your data using editors in the 
            |service| UI, with a single |service| Admin API call, or 
            using the |mcli|.

            To learn more, see :ref:`Atlas Search Index Definition 
            <ref-index-definitions>`

         .. step:: Construct Search Queries

            Build and run |fts| queries using the :pipeline:`$search` 
            MongoDB aggregation pipeline stage.

            To learn more, see :ref:`searching-ref`.

         .. step:: Implement Best Practices

            Refer to our guidelines on scaling and monitoring |fts| 
            workloads.

            To learn more, see :ref:`perf-ref`.

      .. image:: /images/atlas-search-plp/get-started.png
         :alt: MongoDB Atlas Search get started image

   .. tab:: Use Rich Indexing Functionality
      :tabid: atlas-searchi-ndexing

      .. procedure::

         .. step:: Define Field Mappings

            Index field values to run queries that use 
            :ref:`autocomplete-ref`, :ref:`text-ref`, :ref:`range-ref`, 
            and other |fts| :ref:`operators-ref`.

            To learn more, see :ref:`bson-data-types`.

         .. step:: Pre-Process Data With Analyzers

            Define how |fts| creates search terms using built-in 
            analyzers or create your own. 

            To learn more, see :ref:`Atlas Search Analyzers 
            <analyzers-ref>`.

         .. step:: Add Synonyms for Search Terms

            Configure synonyms to index and search collections for 
            words that have the same or similar meanings.

            To learn more, see :ref:`synonyms-ref`.

      .. image:: /images/atlas-search-plp/rich-indexing.gif
         :alt: MongoDB Atlas Search create index UI video

   .. tab:: Deliver Fast, Relevant Results
      :tabid: atlas-search-results

      .. procedure::

         .. step:: Build Complex Queries With Compound

            Combine two or more operators and multiple clauses in a 
            single search query using the :ref:`compound-ref` operator. 

            To learn more, see :ref:`compound-query-custom-score-tutorial`.

         .. step:: Customize Relevance Scores

            Modify the ranking of |fts| results to boost certain 
            documents or match relevance requirements.

            To learn more, see :ref:`scoring-ref`.

         .. step:: Filter and Parse Results Quickly

            Use :ref:`facets <fts-facet-ref>` to group and count |fts| 
            results by multiple categories.

            To learn more, see :ref:`facet-tutorial`.

      .. image:: /images/atlas-search-plp/fast-relevant-results.gif
         :alt: MongoDB Atlas Search custom scoring video
