.. _avs-view-index:

====================================
View an {+avs+} Index
====================================

.. default-domain:: mongodb

.. meta::
   :keywords: atlas vector search, view an atlas vector search index, view an atlas vector search index using the atlas ui, view an atlas vector search index using the atlas vector search ui
   :description: View an Atlas Vector Search index using the Atlas UI, mongosh, or MongoDB driver.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/facts/fts-index-description.rst

You can view {+avs+} indexes for all collections from the
{+atlas-ui+} and {+atlas-admin-api+}.

.. _avs-view-index-access:

Required Access
---------------

You need the :authrole:`Project Search Index Editor` or higher role to view
{+avs+} indexes. 

Supported Clients 
-----------------

You can view an {+avs+} index using the following: 

- {+atlas-ui+}
- {+atlas-admin-api+} :oas-atlas-op:`Return One Atlas Search Index
  <getAtlasSearchIndex>` endpoint
- {+atlas-cli+} v1.14.3 :atlascli:`atlas clusters search indexes list
  </command/atlas-clusters-search-indexes-list/>` command on both the
  :atlascli:`cloud deployment </atlas-cli-local-cloud/>` and
  :atlascli:`local deployment </atlas-cli-deploy-local>`
- {+mongosh+} v2.1.2 or later
  :method:`db.collection.getSearchIndexes()` method

.. _avs-view-index-procedure:

Procedure 
---------

.. tabs:: 

   .. tab:: Atlas UI 
      :tabid: ui 

      .. include:: /includes/steps-avs-view-index-atlas-ui.txt

      The page displays the following details for the indexes on the page: 

      .. list-table:: 
         :stub-columns: 1 
         :widths: 20 80

         * - Name 
           - Label that identifies the index. 

         * - Index Type
           - Label that indicates an |fts| or {+avs+} index. Values
             include:  
     
             - ``search`` for |fts| indexes.
             - ``vectorSearch`` for {+avs+} indexes.

         * - Index Fields
           - List that contains the fields that this index indexes.

         * - Status
           - Current state of the index on the primary node of the
             {+cluster+}. For valid values, see
             :ref:`avs-node-status-ref`. 

         * - Size
           - Size of the index on the primary node.

         * - Documents 
           - Number of indexed documents out of the total number of
             documents in the collection.

         * - Actions 
           - Actions that you can take on the index. You can:
     
             - :ref:`avs-edit-index`
             - :ref:`avs-delete-index`

             .. note:: 

                You can't run queries in the :guilabel:`Search Tester` UI
                against indexes of the ``vectorSearch`` type. If you click the
                :guilabel:`Query` button, {+avs+} displays a sample
                :pipeline:`$vectorSearch` that you can copy, modify, and run
                in :guilabel:`Data Explorer` and using other
                :ref:`supported clients <vectorSearch-agg-pipeline-clients>`. 

   .. tab:: Atlas Administration API 
      :tabid: api 

      To retrieve all the {+avs+} indexes for a collection using the
      {+atlas-admin-api+}, send a ``GET`` request to the |fts|
      ``indexes`` endpoint with the name of the database and collection.
      To learn more about the syntax and parameters for the endpoint,
      see :oas-atlas-op:`Return All Atlas Search Indexes for One
      Collection </listAtlasSearchIndexes>`. 

      To retrieve one {+avs+} index for a collection using the
      {+atlas-admin-api+}, send a ``GET`` request to the |fts|
      ``indexes`` endpoint with the unique ID of the index to retrieve. 
      To learn more about the syntax and parameters for the endpoint,
      see :oas-atlas-op:`Return All Atlas Search Indexes for One
      Collection </getAtlasSearchIndex>`. 

   .. tab:: Atlas CLI 
      :tabid: atlascli 

      To return {+avs+} indexes for a collection using {+atlas-cli+},
      run the following command: 

      .. code-block:: shell 
         :copyable: true
      
         atlas clusters search indexes list [options]
      
      To learn more about the command syntax and parameters, see the
      {+atlas-cli+} documentation for :atlascli:`atlas clusters search
      indexes list </command/atlas-clusters-search-indexes-list/>`
      command. 

   .. tab:: mongosh 
      :tabid: mongosh 

      To delete an {+avs+} index for a collection using {+mongosh+}, do
      the following:
      
      1. :ref:`Connect <connect-mongo-shell>` to the |service|
         {+cluster+} using {+mongosh+}.   
      #. Switch to the database that contains the collection.
      #. Run the :method:`db.collection.getSearchIndexes()` method. 

         The :method:`db.collection.getSearchIndexes()` method has the
         following syntax: 

         .. code-block:: shell 
            :copyable: true 
            :linenos: 

            db.movies.getSearchIndexes( "<index-name>" );
