MongoDB stores data records as :term:`documents <document>`
(:ref:`BSON documents <bson-document-format>`) in
:term:`collections <collection>`. A :term:`database <database>`
holds one or more collections.

You can manage :atlas:`databases </atlas-ui/databases>` and
:atlas:`collections </atlas-ui/collections>` using the Atlas UI,
:binary:`~bin.mongosh`, or |compass|. This page covers Atlas UI
procedures. For self-managed deployments, use
:binary:`~bin.mongosh` or |compass|.

Create a Database
~~~~~~~~~~~~~~~~~

.. tabs::
   :hidden:

   .. tab:: Atlas UI
      :tabid: atlas

      .. sharedinclude:: cet/atlas/data-explorer/steps-atlas-ui-create-database.rst

   .. tab:: mongosh
      :tabid: mongosh

      MongoDB creates the database when you first store data
      for it. Switch to a non-existent database and run:

.. code-block:: javascript

   use myNewDB

   db.myNewCollection1.insertOne( { x: 1 } )

      :method:`~db.collection.insertOne()` creates both the
      database ``myNewDB`` and the collection
      ``myNewCollection1`` if they do not already exist. Be
      sure that both names follow MongoDB
      :ref:`restrictions-on-db-names`.

   .. tab:: MongoDB Compass
      :tabid: compass

      .. procedure::
         :style: normal

         .. step:: Open the :guilabel:`Databases` tab.

         .. step:: Click :guilabel:`Create database`.

         .. step:: Enter the database and collection names.

         .. step:: Click :guilabel:`Create Database`.

.. _collections:

Collections
-----------

MongoDB stores documents in collections. Collections are
analogous to tables in relational databases.

.. include:: /images/crud-annotated-collection.rst

Create a Collection
~~~~~~~~~~~~~~~~~~~

If a collection does not exist, MongoDB creates the collection
when you first store data for that collection.

.. tabs::
   :hidden:

   .. tab:: Atlas
      :tabid: atlas

      .. sharedinclude:: cet/atlas/data-explorer/steps-atlas-ui-create-collection.rst

   .. tab:: mongosh
      :tabid: mongosh

.. code-block:: javascript

   db.myNewCollection2.insertOne( { x: 1 } )
   db.myNewCollection3.createIndex( { y: 1 } )

      Both :method:`~db.collection.insertOne()` and
      :method:`~db.collection.createIndex()` create their
      respective collection if it does not already exist.
      Be sure the collection name follows MongoDB
      :ref:`restrictions-on-db-names`.

   .. tab:: MongoDB Compass
      :tabid: compass

      .. procedure::
         :style: normal

         .. step:: In the left navigation, click the database name.

         .. step:: Click :guilabel:`+` next to the database name.

         .. step:: Enter a name in :guilabel:`Create Collection`.

         .. step:: Click :guilabel:`Create Collection`.

.. tabs::
   :hidden:

   .. tab:: Atlas
      :tabid: atlas

   .. tab:: mongosh
      :tabid: mongosh

      Explicit Creation
      `````````````````

      Use :method:`db.createCollection()` to explicitly
      create a collection with options such as maximum size
      or validation rules. Without these options, MongoDB
      automatically creates collections when you first store
      data.

      To modify these collection options, see
      :dbcommand:`collMod`.

   .. tab:: MongoDB Compass
      :tabid: compass

      Explicit creation
      `````````````````

      .. procedure::
         :style: normal

         .. step:: In the left navigation, click the database name.

         .. step:: Click :guilabel:`Create collection`.

         .. step:: Enter the collection name and optional preferences.

         .. step:: Click :guilabel:`Create Collection`.

            |compass| supports the following additional
            preferences:

            - :compass:`Create a Capped Collection</collections/capped-collection/>`

            - :compass:`Create a Time Series Collection </collections/time-series-collection/>`

Schema Validation
~~~~~~~~~~~~~~~~~

By default, documents in a collection do not share a schema.
Fields and data types can vary across documents.

You can enforce :ref:`schema validation rules
<schema-validation-overview>` during insert and update
operations.

For {+atlas+} deployments, the :ref:`Performance Advisor
<performance-advisor>` and the {+atlas+} UI detect common
schema design issues and suggest modifications that follow
MongoDB best practices. To learn more, see
:atlas:`Schema Suggestions
</performance-advisor/schema-suggestions/#schema-suggestions>`.

Modifying Document Structure
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

To add, remove, or retype fields in a collection's documents,
update the existing documents.

.. _collections-uuids:

Unique Identifiers
~~~~~~~~~~~~~~~~~~

Collections are assigned an immutable
:abbr:`UUID (Universally unique identifier)` that remains
consistent across all replica set members and shards.

      :manual:`listCollections
      </reference/command/listCollections>` command or the
      :method:`db.getCollectionInfos()` method.
