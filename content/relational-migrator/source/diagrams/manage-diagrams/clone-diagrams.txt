.. _rm-clone-diagrams:

===============
Clone a Diagram
===============

.. meta::
   :description: Clone existing diagrams in Relational Migrator to create new ones with the same entities, and then edit, rename, or delete the new diagrams as needed.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can clone existing diagrams on the :guilabel:`Mapping` screen of
{+rel-mig+}. 

About this Task
---------------

- You can clone one or more diagrams to create new diagrams that contain
  the same entities as existing diagrams. 
- Once you clone a diagram, you can edit, :ref:`rename
  <rm-rename-diagrams>`, and :ref:`delete <rm-delete-diagrams>` your new
  diagram. 


Steps
-----

.. procedure::
   :style:  connected

   .. step:: Navigate to the diagram you want to clone

      Click the name of the diagram in the Diagram Toolbar to display the
      diagram. 

      .. image:: /img/diagrams/diagram-toolbar.png
         :alt: Diagram toolbar    

   .. step:: Open the :guilabel:`Diagram Properties` pane

      Click the :icon-fa5:`chevron-circle-right` arrow button to open the
      :guilabel:`Diagram Properties` pane. 

      .. image:: /img/diagrams/diagram-properties.png
         :alt: Diagram Properties pane

      The :guilabel:`Diagram Properties` pane displays the active diagram name and
      notes. 

   .. step:: Click the :icon-fa5:`ellipsis-h` button in the :guilabel:`Diagram Properties` pane

      .. image:: /img/diagrams/diagram-options.png
         :alt: Diagram Properties Options menu

      A menu opens. 

   .. step:: Click the Duplicate Diagram option

      A new diagram with the same entities as the duplicated diagram
      opens. The visibility of entities on your new diagram is identical
      to the duplicated diagram. 
      
      .. note :: 

         {+rel-mig+} does not clone the duplicated diagram's
         name and notes to the new diagram.
