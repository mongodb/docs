==================
configureFailPoint
==================

.. default-domain:: mongodb

Definition
----------

.. dbcommand:: configureFailPoint

   This is a command to support internal
   testing. :dbcommand:`configureFailPoint` onfigures a failure point
   that you can turn on and off while MongoDB runs. To use this
   command, :program:`mongod` must run with the
   :parameter:`enableTestCommands` parameter enabled. You can enable
   the parameter from the command line using the
   :option:`--setParameter <mongod --setParameter>` option:

   .. code-block:: javascript

      --setParameter enableTestCommands=1

   Issue the :dbcommand:`configureFailPoint` command against the
   :term:`admin database`. The command takes the following form:

   .. code-block:: javascript

      {configureFailPoint: "<failure_point>", mode: <behavior> }

   The command contains the following fields:

   .. include:: /reference/command/configureFailPoint-field.rst

Example
-------

   .. code-block:: javascript

      db.adminCommand( { configureFailPoint: "blocking_thread", mode: {times: 21} } )
