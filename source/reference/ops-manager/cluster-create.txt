.. _mcli-om-cluster-create-command:

===================================
mongocli ops-manager cluster create
===================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster create`` command creates a cluster in the specified
|mms-full| project. You can also create a cluster through 
the |mms| :opsmgr:`UI </tutorial/deploy-replica-set/>` or 
:opsmgr:`Automation Configuration API </reference/api/automation-config/>`.

.. _om-cluster-create-syntax:

Syntax 
------

.. code-block:: text 
   :copyable: false 

   mongocli ops-manager|om cluster create 
        --file|-f <om-settings-file> 
        [ --profile|-P <profile-name> ] 
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _om-cluster-create-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--file``, ``-f`` 
     - string 
     - Path to the file that contains the settings for 
       creating the cluster. To learn more about the 
       configuration properties to specify in the file, see 
       :ref:`mms-cluster-settings-file`.
     - yes

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable <mcli-env-var>`.
     - no

.. _om-cluster-create-output:

Output 
------

The command prints output similar to the following to the 
terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` 
for recommended solutions.

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://<hostname>:<port>/v2/<projectId>#deployment/topology for status

where: 

- ``<hostname>:<port>`` is the |mms| base URL
- ``projectId`` is the unique identifier of the project 

.. _om-cluster-create-examples:

Examples 
--------

The following examples show two ways to run the command.

.. _om-cluster-create-example1:

Example 1
~~~~~~~~~

The following ``mongocli ops-manager cluster create`` command 
creates a cluster named ``myCluster`` in the specified project using: 

- A file named ``om-cluster.yaml``, which contains the settings in the 
  :ref:`sample file <mms-sample-config-file>` in ``.yaml`` format
- The default profile

.. code-block:: sh 
   :copyable: false

   mongocli ops-manager cluster create --file om-cluster.yaml --projectId 5e2dec2131cd7c007880b05e

.. _om-cluster-create-example2:

Example 2
~~~~~~~~~

The following ``mongocli om cluster create`` command creates a 
cluster named ``myCluster`` in the specified project using: 

- A file named ``om-cluster.json``, which contains the settings in the 
  :ref:`sample file <mms-sample-config-file>` in ``.json`` format
- A profile named ``OMprofile``

.. code-block:: sh 
   :copyable: false

   mongocli om cluster create --file om-cluster.json --projectId 5e2dec2131cd7c007880b05e --profile OMprofile

Output
~~~~~~

The :ref:`om-cluster-create-example1` and :ref:`om-cluster-create-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://localhost:30700/v2/5e2dec2131cd7c007880b05e#deployment/topology for status
