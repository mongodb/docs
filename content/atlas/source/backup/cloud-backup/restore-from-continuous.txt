====================================
Restore from Continuous Cloud Backup
====================================

.. default-domain:: mongodb

.. meta::
   :keywords: restore, backup, continuous, continuous backup, point in time, atlas cli, atlas ui
   :description: Restore data from a Continuous Cloud Backup in Atlas by specifying a date, time, or oplog entry. Review requirements, access roles, and restore procedures.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

|service| lets you restore data from a 
:ref:`Continuous Cloud Backup <pit-restore>` by specifying one of the 
following options: 

- A specific date and time to which you want to restore
- A specific :term:`oplog` entry from which you want to restore

You can restore :ref:`selected databases or collections
<restore-from-db-coll>` from a {+pit-restore+}. To learn more, see
:ref:`Restore from Selected Databases and Collections
<restore-from-db-coll>`.
