================================
Cloud Provider Snapshot Schedule
================================

.. default-domain:: mongodb

.. include:: /includes/fact-projects-groups-synonyms.rst

.. admonition:: Cloud Provider Snapshots Only
   :class: important

   This resource only applies to clusters using
   :ref:`backup-cloud-provider`.

.. admonition:: Scheduled Snapshots Only
   :class: important

   This resource only applies to scheduled snapshots, not :doc:`on-demand snapshots
   </reference/api/cloud-provider-snapshot-take-one-ondemand>`.

The ``/backup/schedule`` resource enables you to view and modify
the snapshot scheduling and retention settings for an |service| 
cluster with :ref:`backup-cloud-provider` enabled.

Endpoints
---------

.. include:: /includes/api-base-url.rst

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description

   * - ``GET``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/schedule </reference/api/cloud-provider-snapshot-schedule-get-all>`
     - Get the current snapshot schedule and retention settings for the
       cluster with ``{CLUSTER-NAME}``.

   * - ``PATCH``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/schedule </reference/api/cloud-provider-snapshot-schedule-modify-one>`
     - Modify the snapshot schedule or retention settings for the 
       cluster with ``{CLUSTER-NAME}``.
