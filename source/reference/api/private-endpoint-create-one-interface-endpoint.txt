===========================================================
Add One Interface Endpoint to a Private Endpoint Connection
===========================================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. include:: /includes/fact-projects-groups-synonyms.rst

Add one interface endpoint to a private endpoint connection in an 
|service| project.

You must first :aws:`create the endpoint 
</AWSEC2/latest/APIReference/API_CreateVpcEndpoint.html>` in |aws| with
the following information:

- The endpoint service |service| creates with 
  :doc:`/groups/{GROUP-ID}/privateEndpoint </reference/api/private-endpoint-create-one-private-endpoint-connection>`.

  Retrieve the endpoint service name with 
  :doc:`/groups/{GROUP-ID}/privateEndpoint/{privateLinkId} </reference/api/private-endpoint-get-one-private-endpoint-connection>`.
- Unique identifier of the peer |aws| |vpc|.
- Unique identifiers of the subnets your |aws| VPC uses.

If the attempt to add an interface endpoint fails, :doc:`delete it 
</reference/api/private-endpoint-delete-one-interface-endpoint>`, then
try to add a new one.
