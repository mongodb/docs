Overview
--------

**Salted Challenge Response Authentication Mechanism (SCRAM)** is a family of
authentication mechanisms that use a username and password to authenticate
to a server.

The {+driver-short+} supports the following SCRAM-based authentication mechanisms:

- :ref:`SCRAM-SHA-256 <rust-auth-scramsha256>`: An authentication mechanism that
  uses your database username and password, encrypted with the ``SHA-256``
  algorithm. This is the default authentication mechanism.
- :ref:`SCRAM-SHA-1 <rust-auth-scramsha1>`: An authentication mechanism that
  uses your database username and password, encrypted with the ``SHA-1``
  algorithm.

You can use SCRAM to authenticate to MongoDB Atlas, MongoDB
Enterprise Advanced, and MongoDB Community Edition.

.. tip:: SCRAM Mechanisms

   To learn more about the SCRAM family of authentication mechanisms, see
   :rfc:`RFC 5802 <5802>` and
   :wikipedia:`Salted Challenge Response Authentication Mechanism <Salted_Challenge_Response_Authentication_Mechanism>`
   on Wikipedia.

   For more information about the MongoDB implementation of SCRAM, see
   :manual:`SCRAM </core/security-scram>` in the {+server+} manual.

Code Placeholders
~~~~~~~~~~~~~~~~~

The code examples on this page use the following placeholders:

- ``db_username``: Your database username
- ``db_password``: Your database password
- ``db``: The authentication database associated with the user
