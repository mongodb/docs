============================================
mongocli atlas networking peering gcp create
============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``networking peering create gcp`` command creates a peering
connection between the |service| VPC and your Google Cloud Platform VPC
for a given |service| project.

The ``networking peering create gcp`` command checks if a VPC exists for
your |service| project. If one exists, this command creates the peering
connection between that VPC and your VPC. If an |service| VPC does not
exist, this command creates one and creates a connection between it and
your VPC.

Syntax 
------

.. code-block:: text 

   mongocli atlas networking peering gcp create
        [ --atlasCidrBlock <atlas-cidr-block> ]
        --gcpProjectId <gcp-project-id>
        --network <gcp-vpc-name>
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. _atlas-networking-peering-create-gcp-options:

Options 
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--atlasCidrBlock``
     - string
     - CIDR block that |service| uses for your clusters. Required only
       if you do not already have an |service| VPC.

       .. include:: /includes/fact-gcp-cidr-block.rst
     - no

   * - ``--gcpProjectId``
     - string
     - Unique identifer of the Google Cloud Platform project that
       contains your VPC.
     - yes

   * - ``--network``
     - string
     - Name of the peer VPC.
     - yes

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project for which you want to add
       the peering connection. If omitted, uses the
       project ID in the profile or :ref:`environment variable
       <mcli-env-var>`.
     - no

.. _atlas-networking-peering-create-gcp-output:

Output
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   Network peering connection '<peering-connection-id>' created.

Example
-------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command creates a peering connection between the
      |service| VPC and your GCP VPC for a project using the
      {+default-profile+}, which contains credentials and the project
      ID. The output is returned in the default format. 

      .. code-block:: sh

         mongocli atlas networking peering create gcp --atlasCidrBlock 192.168.0.0/18 \ 
         --gcpProjectId grandiose-branch-256701 --network mongocli-test \ 
         --projectId 5f0e15e3d52a043fed8b1c92

      The previous command prints the following to the terminal.

      .. code-block:: sh
         :copyable: false

         Network peering connection '5f621dc701240c5b7c3a888e' created.

   .. tab:: JSON Output
      :tabid: json-output

      The following command creates a peering connection between the
      |service| VPC and your GCP VPC for a project using the
      {+default-profile+}, which contains credentials and the project
      ID. The output is returned in |json| format.

      .. code-block:: sh

         mongocli atlas networking peering create gcp --atlasCidrBlock 192.168.0.0/18 \ 
         --gcpProjectId grandiose-branch-256701 --network mongocli-test \ 
         --projectId 5f0e15e3d52a043fed8b1c92 --output json

      The previous command prints the following to the terminal in
      |json| format.

      .. code-block:: json
         :copyable: false

         {
           "containerId": "5f621dc701240c5b7c3a888d",
           "id": "5f621dc701240c5b7c3a888e",
           "status": "ADDING_PEER",
           "gcpProjectId": "grandiose-branch-256701",
           "networkName": "mongocli-test"
         }
