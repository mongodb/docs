Acknowledge One Alert

Resource
--------

.. code-block:: none

   PATCH /groups/{PROJECT-ID}/alerts/{ALERT-ID}

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 20 20 60
   :header-rows: 1
   :stub-columns: 1

   * - Parameter
     - Type
     - Description

   * - GROUP-ID
     - string
     - *(Required.)* Group identifier.

   * - ALERT-ID
     - string
     - *(Required.)* Alert identifier.

Request Query Parameters
~~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/no-query-single-parameters.rst

Request Body Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 10 10 80
   :header-rows: 1
   :stub-columns: 1

   * - Field
     - Type
     - Description

   * - acknowledgedUntil
     - Required.

       To acknowledge an alert "forever", set the field value to 100
       years in the future.

       To unacknowledge a previously acknowledged alert, set the field
       value to the past.
     - The page to return.

   * - acknowledgementComment
     - string
     - *(Optional.)* If you add a comment, |service| displays the comment next to the
       message that the alert has been acknowledged.

Example Request
---------------


.. only:: cloud

   .. class:: copyable-code
   .. code-block:: sh


      curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
       --header "Accept: application/json" \
       --header "Content-Type: application/json" \
       --include \
       --request PATCH "https://cloud.mongodb.com/api/public/v1.0/groups/{PROJECT-ID}/alerts/{ALERT-CONFIG-ID}"
       --data '
         {
           "acknowledgedUntil": "2016-04-15T00:00:00-0400",
           "acknowledgementComment": "This is normal. Please ignore."
         }'

.. only:: onprem

   .. class:: copyable-code
   .. code-block:: sh


      curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
       --header "Accept: application/json" \
       --header "Content-Type: application/json" \
       --include \
       --request PATCH "https://<ops-manager-host>/api/public/v1.0/groups/{PROJECT-ID}/alerts/{ALERT-CONFIG-ID}"
       --data '
         {
           "acknowledgedUntil": "2016-04-15T00:00:00-0400",
           "acknowledgementComment": "This is normal. Please ignore."
         }'
