.. _advanced-security:

Advanced Security
-----------------

|service| supports
:ref:`Encryption at Rest using Customer Key Management
<security-kms-encryption>`
and :ref:`LDAP User Authentication and Authorization <security-ldaps>`
in |service| projects and clusters.

Excluding |service-fullname| Enterprise and |service-fullname| Platinum
customers, |service| charges a 15% uplift to the cost of each cluster
with **LDAP User Authentication and Authorization** or
**Encryption at Rest using Customer Key Management** enabled.

Configuring **LDAP User Authentication and Authorization** for a
project enables the feature for all clusters in the project.
Configuring **Encryption at Rest using Customer Key Management**
for a project allows you to enable or disable the feature on a
per-cluster basis.

.. note::

   In addition to the |service| 15% uplift for clusters with LDAP User
   Authentication and Authorization or Encryption at Rest with
   customer KMS enabled, clusters using customer-managed keys incur
   costs from your cloud provider's key management service for API
   requests. Your cloud provider charges these costs based on your usage.

   |service| features such as :ref:`data validation <data-validation>`
   make additional KMS requests when operating on encrypted clusters.
   To learn more about KMS usage and costs, see
   :ref:`data-validation-kms-usage`.

.. _privatelink-billing:

Private Endpoints
-----------------

MongoDB |service| supports :ref:`private endpoints <private-endpoint-concepts>`.

You can configure private endpoints for many |service| features,
including:

- :ref:`{+dedicated-clusters+} <dedicated-clusters>`
- Online Archive
- {+adf+}
- |service| Stream Processing

{+dedicated-clusters+}, Online Archive, and {+df+} use the billing
model described in this section.

|service| Stream Processing private endpoint billing charges depend
on the private link connection created with the private endpoint. To
learn more, see :ref:`asp-private-link-billing`.

.. _billing-private-endpoints-clusters:

Private Endpoint Billing
------------------------

|service| bills private endpoints based on:

- **Per-endpoint hourly charges** in each region where you deploy an
  |service| private endpoint.
- **Usage-based charges** measured in |service|
  :ref:`Private Endpoint Capacity Units <privatelink-billing-cu>`.

These charges apply to {+dedicated-clusters+}, Online Archive, and {+df+}.

Select your cloud provider to view private endpoint billing details.

   :defaults: aws

   .. selected-content::
      :selections: aws

      MongoDB |service| supports private endpoints on {+aws+} using the
      `{+aws-pl+} <https://aws.amazon.com/privatelink/>`__ feature.

      For |service| services that use {+aws-pl+} private endpoints
      (for example, :ref:`{+dedicated-clusters+} <dedicated-clusters>`,
      Online Archive, and {+df+}), |service| charges you based on
      your usage across the following dimensions:

      - |service| :ref:`Private Endpoint <privatelink-billing-endpoint>`:
        The sum of the hourly costs incurred to host private endpoints
        in all {+aws+} regions.
      - |service|
        :ref:`Private Endpoint Capacity Units <privatelink-billing-cu>`:
        The sum of the hourly costs incurred to process traffic to
        private endpoints.

      In addition to the costs |service| charges you, {+aws+} charges you
      for each |vpc| interface endpoint you create. To ensure
      {+cluster+} availability in the event of a widespread network
      outage, create |vpc| interface endpoints in at least two
      availability zones per region. {+aws+} also charges you for the
      data volume each |vpc| interface endpoint processes.

      .. seealso::

         `AWS PrivateLink Pricing
         <https://aws.amazon.com/privatelink/pricing/>`__.

      .. _privatelink-billing-endpoint:

      |service| Private Endpoint
      ~~~~~~~~~~~~~~~~~~~~~~~~~~

      |service| charges hourly for each region to which you deploy a
      private endpoint. The :ref:`hourly cost <pl-region-rates>` varies
      by region.

      .. _privatelink-billing-cu:

      |service| Private Endpoint Capacity Units
      ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

      Additionally, |service| charges you per hour based on the traffic
      that your private endpoint processes. The :ref:`hourly cost
      <pl-region-rates>` varies based on the amount of traffic and the
      region to which your private endpoint is deployed.

      |service| private endpoint capacity units measure the dimensions
      on which the |service| private endpoint processes your traffic:

      - **New connections or flows**: Number of newly established
        connections or flows per second.
      - **Active connections or flows**: Peak concurrent connections or
        flows, sampled per minute.
      - **Processed bytes**: The number of bytes processed by the
        private endpoint in Gigabytes (GB).

      For TCP traffic, an |service| private endpoint capacity unit
      contains:

      - **New connections or flows**: 800 new TCP connections per
        second.
      - **Active connections or flows**: 100,000 active TCP connections
        (sampled per minute).
      - **Processed bytes**: 1 GB per hour.

      For UDP traffic, an |service| private endpoint capacity unit
      contains:

      - **New connections or flows**: 400 new UDP flows per second.
      - **Active connections or flows**: 50,000 active UDP flows
        (sampled per minute).
      - **Processed bytes**: 1 GB per hour.

      For TLS traffic, an |service| private endpoint capacity unit
      contains:

      - **New connections or flows**: 50 new TLS connections or flows
        per second.
      - **Active connections or flows**: 3,000 active TLS connections
        or flows (sampled per minute).
      - **Processed bytes**: 1 GB per hour.

      |service| charges you only for the dimension that has the highest
      usage during each hour across all traffic types.

      The following example shows a breakdown of costs for one hour of
      |tls| network traffic to a private endpoint in the ``us-east-1``
      region:

      .. list-table::
         :header-rows: 1
         :widths: 50 25 25

         * - Dimension
           - Amount
           - |service| Private Endpoint Capacity Units Consumed
         * - New connections or flows
           - 25
           - .50
         * - Active connections or flows
           - 1500
           - .50
         * - Processed bytes
           - 2.5 GB
           - 2.5

      The **Processed bytes** dimension has the largest capacity unit
      consumption rate. For this hour, |service| charges you the
      following for using |service| private endpoint capacity units
      in the ``us-east-1`` region:

      .. list-table::
         :header-rows: 1

         * - Capacity Units
           - Region Rate per Capacity Unit
           - Fixed Cost
           - Total
         * - 2.5
           - $0.0120
           - $0.045
           - $0.075

             ``0.045 + (2.5 capacity units * 0.0120)``

      Remote Region
      ~~~~~~~~~~~~~

      |service| charges an additional hourly cost for each active
      remote region with cross-region private endpoint connectivity.

      - **Hourly Cost per Active Remote Region:** $0.05 per hour for
        each active remote region.

      A remote region is considered active when at least one interface
      endpoint from that region is connected to the private endpoint
      service. You are charged for each hour (or partial hour) that a
      remote region is active.

      .. note::

         This charge applies in addition to the standard private
         endpoint and capacity unit charges. If you configure a remote
         region but do not connect any endpoints from that region, you
         are not charged for that region.

.. _database-auditing:

Database Auditing
-----------------

|service| supports database auditing. To learn more, see
:ref:`Set up Database Auditing <set-up-database-auditing>`.

Excluding |service-fullname| Enterprise and |service-fullname|
Platinum customers, |service| charges a 10% uplift to the hourly
cost of all dedicated clusters. The 10% uplift applies to projects
using Database Auditing.

.. _charts-for-atlas:

Charts for Atlas
----------------

Charts on |service| is billed based on the volume of data transferred
from the Charts web server to web browser clients. Each Charts instance
includes ``1 GB`` of data transfers per month at no charge. After you
use the included ``1 GB``, each additional GB of data transfers is
billed at a fixed rate.

.. seealso::

   :charts:`Charts on Atlas Pricing </pricing>`.

.. _oa-for-atlas:

Atlas Online Archive
--------------------

You may incur data transfer and compute costs for each Online Archive
on your |service| cluster. If you have Online Archive configured on
your cluster, your |service| invoice includes these costs as a line
item.

.. seealso::

   |service-pricing|

Support Plan
------------

If you have an upgraded :ref:`support plan <atlas-support>`, your invoice
lists the total monthly support cost in the :guilabel:`Summary`
section. Support cost line items appear in the :guilabel:`Details`
section.
