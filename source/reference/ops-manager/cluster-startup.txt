====================================
mongocli ops-manager cluster startup
====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster startup`` command starts the replica set
:manual:`mongod </reference/program/mongod/#bin.mongod>` 
processes for the specified cluster in the specified 
|mms| project. The command prompts you to confirm the 
operation when you run the command without the 
``--force`` option. You can also restart the :manual:`mongod 
</reference/program/mongod/#bin.mongod>` process from the 
|mms| :opsmgr:`UI </tutorial/restart-deployment/>`.

.. _om-cluster-startup-syntax:

Syntax 
------

.. code-block:: text 

   mongocli ops-manager|om cluster startup <cluster-name> 
        [ --force ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ] 
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _om-cluster-startup-options:

Options 
------- 

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``<cluster-name>``
     - string
     - Name of the cluster.
     - yes

   * - ``--force``
     - 
     - Flag that indicates that the replica set :manual:`mongod 
       </reference/program/mongod/#bin.mongod>` processes can be 
       started without prompting for confirmation.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are set. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to startup. If omitted, uses the project ID in 
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no

.. _om-cluster-startup-output:

Output 
------

The command prints output similar to the following to the terminal if 
the command succeeds. If the command prints errors, see 
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: text 
   :copyable: false 

   Changes are being applied, please check http://<hostname>:<port>/v2/<projectID>#deployment/topology for status

where: 
   
- ``<hostname>:<port>`` is the |mms| base URL.
- ``<projectID>`` is the unique identifier of the project that contains 
  the cluster to start.

.. _om-cluster-startup-command-examples: 

Examples 
--------

The following examples show the different ways to run the command.

.. _om-cluster-startup-command-example1: 

Example 1
~~~~~~~~~

The following ``mongocli om cluster startup`` command 
prompts for confirmation before starting the the replica set 
``mongod`` processes for the cluster named ``myCluster`` in the specified 
project. It uses the {+default-profile+} to access the 
specified project.

.. code-block:: text 
   :copyable: false
   
   mongocli om cluster startup myCluster --projectId 5e2dec2131cd7c007880b05e
   Are you sure you want to startup: myCluster Yes

.. _om-cluster-startup-command-example2: 

Example 2
~~~~~~~~~

The following ``mongocli ops-manager cluster startup`` command 
*does not* prompt for confirmation before starting the replica set 
``mongod`` processes for the cluster named ``myCluster`` in the specified 
project. It uses the {+default-profile+} for accessing the specified project.

.. code-block:: text 
   :copyable: false
   
   mongocli ops-manager cluster startup myCluster --projectId 5e2dec2131cd7c007880b05e --force 

Output
~~~~~~

The :ref:`om-cluster-startup-command-example1` and :ref:`om-cluster-startup-command-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false 

   Changes are being applied, please check http://localhost:30700/v2/5e2dec2131cd7c007880b05e#deployment/topology for status
