.. _mcli-cm-cluster-shutdown-command:

=======================================
mongocli cloud-manager cluster shutdown
=======================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster shutdown`` command shuts down the replica set 
:manual:`mongod </reference/program/mongod/#bin.mongod>` processes 
for the specified cluster in the specified |cloud-short| project. 
The command prompts you to confirm the operation when you run the 
command without the ``--force`` option. You can also shut down the 
:manual:`mongod </reference/program/mongod/#bin.mongod>` processes 
from the |cloud| :cloudmgr:`UI </tutorial/shut-down-deployment/>`.

.. note:: 

   When you shutdown a process, replica set, or cluster, |cloud-short| 
   continues to manage it, even though it is not running. Your data 
   is not removed and you can :ref:`restart your process, cluster, or 
   replica set <mcli-cm-cluster-startup-command>` and access your data.

.. _cm-cluster-shutdown-syntax:

Syntax 
------

.. code-block:: text 

   mongocli cloud-manager|cm cluster shutdown <cluster-name>
        [ --force ]
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _cm-cluster-shutdown-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``<cluster-name>``
     - string
     - Name of the cluster that contains the replica 
       set to shut down.
     - yes

   * - ``--force``
     - 
     - Flag that indicates that the replica set :manual:`mongod 
       </reference/program/mongod/#bin.mongod>` processes can be 
       shut down without prompting for confirmation.
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are set. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to shutdown. If omitted, uses the project ID in 
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no


.. _cm-cluster-shutdown-output: 

Output 
------

The command output to the terminal looks similar to the following if the 
command succeeds. If the command prints errors, see :ref:`Troubleshooting 
<troubleshooting>` for recommended solutions.

.. code-block:: text
   :copyable: false

   Changes are being applied, please check https://cloud.mongodb.com/v2/<projectID>#deployment/topology for status

where ``projectID`` is the unique identifier of the project that contains 
the cluster being shut down.

.. _cm-cluster-shutdown-examples: 

Examples 
--------

The following examples show the different ways to run the command.

.. _cm-cluster-shutdown-example1: 

Example 1
~~~~~~~~~

The following ``mongocli cloud-manager cluster shutdown`` command *does not* prompt for confirmation before shutting down the replica set :manual:`mongod </reference/program/mongod/#bin.mongod>` processes for the cluster named ``Cluster1`` in the specified project. It uses the ``default`` profile for accessing the specified project.

.. code-block:: text
   :copyable: false 

   mongocli cm cluster shutdown Cluster1 --projectId 5e2f06127a3e5a5700de8d8d --force

.. _cm-cluster-shutdown-example2: 

Example 2
~~~~~~~~~

The following ``mongocli cm cluster shutdown`` command prompts for confirmation before shutting down the the replica set :manual:`mongod </reference/program/mongod/#bin.mongod>` processes for the cluster named ``Cluster2`` in the specified project. It uses the profile named ``egCMprofile`` for accessing the specified project.

.. code-block:: text 
   :copyable: false 

   mongocli cm cluster shutdown Cluster2 --projectId 5e2f06127a3e5a5700de8d8d --profile egCMprofile 
   Are you sure you want to shutdown: Cluster2 Yes

Output
~~~~~~

The :ref:`cm-cluster-shutdown-example1` and :ref:`cm-cluster-shutdown-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false 

   Changes are being applied, please check https://cloud.mongodb.com/v2/5e2f06127a3e5a5700de8d8d#deployment/topology for status
