==========================
Download |mms| Server Logs
==========================

.. meta::
   :description: Download the .tar.gz archive for a completed Ops Manager server log collection job. The archive holds every collected log file by hostname and log type.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

Use this endpoint to download a ``.tar.gz`` archive that contains
every log file that |mms| collected for the specified job. You
cannot download from a job whose status is ``IN_PROGRESS`` or
``EXPIRED``. If a job partially failed, the archive includes only
the log files that |mms| collected successfully.

.. include:: /includes/api/base-api-uri.rst

Resource
--------

.. code-block:: http

   GET /admin/omLogCollectionJobs/{JOB-ID}:download

Response
--------

This endpoint returns a binary stream with
``Content-Type: application/gzip`` and the file name format
``om-logs_{timestamp}.tar.gz``. The archive uses the following
layout:

.. code-block:: text

   om-server-1.example.com/
      application/
         mms0.log
         mms0.20260412.log
      http_access/
         mms0-access.log
      migration/
         mms-migration.log
   om-server-2.example.com/
      application/
         mms0.log
      http_access/
         mms0-access.log
         mms0-access.20260412.log

.. important::

   When you call this endpoint with ``curl``, you must pass
   ``--output`` and a file name. Otherwise, ``curl`` writes the
   binary archive to your terminal.

Example Request
---------------

.. code-block:: sh

   curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
        --header "Accept: application/gzip" \
        --output om-logs.tar.gz \
        --request GET \
        "https://{+opsmgr-url+}/api/public/v1.0/admin/omLogCollectionJobs/{JOB-ID}:download"
