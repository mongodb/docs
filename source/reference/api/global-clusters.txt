===============
Global Clusters
===============

.. default-domain:: mongodb

.. include:: /includes/fact-projects-groups-synonyms.rst

The |global-write-clusters| endpoints provide the ability to add, 
remove, and retrieve managed namespaces and custom zone mappings 
associated with :doc:`Global Clusters </global-clusters>`. Each 
collection in a |global-write-cluster| is associated with a managed 
namespace. For example, the namespace for the ``publishers`` collection 
in the ``partners`` database is ``partners.publishers``. 

When you create a managed namespace for a |global-write-cluster|, 
|service| creates an empty collection for that namespace. In addition, 
|service| recreates the collection associated with a managed namespace 
if you delete the collection using the ``mongo`` shell. However, the 
recreated collection is empty; |service| does not repopulate it with 
data.

.. note::

   Creating a managed namespace does not populate a collection with data.
   Similarly, deleting a managed namespace does not delete the associated
   collection.

|service| shards the empty collection using the required ``location`` 
field and a custom shard key. For example, if the your custom shard
key is ``city``, the compound shard key is ``location, city``.

Each |global-write-cluster| is also associated with one or more 
:ref:`Global Writes Zones <global-writes-zones>`. When a user creates
a |global-write-cluster|, |service| automatically maps each location 
code to the closest geographical zone. Custom zone mappings allow
administrators to override these automatic mappings. For example, a use
case may require mapping a location code to a geographically distant
zone. Administrators can manage custom zone mappings with the APIs
below and the :guilabel:`Global Cluster Configuration` pane
when you create or modify your |global-write-cluster|.

Endpoints
---------

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description

   * - ``GET``
     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/globalWrites </reference/api/global-clusters-retrieve-namespaces>`
     - Retrieve all managed namespaces and custom zone mappings associated
       with the specified |global-write-cluster|.

   * - ``POST``
     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/globalWrites/managedNamespaces </reference/api/global-clusters-add-namespace>`
     - Adds a managed namespace to the specified |global-write-cluster|.

   * - ``DELETE``
     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/globalWrites/managedNamespaces </reference/api/global-clusters-delete-namespace>`
     - Deletes the specified managed namespace from the specified
       |global-write-cluster|

   * - ``POST``
     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/globalWrites/customZoneMapping </reference/api/global-clusters-add-customzonemapping>`
     - Add an entry to the list of custom zone mappings for the specified |global-write-cluster|.

   * - ``DELETE``
     - :doc:`/api/atlas/v1.0/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/globalWrites/customZoneMapping </reference/api/global-clusters-delete-customzonemappings>`
     - Remove all custom zone mappings for the specified |global-write-cluster|.
