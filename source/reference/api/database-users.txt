==============
Database Users
==============

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: twocols

.. include:: /includes/fact-projects-groups-synonyms.rst

.. include:: /includes/api-base-url.rst

The ``databaseUsers`` resource lets you retrieve, create and modify the
database users in your cluster.

Each database user requires the following parameters to authenticate
with a MongoDB database:

- username
- password
- :ref:`authentication database <authentication-database>`

|service| sets ``admin`` as the authentication database for all users.
The authentication database doesn't set the user's privileges on the
project's databases.

Each database user has a list of roles that authorize certain
privileges on a project's databases.

By default, a database user's roles apply to all the clusters in the
project:

.. example::

   Two clusters have a ``products`` database. One user, Pat, has a role
   granting ``read`` access on the ``products`` database. Pat can
   access ``products`` on both clusters.

.. note::

   If a database user is assigned a
   :ref:`custom role <atlas-user-privileges>`,
   they cannot be assigned any other roles.

You can limit a database user's access to one or more specific clusters
or data lakes.

The ``databaseUsers`` resource supports creating temporary database
users that automatically expire within a user-configurable 7-day
period.

The ``databaseUsers`` resource requires your :ref:`group-id`.

.. list-table::
   :header-rows: 1
   :widths: 10 70 15

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/databaseUsers </reference/api/database-users-get-all-users>`

     - Get all users in the project.

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/databaseUsers/admin/{USERNAME} </reference/api/database-users-get-single-user>`

     - Get a single user in the project.

   * - ``POST``

     - :doc:`/groups/{GROUP-ID}/databaseUsers </reference/api/database-users-create-a-user>`

     - Create a user for the project.

   * - ``PATCH``

     - :doc:`/groups/{GROUP-ID}/databaseUsers/admin/{USERNAME} </reference/api/database-users-update-a-user>`

     - Update a user for the project.

   * - ``DELETE``

     - :doc:`/groups/{GROUP-ID}/databaseUsers/admin/{USERNAME} </reference/api/database-users-delete-a-user>`

     - Delete a user for the project.
