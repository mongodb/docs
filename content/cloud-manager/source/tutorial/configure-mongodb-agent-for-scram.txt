==========================================
Configure MongoDB Agent for Authentication
==========================================

.. meta::
   :description: Configure the MongoDB Agent for authentication using SCRAM mechanisms, and manage credentials for deployments with or without Automation in Cloud Manager.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. start-include-content

MongoDB supports the following authentication mechanisms depending on your MongoDB version:

.. list-table::
   :header-rows: 1
   :widths: 30 70

   * - MongoDB Version
     - Default Authentication Mechanism

   * - 4.0 or later
     - :manual:`SCRAM </core/security-scram/>` authentication
       mechanisms with the ``SHA-256`` and ``SHA-1`` hash functions.
       ``SCRAM-SHA-1`` (:rfc:`RFC 5802 <5802>`) and ``SCRAM-SHA-256``
       (:rfc:`RFC 7677 <7677>`) are |ietf| standards that define best
       practice methods for implementation of challenge-response
       mechanisms for authenticating users with passwords.

   * - 3.0 to 3.6
     - :manual:`SCRAM </core/security-scram/>` authentication mechanism
       with``SHA-1`` hash function.

   * - 2.6 or earlier
     - MongoDB Challenge and Response (``MONGODB-CR``). ``MONGODB-CR``
       is a challenge-response mechanism that authenticates users
       through passwords.

The {+mdbagent+} can use ``SCRAM-SHA-1`` or ``SCRAM-SHA-256``
to authenticate to hosts that enforce access control.

      Configure Deployments Managed by {+aagent+}
      ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

      When you install the {+mdbagent+} with {+aagent+}, |mms| creates a
      user to authenticate on the MongoDB database. |mms| create this
      MongoDB user (``mms-automation``) in the ``admin`` database with
      the correct privileges for each {+mdbagent+} function.

      Configure these credentials in |mms|.

      1. In |cloud|, go to the :guilabel:`Security` page for your
         project.

         a. If it's not already displayed, select the organization
            that contains your desired project from the
            |ui-org-menu| in the navigation bar.

         #. If it's not already displayed, select your desired
            project from the :guilabel:`Projects` menu in the
            navigation bar.

         #. In the sidebar, click :guilabel:`Security` under the
            :guilabel:`Database` heading.

         The `Security <https://cloud.mongodb.com/go?l=https%3A%2F%2Fcloud.mongodb.com%2Fv2%2F%3Cproject%3E%23%2Fdeployment%2Fsecurity%2Fusers>`__
         page displays.

      2. Click :guilabel:`Settings`.

      3. Continue through the modal until you see the
         :guilabel:`Configure Cloud Manager Agents` page.

      4. Add the appropriate credentials:

         .. list-table::
            :header-rows: 1
            :widths: 30 70

            * - Setting
              - Value

            * - :guilabel:`{+mdbagent+} Username`
              - Enter the {+mdbagent+} username.

            * - :guilabel:`{+mdbagent+} Password`
              - Enter the password for the {+mdbagent+} username.

      Configure Deployments Not Managed by {+aagent+}
      ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

      Each {+mdbagent+} function uses a different set of
      MongoDB shell ({+mongosh+}) commands to configure a user with the
      appropriate roles and privileges.

      User creation commands vary depending on the version of
      MongoDB that you use:

      .. collapsible::
         :heading: FCV 4.2 or later
         :sub_heading: Create a user to back up MongoDB instances running FCV 4.2 and later.
         :expanded: false

         To back up MongoDB instances running |fcv-link| 4.2 and
         later, create a user in the ``admin`` database. Assign this
         user the roles provided in the following example.

         Use the following {+mongosh+} command to create the user:

         .. code-block:: javascript

            db.getSiblingDB("admin").createUser(
               {
                  user: "<username>",
                  pwd: "<password>",
                  roles: [ {
                     role: "clusterAdmin", db: "admin"
                  } ]
               }
            )

         To learn more about the required access, see :ref:`Required
         Access for MongoDB Agent Backup
         <required-permissions-mms-backup>`.

      .. collapsible::
         :heading: FCV 4.0 or earlier
         :sub_heading: Create a user to back up MongoDB instances running FCV 4.0 or earlier.
         :expanded: false

         To back up MongoDB instances running |fcv-link| 4.0 or
         earlier, create a user in the ``admin`` database. Assign
         this user the roles provided in the following example.

         Use the following {+mongosh+} command to create the user:

         .. code-block:: javascript

            db.getSiblingDB("admin").createUser(
               {
                  user: "<username>",
                  pwd: "<password>",
                  roles: [ {
                     role: "backup", db: "admin"
                  } ]
               }
            )

         To learn more about the required access, see :ref:`Required
         Access for MongoDB Agent Backup
         <required-permissions-mms-backup>`.

      Configure Deployments Not Managed by {+aagent+}
      ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

      Each {+mdbagent+} function uses a different set of
      MongoDB shell ({+mongosh+}) commands to configure a user with the
      appropriate roles and privileges.

      To monitor MongoDB 4.0 or later instances that use SCRAM
      authentication, add a user to the ``admin`` database in
      MongoDB. Assign this user the roles provided in the
      following example.

      .. code-block:: javascript

         db.getSiblingDB("admin").createUser(
           {
             user: "<username>",
             pwd: "<password>",
             roles: [ {
               role: "clusterMonitor", db: "admin"
             } ]
           }
         )

      .. note::

         For |mms| projects that contain a sharded cluster with MongoDB 8.0 or greater feature compatibility
         version, the monitoring agent requires the :authrole:`directShardOperations` role if the cluster has:

         - Two or more shards
         - At least one non-system database

         This role allows the monitoring agent to perform necessary maintenance and diagnostic
         operations directly on the shards.

         This requirement is specific to |mms|. If you are using {+service+},
         {+service+} manages these permissions automatically.

         .. code-block:: javascript

            db.getSiblingDB("admin").createUser(
              {
                user: "<username>",
                pwd: "<password>",
                roles: [
                  { role: "clusterMonitor", db: "admin" },
                  { role: "directShardOperations", db: "admin" }
                ]
              }
            )

      To learn what roles this function requires, see
      :ref:`Monitoring settings <mongodb-agent-monitoring-settings>`.

      After you create the user for the {+magent+} function, add
      the credentials to the Monitoring Settings.

      1. In |cloud|, go to the :guilabel:`Processes` page for your
         project.

         a. If it's not already displayed, select the organization
            that contains your desired project from the
            |ui-org-menu| in the navigation bar.

         #. If it's not already displayed, select your desired
            project from the :guilabel:`Projects` menu in the
            navigation bar.

         #. In the sidebar, click :guilabel:`Processes` under the
            :guilabel:`Database` heading.

         The `Processes <https://cloud.mongodb.com/go?l=https%3A%2F%2Fcloud.mongodb.com%2Fv2%2F%3Cproject%3E%23%2Fdeployment%2Flist>`__
         page displays.

      2. Click the :icon-mms:`ellipsis`, and then
         :guilabel:`Monitoring Settings`.

      3. Click :guilabel:`Credentials`.

      4. Add the appropriate credentials:

         .. list-table::
            :header-rows: 1
            :widths: 30 70

            * - Setting
              - Value

            * - :guilabel:`{+magent+} Username`
              - Enter the {+magent+} username.

            * - :guilabel:`{+magent+} Password`
              - Enter the password for the {+magent+} username.
