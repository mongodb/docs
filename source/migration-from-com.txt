.. _migrate-from-com:

====================================================
Live Migrate: Push Your Data from |com| to |service|
====================================================

.. default-domain:: mongodb

.. meta::
   :keywords: live migration (push) to MongoDB Atlas

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can live migrate data from existing MongoDB deployments monitored by
|com| into |service| using the live migration service through the
|service| UI.

To locate live migration (push) in the {+atlas-ui+}, in the left-side
panel of your organization's page, go to :guilabel:`Live Migration to Atlas`
and select :guilabel:`Migrate from Ops Manager or Cloud Manager`.


.. list-table::
   :header-rows: 1
   :widths: 35 65
   
   * - Method
     - Description
   
   * - :doc:`Live migrate (push) a replica set monitored in Ops Manager or
       Cloud Manager to Atlas </import/migrate-from-com-rs>`
   
     - |com| **pushes** data from the source replica set to a destination
       replica set in |service|.
   
   * - :doc:`Live migrate (push) a sharded cluster monitored in Ops Manager or Cloud Manager to Atlas
       </import/migrate-from-com-sharded>`
   
     - |com| **pushes** data from the source sharded {+cluster+} to a
       destination sharded {+cluster+} in |service|.
   
   * - Live migrate a standalone MongoDB node
   
     - :opsmgr:`Convert </tutorial/convert-standalone-to-replica-set/>`
       the standalone MongoDB node in Ops Manager, or
       :cloudmgr:`convert </tutorial/convert-standalone-to-replica-set/>`
       the standalone MongoDB node in |mms| to a single-node
       replica set and then :doc:`live migrate (push) <import/migrate-from-com-rs>`
       it to |service|.
