.. _mcli-om-agents-list-command:

================================
mongocli ops-manager agents list
================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``agents list`` command retrieves all agents of a provided type for
the specified |onprem| project.

You must have at least ``GROUP_READ_ONLY`` privileges for a project to
retrieve the agents for that project.

.. _om-agents-list-syntax:

Syntax 
------

.. code-block:: text 

   mongocli ops-manager|om agents list|ls <type>
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --projectId <id-of-project> ]

.. _mcli-om-agents-list-arguments:

Arguments  
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - ``<type>``
     - string
     - Agent type to retrieve. Accepted values are:

       - ``AUTOMATION``
       - ``MONITORING``
       - ``BACKUP``

     - yes

.. _om-agents-list-options:

Options 
-------

.. list-table::
   :widths: 20 10 60 10

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       agents you want to retrieve. If omitted, uses the project ID in 
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no

.. _om-agents-list-output:

Output
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   HOSTNAME             TYPE     STATE
   <agent-1-hostname>   <type>   <agent-1-state>
   <agent-2-hostname>   <type>   <agent-2-state>
   <agent-3-hostname>   <type>   <agent-3-state>

.. include:: /includes/fact-default-output.rst

- :opsmgr:`Ops Manager API </reference/api/agents-get-by-type/#response>`

Example
-------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command retrieves Automation Agents for a project
      using the {+default-profile+}, which contains credentials and the
      project ID. The output is returned in the default format.

      .. code-block:: sh

         mongocli om agents list AUTOMATION

      The previous command prints the following to the terminal.

      .. code-block:: sh
         :copyable: false

         HOSTNAME          TYPE         STATE
         db1.example.com   AUTOMATION   ACTIVE
         db2.example.com   AUTOMATION   NO_PROCESSES
         db3.example.com   AUTOMATION   NO_PROCESSES
