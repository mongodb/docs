Organization API Key Endpoints
------------------------------

Use the following resources to view, create, or delete
:ref:`programmatic API Keys <atlas-prog-api-key>` within the specified
|service| :ref:`organization <organizations>`.

Users with the :authrole:`Organization Owner` role in the
:ref:`organization <organizations>` associated with the
:ref:`API Key <atlas-prog-api-key>` can use the Organization |api| Key
to access these endpoints.

.. list-table::
   :header-rows: 1
   :widths: 10 40 50

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/orgs/{ORG-ID}/apiKeys </reference/api/apiKeys-orgs-get-all>`

     - Get all Organization |api| Keys associated with ``{ORG-ID}``.

   * - ``GET``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID} </reference/api/apiKeys-orgs-get-one>`

     - Get one Organization |api| Key.

   * - ``PATCH``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID} </reference/api/apiKeys-orgs-update-one>`

     - Update an Organization |api| Key.

   * - ``POST``

     - :doc:`/orgs/{ORG-ID}/apiKeys  </reference/api/apiKeys-orgs-create-one>`

     - Create an Organization |api| Key for the specified organization.

   * - ``DELETE``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID} </reference/api/apiKey-delete-one-apiKey>`

     - Delete the Organization |api| Key with ID ``{API-KEY-ID}``.

Organization API Key Access List Endpoints
------------------------------------------

Use the following resources to view, create, and update access lists for
Organization :ref:`programmatic API Keys <atlas-prog-api-key>`.

Users with the :authrole:`Organization Owner` role in the
:ref:`organization <organizations>` associated with the
:ref:`API Key <atlas-prog-api-key>` can use the Organization |api| Key
to access these endpoints.

.. list-table::
   :header-rows: 1
   :widths: 10 40 50

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID}/accesslist </reference/api/api-access-list/get-all-api-access-entries>`

     - Get all API Key access list entries that belong to
       ``{API-KEY-ID}``.

   * - ``GET``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID}/accesslist/{IP-ADDRESS} </reference/api/api-access-list/get-one-api-access-entry>`

     - Get the API Key access list entry for ``{API-KEY-ID}`` specified
       by ``{IP-ADDRESS}``.

   * - ``POST``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID}/accesslist </reference/api/api-access-list/create-api-access-entries>`

     - Create one or more API Key access list entries for
       ``{API-KEY-ID}``.

   * - ``DELETE``

     - :doc:`/orgs/{ORG-ID}/apiKeys/{API-KEY-ID}/accesslist/{IP-ADDRESS} </reference/api/api-access-list/delete-one-api-access-entry>`

     - Delete the API Key access list entry for ``{API-KEY-ID}``
       specified by ``{IP-ADDRESS}``.

Organization API Keys on Projects Endpoints
-------------------------------------------

Use the following resources to view, create and assign, or unassign
Organization :ref:`programmatic API Keys <atlas-prog-api-key>` within
the specified |service| :ref:`project <projects>`.

Users with the :authrole:`Project Owner` role in the project associated
with the :ref:`API Key <atlas-prog-api-key>` can use the Organization
|api| Key to access these endpoints.

.. list-table::
   :header-rows: 1
   :widths: 10 40 50

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/apiKeys </reference/api/projectApiKeys/get-all-apiKeys-in-one-project>`

     - Get All Organization |api| Keys Assigned to One Project with ID
       ``{GROUP_ID}``.

   * - ``PATCH``

     - :doc:`/groups/{GROUP-ID}/apiKeys/{API-KEY-ID} </reference/api/projectApiKeys/update-one-apiKey-in-one-project>`

     - Modify Roles of One Organization |api| Key to One Project.

   * - ``POST``

     - :doc:`/groups/{GROUP-ID}/apiKeys </reference/api/projectApiKeys/create-one-apiKey-in-one-project>`

     - Create and Assign One Organization |api| Key to One Project.

   * - ``PATCH``

     - :doc:`/groups/{GROUP-ID}/apiKeys/{API-KEY-ID} </reference/api/projectApiKeys/assign-one-org-apiKey-to-one-project>`

     - Assign One Organization |api| Key to One Project.

   * - ``DELETE``

     - :doc:`/groups/{GROUP-ID}/apiKeys/{API-KEY-ID} </reference/api/projectApiKeys/delete-one-apiKey-in-one-project>`

     - Unassign One Organization API Key (``{API-KEY-ID}``) from One
       Project.
