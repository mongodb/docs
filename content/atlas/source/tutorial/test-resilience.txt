===============
Test Resilience
===============

.. meta::
   :description: Test Atlas cluster resilience by simulating a primary failover or a regional outage using the Atlas UI or API. Learn how each test works and how your application should respond.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can simulate a failover for clusters with :guilabel:`Test Failover`
and a regional outage for a multi-region cluster with :guilabel:`Test
Outage`.

- You can simulate a failover for clusters with :guilabel:`Test
  Failover`  in the {+atlas-ui+} or |api|. On average, an election
  takes five seconds. To learn more, see :ref:`test-failover`.

- You can simulate a regional outage for a multi-region
  cluster with :guilabel:`Simulate Regional Outage`  in the |service| UI
  or |api|. The simulation runs for a configurable duration of 1, 3, or
  7 days. To learn more, see :ref:`test-outage`.

Required Access
---------------

You must have :authrole:`Project Cluster Resilience Tester`,`
:authrole:`Project Cluster Manager`, or higher role to perform a
:ref:`Test Failover <test-failover>` or :ref:`Simulate Regional Outage
<test-outage>`.
