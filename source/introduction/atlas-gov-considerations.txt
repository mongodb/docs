==========================================
Considerations for |service| in |govcloud|
==========================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

|govcloud| is a data platform for the U.S. government 
(federal agencies and state, local, and education (SLED) entities). 
When you are logged into the |govcloud| console, an indicator at the 
top of the page displays:

.. figure:: /images/cloudgov-indicator.png
   :figwidth: 50%
   :align: center
   :alt: Image of a blue indicator reading 
         "MongoDB Cloud for Government"

Much of the information and many of the procedures in the 
:atlas:`Atlas documentation </>` hold true for |govcloud-short|, but 
there are key differences. Review the following sections to understand 
the restrictions and limitations of the |govcloud| environment.

Sign Up and Login
-----------------

To create a |govcloud| account, you must be invited by either 
MongoDB Sales or an active user. 

To learn more about signing up for |govcloud-short|, see 
:ref:`Create an Account <create-account>`.

Available Regions
-----------------

|govcloud-short| allows you to deploy clusters in government regions 
that meet more stringent security standards.

.. include:: /includes/considerations/gov-commercial-regions.rst

To learn more about the regions available for |govcloud-short| 
deployments, see :ref:`<supported-regions>`.

To learn more about AWS ``gov`` regions, see the 
:aws:`AWS documentation </govcloud-us/>`.

|service| Clusters
------------------

|govcloud-short| clusters must be tier ``M20`` or higher.

To learn how to deploy a |govcloud-short| cluster, see 
:ref:`Create a Cluster <create-cluster>`.

Database Users
--------------

|govcloud-short| database users who authenticate with 
:manual:`SCRAM </core/security-scram/>` must use ``SCRAM-SHA-256``.

To learn more about configuring |govcloud-short| database users,
see :ref:`Configure Database Users <configure-dbusers>`.
