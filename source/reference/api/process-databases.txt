=============================================
Get Available Databases for a MongoDB Process
=============================================

.. default-domain:: mongodb

.. meta::
   :canonical: https://www.mongodb.com/docs/atlas/reference/api-resources-spec/#tag/Monitoring-and-Logs

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. important:: Operational Limitations of |service| ``M0`` {+free-clusters+} and ``M2/M5`` {+shared-clusters+}

   ``M0`` {+free-clusters+}, ``M2``, and
   ``M5`` {+shared-clusters+} have some operational limits. 
   To learn more, see :ref:`shared-limits-ops`.

.. include:: /includes/fact-projects-groups-synonyms.rst

Get the available databases for an |service| MongoDB Process. The |service|
MongoDB process must be a :binary:`mongod <bin.mongod>`.

.. include:: /includes/api-requirements.rst

.. include:: /includes/api-base-url.rst

Resource
--------

.. code-block:: none

   GET api/atlas/v1.0/groups/{GROUP-ID}/processes/{HOST}:{PORT}/databases

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/list-tables/requests/measurements-path-parameters.rst

.. include:: /includes/api/no-query-list-parameters.rst

Response Elements
-----------------

.. include:: /includes/api/get-all-note.rst

.. list-table::
   :widths: 20 14 66
   :header-rows: 1

   * - Name
     - Type
     - Description

   * - databaseName
     - string
     - Name of the database. You can use the value to
       :doc:`retrieve measurements for the
       database </reference/api/process-databases-measurements>`.

   * - links
     - array
     - .. include:: /includes/api/links-explanation.rst

Example Request
---------------

Replace the information in brackets ``{}`` with your own |service|
information to execute this example request:


.. code-block:: shell

   curl -u "<ATLAS-USERNAME>:<API-KEY>" --digest -i "https://cloud.mongodb.com/api/atlas/v1.0/groups/{GROUP-ID}/processes/{HOST}:{PORT}/databases?pretty=true"
