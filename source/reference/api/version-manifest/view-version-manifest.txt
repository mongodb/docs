=============================
Retrieve the Version Manifest
=============================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. include:: /includes/styles/corrections.rst

Use this resource to retrieve the current version manifest from
MongoDB, Inc. via the Internet. Transfer this |json| file to a host
that can access your |onprem| instance. To upload this file to your
|onprem| instance, use the
:doc:`Update the Version Manifest </reference/api/version-manifest/update-version-manifest>`
resource.

.. warning::

   Don't change any values in the |json| file. The parameters and
   values in this file -- as well as how they are structured -- can
   change without notice.

Resource
--------

Base URL: https://opsmanager.mongodb.com/static/version_manifest

.. code-block:: http

   GET {+manifest-version+}.json

Response
--------

These values may change at any time without notice. They remain
compatible with |onprem|.

Example Request
---------------

.. code-block:: sh

   curl --header "Content-Type: application/json" \
        --request GET "https://opsmanager.mongodb.com/static/version_manifest/{+manifest-version+}.json?pretty=true" \
        --output {+manifest-version+}.json

Example Response
----------------

Response Header
~~~~~~~~~~~~~~~

.. include:: /includes/api/api-headers/200.rst

Response Body
~~~~~~~~~~~~~

The |json| file includes all data that your |onprem| instance needs.
This covers all available versions and variants of MongoDB to download
and deploy to |onprem|\-managed MongoDB hosts.

The parameters and values in this file -- as well as how they are
structured -- can change without notice.
