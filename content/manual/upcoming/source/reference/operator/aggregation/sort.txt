=========================
$sort (aggregation stage)
=========================

.. default-domain:: mongodb

.. facet::
   :name: programming_language
   :values: shell

.. meta::
   :description: Use the MongoDB $sort aggregation stage to sort documents in ascending or descending order by one or more fields. The sorted documents are passed to the next pipeline stage.

Definition
----------

.. pipeline:: $sort

   Sorts all input documents and returns them to the pipeline in sorted
   order.

Compatibility
-------------

.. |operator-method| replace:: ``$sort``

.. include:: /includes/fact-compatibility.rst

Syntax
------

``$sort`` has the following form:

.. code-block:: javascript

   { $sort: { <field1>: <sort order>, <field2>: <sort order> ... } }

``$sort`` takes a document that specifies the fields to
sort by and their sort order. ``<sort order>`` can have one
of the following values:

.. list-table::
   :header-rows: 1
   :widths: 35 65

   * - Value
     - Description

   * - ``1``
     - Sort ascending.

   * - ``-1``
     - Sort descending.

   * - ``{ $meta: "textScore" }``
     - Sort by the computed ``textScore`` metadata in descending
       order. For an example, see :ref:`sort-pipeline-metadata`.

When sorting on multiple fields, ``$sort`` evaluates sort order from
left to right. Documents are first sorted by ``<field1>``, then
documents with the same ``<field1>`` value are sorted by
``<field2>``.

Sort Consistency
~~~~~~~~~~~~~~~~

.. include:: /includes/fact-sort-consistency.rst

Consider the ``movies`` collection in the ``sample_mflix`` sample
dataset. The following command sorts movies released in 1925 or 1926
by ``year`` in ascending order:

.. literalinclude:: /code-examples/tested/command-line/mongosh/aggregation/stages/sort/sort-by-year.snippet.sort-by-year.js
   :language: javascript
   :copyable: true
   :category: usage example

Because the ``year`` field has duplicate values for ``1925`` and
``1926``, documents with the same value may sort differently across
executions. For example, two runs of the previous command produce
different orderings:

.. literalinclude:: /code-examples/tested/command-line/mongosh/aggregation/stages/sort/sort-by-year-ordering-1-output.sh
   :language: javascript
   :copyable: false

.. literalinclude:: /code-examples/tested/command-line/mongosh/aggregation/stages/sort/sort-by-year-ordering-2-output.sh
   :language: javascript
   :copyable: false

An operation sorts values for ``year`` in ascending order, but the order
of the documents containing duplicate values for ``year`` is not the
same. For example, duplicate values for ``1925`` and ``1926``. 

To achieve a *consistent sort*, add a field that contains only
unique values to the sort. The following command sorts on both the
``year`` field and the ``_id`` field:

.. literalinclude:: /code-examples/tested/command-line/mongosh/aggregation/stages/sort/sort-by-year-and-id.snippet.sort-by-year-and-id.js
   :language: javascript
   :copyable: true
   :category: usage example

The ``_id`` field always contains unique values and the sort
order is consistent for multiple sort executions.

$sort Index Use
~~~~~~~~~~~~~~~

The ``$sort`` operator can use an index when it is the first
stage in a pipeline or is preceded only by a :pipeline:`$match`
stage.

When you use ``$sort`` on a sharded cluster, each shard
sorts its result documents using an index where available. Then the
:binary:`~bin.mongos` or one of the shards performs a streamed merge
sort.

      ~~~~~~~~~~~~~~~~~~~~~~~~~

      .. include:: /includes/text-search-legacy-atlas-section.rst

      For a pipeline that includes :query:`$text`, you can sort by
      descending relevance score using the :expression:`{ $meta: "textScore"
      } <$meta>` expression. In the ``{ <sort-key> }`` document, set the
      :expression:`{ $meta: "textScore" } <$meta>` expression to an arbitrary
      field name. The field name is ignored by the query system. For example:

      .. literalinclude:: /code-examples/tested/command-line/mongosh/aggregation/stages/sort/text-score-sort.snippet.text-score-sort.js

         :copyable: true
         :category: usage example

      This operation matches documents with ``$text``, then sorts
      descending by ``"textScore"`` metadata and by ``year``, and
      returns the first five results. The ``score`` field name in the
      sort document is ignored by the query system. The projection doesn't
      include the ``"textScore"`` metadata and does not return the
      metadata as part of the matching documents. See
      :expression:`$meta` for more information.
