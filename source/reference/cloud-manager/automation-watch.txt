.. _mcli-cm-automation-watch-command:

=======================================
mongocli cloud-manager automation watch
=======================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``automation watch`` command monitors the specified 
|cloud-short| project's automation configuration status until it reaches
a goal state, at which time it displays a confirmation message.

.. _cm-automation-watch-syntax:

Syntax 
------

.. code-block:: text
   
   mongocli cloud-manager|cm automation watch 
        [ --profile|-P <profile-name> ]
        [ --projectId projectId ]

.. include:: /includes/fact-command-line-help.rst

.. _cm-automation-watch-options:

Options
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project
       for which you want to poll the automation configuration status. 
       If omitted, uses the project ID in the profile or the 
       :ref:`environment variable <mcli-env-var>`. 
     - no

.. _cm-automation-watch-output: 

Output 
------

The ``automation watch`` command continuously polls a |cloud-short| 
project's automation configuration status until it reaches a goal 
state. While the command is polling, it periodically prints dots 
(``.``) to the terminal.

When the automation configuration reaches a goal state, or if it is 
already in an stable state, the command prints a confirmation message.

If the command prints an error, see 
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. _cm-automation-watch-examples:

Example
-------

The following ``mongocli cloud-manager automation watch`` command 
monitors the |cloud-short| project associated with the default 
profile. 

.. code-block:: text
   :copyable: false

   mongocli cloud-manager automation watch

In this example project, the |cloud-short| replica set is restarted 
just before running the command. The command prints dots while the 
replica set is restarting, then prints a status message:

.. code-block:: text
   :copyable: false

   .......................
   Changes deployed successfully
