====================================
mongocli ops-manager security enable
====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``security enable`` command enables an :opsmgr:`authentication
mechanism </tutorial/enable-mongodbcr-authentication-for-group/>`
for the specified |mms| clusters in the given project. You can also
enable authentication through the |mms| UI.

Syntax
------

.. code-block:: text

   mongocli ops-manager security enable [MONGODB-CR|SCRAM-SHA-256] 
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]
        
.. include:: /includes/fact-command-line-help.rst

Options
-------

.. include:: /includes/basic-options.rst

Output 
------

A successful ``security enable`` command returns an in-progress status message
with a link to your |mms| topology page.

Example
-------

The example below uses the ``mongocli ops-manager security enable`` command to
enable ``SCRAM-SHA-256`` on a project specified in the configuration file.

The command uses the profile named ``omProfile`` for accessing 
|mms|.

.. code-block:: sh

   ~ mongocli ops-manager security enable SCRAM-SHA-256 --profile omProfile

You can also enable multiple authentication mechanisms with a single ``mongocli``
command. The following examples enables ``MONGODB-CR`` and  ``SCRAM-SHA-256``
on a project specified in the configuration file ``omProfile``.

.. code-block:: sh

   ~ mongocli cloud-manager security enable MONGODB-CR SCRAM-SHA-256 --profile omProfile
