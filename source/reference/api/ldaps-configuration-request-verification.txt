=============================
Verify One LDAP Configuration
=============================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

Requests a verification of an |ldap| configuration over |tls| for an
|service| project. Pass the ``requestId`` in the response object to the
:doc:`Verify |ldap| Configuration </reference/api/ldaps-configuration-verification-status>`
endpoint to get the status of a verification request. |service|
retains only the most recent request for each project.

.. note::

   - .. include:: /includes/facts/explain-ldap.rst

   - .. include:: /includes/facts/projects-groups-synonyms.rst

   - This endpoint does not verify the ``ldap.userToDNMapping``
     document array. To verify that users can authenticate with this
     parameter, use the
     :manual:`mongoldap </reference/program/mongoldap>` package
     component bundled with MongoDB Enterprise 3.6 or later with a
     config file that includes the same |ldap| parameters that you
     specify for |service|.

Resource
--------

.. include:: /includes/api/base-api-uri.rst

.. code-block:: http

   POST /groups/{GROUP-ID}/userSecurity/ldap/verify

   :widths: 20 14 11 55

   * - GROUP-ID
     - string
     - Required
     - Unique 24-hexadecimal digit string for the |service| project
       associated with the |ldap| over |tls| configuration.
