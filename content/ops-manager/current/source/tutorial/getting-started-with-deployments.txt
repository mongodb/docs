================================
Getting Started with Deployments
================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

|onprem| is a web application that lets you create, manage, monitor,
and back up MongoDB deployments. The web application runs on the Ops
Manager server, and software agents run on the servers running MongoDB.

You can use |onprem| to create new deployments and manage existing
ones. To monitor existing deployments, you need to download |onprem|
agents to the servers in the MongoDB deployment.

Prerequisites
-------------

Before you begin, confirm the following are in place:

- |onprem| is installed, configured, and running. For installation
  guidance, see :ref:`om-install-rpm`, :ref:`om-install-deb`, or
  :ref:`om-install-archive`.
- An |onprem| organization and project exist for the deployment.
- The MongoDB Automation Agent is installed on each target host.
  To learn more, see :ref:`add-servers-automation`.
- The user account has the :authrole:`Project Automation Admin` role
  or the :authrole:`Project Owner` role.

**Networking requirements:**

- The Automation Agent on each target host must reach |onprem| on
  port 8080 (HTTP) or 8443 (HTTPS).
- Port 27017 (default) must be open between all MongoDB hosts in the
  same deployment.
- |onprem| requires outbound HTTPS access to
  ``downloads.mongodb.com`` and ``fastdl.mongodb.org`` to download
  MongoDB binaries. For air-gapped environments, configure Local
  Mode instead.

Verify the Deployment
---------------------

After the deployment provisions, confirm it is healthy before
connecting.

.. list-table::
   :header-rows: 1
   :widths: 40 60

   * - Checkpoint
     - Verification Step

   * - Deployment submitted
     - The |onprem| UI shows the deployment in your project with a
       status of ``OK``.

   * - Automation Agent goal state reached
     - In the |onprem| UI, the Automation Agent status for each host
       shows a green check, indicating the goal state was reached.

   * - All members healthy
     - In the |onprem| UI, all deployment members show a healthy
       status with their expected roles.

   * - Connection succeeds
     - Connect to the primary using {+mongosh+} and run
       ``rs.status()`` to confirm all members are in the expected
       state.

.. _om-getting-started-deployments-next-steps:

Next Steps
----------

After you verify your deployment, you can:

- **Deploy a replica set**: Create a managed replica set. To learn
  how, see :ref:`om-deploy-replica-set`.

- **Configure authentication**: Set up access control to secure your
  deployment. To learn how, see :ref:`enable-authentication`.

- **Set up backup**: Configure continuous backup to protect your
  data. To learn how, see :ref:`backup-overview`.

- **Add Automation Agents**: Bring more hosts under management. To
  learn how, see :ref:`add-servers-automation`.
