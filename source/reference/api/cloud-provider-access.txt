=====================
Cloud Provider Access
=====================

.. default-domain:: mongodb

.. include:: /includes/fact-projects-groups-synonyms.rst

The ``cloudProviderAccess`` resource allows you to register and authorize
|aws| IAM roles in |service|.

.. include:: /includes/api-base-url-new.rst

Endpoints
---------

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description

   * - ``GET``
     - :doc:`/cloudProviderAccess </reference/api/cloud-provider-access-get-roles>`
     - Retrieve existing |aws| IAM roles.

   * - ``POST``
     - :doc:`/cloudProviderAccess </reference/api/cloud-provider-access-create-a-role>`
     - Create an |aws| IAM role.

   * - ``PATCH``
     - :doc:`/cloudProviderAccess/{ROLE-ID} </reference/api/cloud-provider-access-authorize-one-role>`
     - Authorize and configure an |aws| Assumed IAM role.

   * - ``DELETE``
     - :doc:`/cloudProviderAccess/{ROLE-ID} </reference/api/cloud-provider-access-deauthorize-one-role>`
     - Deauthorize an |aws| Assumed IAM role.
