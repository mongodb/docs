.. _mcli-om-admin-backup-sync-update-cmd:

=============================================
mongocli ops-manager admin backup sync update
=============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/admonitions/admin-requires-global-owner.rst

The ``admin backup sync update`` command updates one
:opsmgr:`sync store </reference/glossary/#term-sync-store>`
configuration. You can also update a sync store configuration
using the using the :guilabel:`Snapshot Storage` page in the
|onprem| console.

.. _mcli-om-admin-backup-sync-update-syntax: 

Syntax
------

.. code-block:: shell 

   mongocli ops-manager|om admin backup sync update 
   <sync-store-config-ID>
           [ --assignment ]
        [ --encryptedCredentials ]
        [ --label <tags> ]
        [ --maxCapacityGB <amount-of-data> ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --ssl ]
        --uri <hosts>
        [ --writeConcern <write-concern> ]

.. _mcli-om-admin-backup-sync-update-args: 

Arguments
---------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Argument
     - Type
     - Description
     - Required?

   * - ``sync-store-config-ID``
     - string
     - Unique identifier of the sync store that you want to update.
       You can retrieve the list of sync store IDs for
       a project with the
       :ref:`List Backup Sync Store Configurations <mcli-om-admin-backup-sync-list-cmd>` command.
     - yes

.. _mcli-om-admin-backup-sync-update-options: 

Options
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   
   * - ``--assignment``
     - 
     - Flag that indicates whether this sync store can be
       assigned backup jobs.
     - no

   * - ``--encryptedCredentials``
     - 
     - Flag that indicates whether the username and password for
       this sync store were encrypted using the
       :opsmgr:`credentialstool </tutorial/encrypt-user-credentials/>`.
     - no

   * - ``--label``
     - array of strings
     - Array of tags that specify which :opsmgr:`backup jobs 
       </reference/glossary/#term-backup-job>` |onprem| can assign
       to which sync stores. If omitted, this 
       sync store can only process the backup jobs for projects that
       don't use labels to filter their jobs.
     - no

   * - ``--maxCapacityGB``
     - number
     - Maximum amount of data in GB that this sync store can store.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private
       keys for the project are saved. If omitted, uses the
       {+default-profile+}. To learn more about creating a
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--ssl``
     - 
     - Flag that indicates whether this sync store only accepts
       connections encrypted using |tls|.
     - no

   * - ``--uri``
     - string
     - Comma-separated list of hosts in the ``<hostname:port>`` 
       format to use to access this sync store.
     - yes

   * - ``--writeConcern``
     - string
     - :manual:`Write concern </reference/write-concern/>` to use for 
       this sync store. Valid values are: 

       - ``ACKNOWLEDGED``
       - ``W2``
       - ``JOURNALED``
       - ``MAJORITY``

     - no

.. _mcli-om-admin-backup-sync-update-output: 

Output 
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh 
   :copyable: false 

   Sync store configuration '<sync-store-config-ID>' updated.

The default output contains a subset of the fields returned by this
command. For the complete list of |json| fields returned by the
command, see the :opsmgr:`API reference 
</reference/api/admin/backup/sync/mongoConfigs/update-one-sync-store-configuration-by/#response>`.

.. _mcli-om-admin-backup-sync-update-egs:

Example
-------

The following ``mongocli om admin backup sync update`` command
updates the ``test`` sync store. The  command uses the {+default-profile+},which contains the credentials for accessing |onprem|.

.. tabs:: 

   .. tab:: Default Output 
      :tabid: default 

      .. code-block:: shell 

         mongocli om admin backup sync update test --uri mongodb://localhost:27017

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         Sync store configuration 'test' updated.

   .. tab:: JSON Output 
      :tabid: json 

      .. code-block:: shell 

         mongocli om admin backup sync log update test --uri mongodb://localhost:27017 --output json

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         .. code-block:: shell
         :copyable: false

         {
           "id": "test",
           "uri": "mongodb://localhost:27017",
           "ssl": false,
           "assignmentEnabled": true,
           "encryptedCredentials": false,
           "loadFactor": 1
         }
