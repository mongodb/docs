====================
Public API Resources
====================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The |mms| :doc:`Public API </core/api>` exposes the following resources:

.. list-table::
   :widths: 25 75

   * - :doc:`/reference/api/root`

     - The starting point for the |mms| Public API. From here, you can
       traverse the links to reach all other API resources.

   * - :doc:`/reference/api/hosts`

     - Retrieves and edits the MongoDB processes in your deployment.

   * - :doc:`/reference/api/agents`

     - Retrieves the deployment's Automation, Monitoring, and Backup Agents.

   * - :doc:`/reference/api/metrics`

     - Deprecated: Retrieves data points for your deployments.

   * - :doc:`/reference/api/clusters`

     - Retrieves and edits the deployment's :term:`replica sets <replica set>`
       and :term:`sharded clusters <sharded cluster>`.

   * - :doc:`/reference/api/groups`

     - Retrieves and edits groups and group membership.

   * - :doc:`/reference/api/users`

     - Retrieves and edits users and their roles.

   * - :doc:`/reference/api/alerts`

     - Retrieves and acknowledges alerts.

   * - :doc:`/reference/api/alert-configurations`

     - Retrieves and edits :doc:`alert configurations
       </tutorial/manage-alert-configurations>`, which define the conditions
       that trigger alerts and the methods of notification.

   * - :doc:`/reference/api/maintenance-windows`

     - Turns off alert notifications for specified alert types for a period of
       time, to allow maintenance to occur.

   * - :doc:`/reference/api/backup-configurations`

     - Retrieves and edits the settings used to back up MongoDB.

   * - :doc:`/reference/api/snapshot-schedule`

     - Retrieves and edits how often |mms| captures snapshots and how long
       |mms| retains them.

   * - :doc:`/reference/api/snapshots`

     - Retrieves snapshot metadata and removes existing snapshots.

   * - :doc:`/reference/api/checkpoints`

     - Retrieves :ref:`checkpoint <checkpoint>` metadata.

   * - :doc:`/reference/api/restore-jobs`

     - Retrieves and creates requests to restore your data to a previous state.

   * - :doc:`/reference/api/whitelist`

     - Retrieves and edits addresses on a user's :ref:`whitelist
       <whitelist-for-api-operations>`.

   * - :doc:`/reference/api/automation-config`

     - Retrieves and update the groupâ€™s :doc:`automation configuration
       </reference/cluster-configuration>`, which defines the deployment's
       MongoDB processes and agents.

   * - :doc:`/reference/api/automation-status`

     - View the status of an updated :doc:`automation configuration
       </reference/cluster-configuration>`.
