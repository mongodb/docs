*Release Date TBD*

v20260520
---------

*Released 20 May, 2026*

Updates guidance related to cluster reliability in the following pages:

- :ref:`arch-center-reliability` page: Restructured to present
  high availability, disaster recovery, and business continuity
  planning as three complementary strategies. Adds
  :ref:`arch-center-bcp` to the table of contents.
- :ref:`arch-center-bcp` page: New page that describes how to
  combine high availability, disaster recovery, and planning
  into a comprehensive resilience strategy.
- :guilabel:`Guidance for Atlas Backups` page: Deprecated and redirects to
  the new :ref:`arch-center-dr` page. Information about backup
  strategies and RTO/RPO trade-offs has been moved to the new
  :ref:`arch-center-dr` page.  
- :ref:`arch-center-high-availability` page: Adds a deployment
  paradigm RTO/RPO trade-off table comparing single-region,
  multi-region, multi-geo, and multi-cloud deployments, and
  adds guidance on predictive auto-scaling.
- :ref:`arch-center-dr` page: Adds guidance on when to use
  disaster recovery versus high availability, backup strategy RTO/RPO
  trade-offs, and detailed failure scenario recovery steps.
- :ref:`arch-center-landing-zone` page: Updates the reliability
  section to present high availability and disaster recovery as
  complementary strategies, and adds a reference to the new
  :ref:`arch-center-bcp` page.
- :ref:`arch-center-paradigms-single` page: Clarifies automatic
  failover behavior, adds guidance on the majority write concern
  requirement for zero data loss, and adds Terraform automation guidance
  using the Atlas Cluster Terraform Module.

Updates guidance related to auto-scaling in the following pages: 

- :ref:`arch-center-scalability` page: Adds distinction
  between reactive auto-scaling (based on current usage) and
  predictive auto-scaling (ML-based forecasting for cyclical
  patterns), and guidance on when to pre-scale manually.
- :ref:`arch-center-monitoring-alerts` page: Adds a section on
  auto-scaling alert events for both reactive and predictive
  auto-scaling, and explains how auto-scaling events appear in
  the activity feed.

Updates Terraform guidance in the following pages: 

- Adds guidance for automating deployments using the MongoDB Atlas
  Cluster Terraform Module to the following pages: 

  - :ref:`arch-center-paradigms-single`
  - :ref:`arch-center-paradigms-multi-region`
  - :ref:`arch-center-paradigms-multi-cloud`
- :ref:`arch-center-data-encryption` page: Updates Terraform
  examples to use registry-based modules for AWS KMS and
  Azure Key Vault configurations.
- :ref:`arch-center-network-security` page: Updates PrivateLink
  Terraform configuration examples to use registry-based
  modules for AWS and Azure.

v20251125
---------

**Released 25 November, 2025**

- Updates all Terraform examples to use MongoDB Atlas Terraform Provider version 2.x (``~> 2.2``).
  If you're upgrading from provider version 1.x, see
  the `MongoDB Atlas Terraform Provider 2.0.0 Upgrade Guide
  <https://registry.terraform.io/providers/mongodb/mongodbatlas/latest/docs/guides/2.0.0-upgrade-guide>`__
  for breaking changes and migration steps. Examples now use the ``mongodbatlas_advanced_cluster``
  resource with v2.x syntax.
- Adds the :ref:`arch-center-compliance-gdpr` page.

v20250829
---------

**Released 29 August, 2025**

- Adds opinionated :ref:`Multi-Region 
  <arch-center-paradigms-multi-region>` 
  guidance.
- Adds a new :ref:`Reliability <arch-center-reliability>` section.
- Enhances the :ref:`Getting Started <arch-center-getting-started>` 
  section with a new :ref:`Operational Readiness Checklist 
  <arch-center-checklist>`.
- Enhances :ref:`Security <arch-center-security>` guidance.

Specific page and section updates:

- :ref:`arch-center-landing-zone` page: Content reorganization for clarity, and updates to add that the primary is 
  mapped *randomly* to a zone, and link to new guidance pages.
- :ref:`arch-center-hierarchy` page: Adds organization creation example for Terraform. 
- :ref:`arch-center-network-security` page: Minor update to mention limitation for changing the number of service attachments 
  and recommendation to use the DNS seedlist connection string. 
- :ref:`arch-center-compliance` page: Minor update to introduce |atlas-gov| as an {+service+} compliance option.
- :ref:`arch-center-high-availability` page: Adds guidance for 3- and 5-node replica set topologies. 
- :ref:`arch-center-dr` page: Minor update to recommend multi region clusters and electable nodes for high availability,
  and link to other pages with existing guidance.
- :ref:`arch-center-cost-saving-config` page: Minor update to link to cost-analysis guidance for different deployment topologies.
- Adds the following new pages:
 
  - :ref:`arch-center-migration`: Describes how to migrate data from your on-premises MongoDB deployments to {+service+}.
  - :ref:`arch-center-checklist`: Provides a checklist to help you prepare your environment and team for a successful {+service+} deployment. 
  - :ref:`arch-center-paradigms`: Introduces and compares different {+service+} deployment paradigms, which are described with use-cases in the following new pages:

    - :ref:`arch-center-paradigms-single` 
    - :ref:`arch-center-paradigms-multi-region`
    - :ref:`arch-center-paradigms-multi-cloud`
    - :ref:`arch-center-paradigms-hybrid` 
- Adds the :ref:`arch-center-compliance-dora` page.
