.. _dl-drop-database-cmd:

================
``dropDatabase``
================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

The ``dropDatabase`` command removes the specified database from the storage 
configuration. If you drop a database that contains one or more collections, 
the collections are also removed from the storage configuration.

.. _dl-drop-database-cmd-syntax:

Syntax 
~~~~~~

.. code-block:: json 

   db.runCommand({ "dropDatabase": 1 })

.. _dl-drop-database-cmd-params:

Parameters 
~~~~~~~~~~

.. list-table:: 
   :header-rows: 1
   :widths: 10 10 70 10

   * - Parameter
     - Type
     - Description
     - Required?

   * - ``1``
     - int
     - The flag to pass when dropping a database from the storage 
       configuration.
     - yes

.. _dl-drop-database-cmd-output:

Output 
~~~~~~ 

The command prints the following output if the command succeeds. To check 
whether the database was dropped from the storage configuration, run 
the ``show dbs`` and ``storageGetConfig`` commands. If the command prints errors, 
see :ref:`dl-drop-database-cmd-errors` below for recommended solutions.

.. code-block:: json 
   :copyable: false

   { "ok" : 1, "dropped" : "<database>" }

where ``<database>`` is the name of the database that was dropped from 
the storage configuration.

If there is no active database to drop and therefore no database was dropped, 
the command returns the following output:

.. code-block:: json 
   :copyable: false

   { "ok" : 1 }

.. _dl-drop-database-cmd-egs:

Example 
~~~~~~~

The following example uses the ``dropDatabase`` command to drop a 
database named ``egS3Store`` from the {+data-lake+} storage 
configuration. 

.. code-block:: json 

   use egS3Store
   db.runCommand({ "dropDatabase": 1 })

The previous command prints the following output:

.. code-block:: json 
   :copyable: false 

   { "ok" : 1, "dropped" : "egS3Store" }

.. _dl-drop-database-cmd-errors:

Troubleshoot Errors 
~~~~~~~~~~~~~~~~~~~

If the command fails, it prints the following error: 

.. code-block:: json 
   :copyable: false

   {
      ok: 0,
      errmsg: “have to pass 1 as db parameter”,
      code: 20,
      codeName: "IllegalOperation"
   }

**Solution:** Specify ``1`` as the parameter to the command.
