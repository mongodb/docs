.. _cloud-backup-schedules:

=====================================
{+Cloud-Backup+} Schedule
=====================================

.. default-domain:: mongodb

.. include:: /includes/fact-projects-groups-synonyms.rst

.. important:: Applies to Scheduled {+Cloud-Backup+}s Only

   This resource applies to:

   - Clusters using :ref:`backup-cloud-provider` and
   - Scheduled snapshots, not :doc:`on-demand snapshots </reference/api/cloud-backup/backup/take-one-ondemand-backup>`.

The ``/backup/schedule`` resource enables you to view and modify
the snapshot scheduling and retention settings for an |service|
cluster with :ref:`backup-cloud-provider` enabled.

.. include:: /includes/api-base-url.rst

Endpoints
---------

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description

   * - ``GET``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/schedule </reference/api/cloud-backup/schedule/get-all-schedules>`
     - Get the current snapshot schedule and retention settings for the
       cluster with ``{CLUSTER-NAME}``.

   * - ``PATCH``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/schedule </reference/api/cloud-backup/schedule/modify-one-schedule>`
     - Modify the snapshot schedule or retention settings for the
       cluster with ``{CLUSTER-NAME}``.

       .. include:: /includes/api/facts/api-access-list-required.rst

   * - ``DELETE``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/schedule </reference/api/cloud-backup/schedule/delete-all-schedules>`
     - Delete all the snapshot schedules for the cluster with 
       ``{CLUSTER-NAME}``.
