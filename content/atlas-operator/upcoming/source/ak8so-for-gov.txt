============================================================
|ak8so| for |service| for Government Quick Start
============================================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 0
   :class: singlecol

You can use the |ak8so| to manage resources in |service| for Government.
The |ak8so| only supports managing |service| for Government on |aws| and |gcp|. It
doesn't support {+adf+}, {+Serverless-instances+}, or {+Flex-clusters+}.
For more information, see :atlas:`Supported Features
</government/overview/supported-features/>` in the |service-fullname|
for Government documentation. 

This page describes how to configure the |ak8so| to manage resources in
|service| for Government.

Prerequisites
-------------

You must install the |ak8so| using {+atlas-cli+} with the ``--gov``
flag. To learn more about installing the |ak8so| using the
{+atlas-cli+}, see :ref:`ak8so-quick-start-ref`.

Procedure 
---------

After you install the |ak8so|, do the following:

.. procedure:: 
   :style: normal 

   .. step:: Deploy |ak8so|. 

      Follow the procedure in the :ref:`ak8so-quick-start-ref` to deploy
      |ak8so|.

   .. step:: Set the cloud provider region for |service| for Government.

      Use the ``spec.regionUsageRestrictions`` parameter in the
      ``AtlasProject`` custom resource to set the cloud provider region for
      |service| for Government. If you omit the
      ``spec.regionUsageRestrictions`` parameter,
      this setting defaults to ``GOV_REGIONS_ONLY`` To learn more, see
      :ref:`spec.regionUsageRestrictions <atlasproject-spec>`.
