Overview
--------

In this guide, you can learn how to use {+odm+} to run
{+vector-search+} queries. {+vector-search+} allows you to query your
documents based on their semantic meaning by using vector embeddings.

{+odm+} provides the ``vector_search`` method, which builds and runs a
``$vectorSearch`` aggregation stage for you. To learn more about the
``$vectorSearch`` aggregation stage, see the :ref:`mongoid-aggregation` guide.

.. note:: Deployment Compatibility

   To use {+vector-search+}, you must connect to a MongoDB Atlas
   cluster or a self-managed deployment running MongoDB 8.2 or later.

To learn more about {+vector-search+}, see the :atlas:`{+vector-search+}
Overview </atlas-vector-search/vector-search-overview/>` in the Atlas
documentation.

field and a vector search index on your model, and then create the index on
your collection. To learn how to declare and create vector search

Indexes guide.

Run a Vector Search Query
-------------------------

To run a {+vector-search+} query, call the ``vector_search`` class
method on your model and pass a query vector as the first argument. The
query vector is a vector embedding that represents your search input.

The following code example runs a {+vector-search+} query on the
``plot_embedding`` field and prints the title of each matching movie.
Because the ``Movie`` model declares only one vector search index,
{+odm+} automatically selects the index and vector field to use.

.. io-code-block::
   :copyable: true

   .. input:: /includes/interact-data/vector-search.rb
      :language: ruby
      :start-after: start-basic-query
      :end-before: end-basic-query

   .. output::
      :language: none
      :visible: false

      Thrill Seekers
      About Time
      The Time Machine
      Timecop
      Crusade in Jeans

The ``vector_search`` method returns an array of ``Movie`` documents.
By default, it returns the 10 most similar documents. To change the
number of results, set the ``limit`` option. The preceding example specifies a
limit of ``5`` results.

Vector Search Options
---------------------

You can pass the following keyword arguments to the ``vector_search``
method to customize your query:

.. list-table::
   :header-rows: 1
   :widths: 20 20 40 20

   * - Option
     - Type
     - Description
     - Default Value

   * - ``index``
     - ``String`` or ``Symbol``
     - Name of the vector search index to use. Required only if the
       model declares more than one vector search index.
     - Inferred from the model

   * - ``path``
     - ``String`` or ``Symbol``
     - Field that contains the vector embeddings. Required only if the
       vector field can't be inferred from the index definition.
     - Inferred from the index

   * - ``limit``
     - ``Integer``
     - Maximum number of documents to return.
     - ``10``

   * - ``num_candidates``
     - ``Integer``
     - Number of nearest neighbors to consider during the search.
     - ``limit * 10``

   * - ``filter``
     - ``Hash``
     - Query used to pre-filter documents before the search.
     - No filtering

   * - ``pipeline``
     - ``Array``
     - Additional aggregation stages to append after the vector search.
     - No additional stages
