==============================
Generated Custom Resources
==============================

.. default-domain:: mongodb

.. meta::
   :description: Learn about Atlas Kubernetes Operator custom resources auto-generated from the MongoDB Atlas Administration API OpenAPI specification.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

Overview
--------

|ak8so| provides a set of custom resource definitions (CRDs) that are
auto-generated from the MongoDB {+atlas-admin-api+} OpenAPI
specification. These generated CRDs offer direct parity with the
{+atlas-admin-api+} and are updated regularly as the API evolves. Over
time, generated CRDs will replace the existing
:ref:`custom resources <custom-resources>` as the standard interface
for managing |service| resources through |ak8so|.

.. important:: Incompatible with Existing Custom Resources

   Generated CRDs are **not** compatible with the existing set of
   :ref:`custom resources <custom-resources>`. You cannot mix generated
   and existing CRDs to manage the same |service| resource. Choose one
   set of CRDs for each resource you manage through |ak8so|.

.. _generated-crds-choosing:

Choosing Between CRD Sets
--------------------------

Because |ak8so| currently offers a limited number of generated CRDs,
you should evaluate which set best fits your needs:

.. list-table::
   :widths: 50 50
   :header-rows: 1

   * - Existing Custom Resources
     - Generated Custom Resources

   * - Full coverage of |service| resource types.
     - Limited but expanding coverage of |service| resource types.

   * - Will be deprecated in a future release.
     - Will become the standard for managing |service| resources
       through |ak8so|.

   * - Stable and well-established.
     - Closely tracks the {+atlas-admin-api+} OpenAPI specification.

If the resource you need is available as a generated CRD, consider
adopting it now to reduce the future effort of migrating from the
existing CRDs. If a generated CRD is not yet available for your
resource, continue to use the existing :ref:`custom resources
<custom-resources>`.

.. _generated-crds-list:

Available Generated CRDs
-------------------------

|ak8so| currently supports the following generated CRDs:

.. list-table::
   :header-rows: 1
   :widths: 30 70

   * - Resource
     - Description

   * - :ref:`group-custom-resource`
     - Group in |service|.

   * - :ref:`cluster-custom-resource`
     - Cluster in |service|.

   * - :ref:`databaseuser-custom-resource`
     - Database user in |service|.

   * - :ref:`flexcluster-custom-resource`
     - Flex cluster in |service|.

   * - :ref:`ipaccesslistentry-custom-resource`
     - IP access list entry in |service|.

As the {+atlas-admin-api+} OpenAPI specification evolves, additional
generated CRDs will become available. Check this page for updates.
