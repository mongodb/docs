===========
Development
===========

.. meta::
   :description: Learn how to use MongoDB in your applications. Connect to databases, perform CRUD operations, optimize with indexes, and design data models.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

This page provides guidance for developers to build applications with
MongoDB. Learn how to connect to deployments, work with your data, add
application features, and scale for production. Select a page section
depending on where you are in your development journey.

Start Development
-----------------

If you are just starting with MongoDB, follow this path to build your first
application:

1. :ref:`Connect to your deployment <mongodb-uri>` from your
   application.
2. :ref:`Perform CRUD operations <crud>` to create, read, update, and
   delete data.
3. :ref:`Model your data <manual-data-modeling-intro>` to fit your
   application's access patterns.
4. :ref:`Run aggregation pipelines <aggregation>` to analyze and
   transform your data.

Learn the Basics
----------------

Use the following pages to understand your deployment and start
interacting with your data:

:ref:`Documents <bson-document-format>`
  Documents are the basic unit of data storage in MongoDB. Learn how
  MongoDB stores and structures data as BSON documents.

:ref:`Databases and Collections <databases-and-collections>`
  Databases and collections are used to organize documents in MongoDB.
  Learn how to create and manage databases and collections.

:ref:`CRUD Operations <crud>`
  Learn how to create, read, update, and delete documents in
  collections.

:ref:`Data Modeling <manual-data-modeling-intro>`
  Data modeling refers to the organization of data within a database
  and the links between related entities. Learn how to design your data
  model to best suit your application's needs.

:ref:`Aggregation Operations <aggregation>`
  Use aggregation pipelines to process documents and return computed
  results. Learn how to use aggregation operations to analyze and
  transform your data.

:ref:`Indexes <indexes>`
  Indexes support efficient query execution in MongoDB. Learn how to
  create and manage indexes to optimize query performance.

:ref:`Reference <reference-landing>`
  Reference documentation for MongoDB commands, operators, and
  configuration options.

Build App Features
------------------

Use the following pages to add features such as |fts| and |avs| to
your application:

:ref:`Search <fts-top-ref>`
  |fts| is an embedded full-text search that gives you a seamless,
  scalable experience for building relevance-based app features. Learn
  how to get started with |fts| and implement advanced search
  capabilities in your applications.

:ref:`Vector Search <fts-vector-search>`
  By using MongoDB as a vector database, you can use |avs| to
  seamlessly search and index your vector data alongside your other
  MongoDB data. Learn how to implement vector search in your
  applications.

:ref:`AI Integrations <vector-search-integrations>`
  Learn how to integrate MongoDB with AI frameworks, tools, and
  platforms to build AI applications and agents.

:ref:`Transactions <transactions>`
  Transactions allow you to perform multiple read and write operations
  across one or more documents as a single atomic operation. Learn how
  to use transactions in your application and client library.

:ref:`Change Streams <changeStreams>`
  Change streams let applications access and respond to real-time data
  changes. Learn how to configure change streams to respond to changes
  in data.

:ref:`Time Series <manual-timeseries-landing>`
  Time series data is a sequence of data points that lets you analyze
  changes over time. Learn how to create and manage time series
  collections and work with time series data.

:ref:`In-Use Encryption <security-in-use-encryption>`
  In-use encryption secures data when transmitted, stored, and
  processed, and enables supported queries on that encrypted data. Learn
  about the different approaches to in-use encryption in MongoDB and how
  to use them in your applications.

:ref:`Data Federation <atlas-data-federation>`
  Data Federation lets you query data across multiple data sources,
  including S3, Atlas clusters, and other MongoDB databases. Learn how
  to set up and use Data Federation in your applications.
