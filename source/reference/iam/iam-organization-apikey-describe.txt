.. _mcli-iam-organization-apikey-describe:

=========================================
mongocli iam organization apikey describe
=========================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``iam organization apikey describe`` command provides details about
the specified organization |api| key.

.. _mcli-iam-organization-apikey-describe-syntax:

Syntax 
------

.. code-block:: text 

   mongocli iam organization|organizations|org|orgs apiKey|apiKeys|apikeys describe <api-key-id>
        [ --orgId <organization-id> ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. include:: /includes/fact-command-line-help.rst

.. _mcli-iam-organization-apikey-describe-arguments:

Arguments  
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - ``<api-key-id>``
     - string
     - Unique identifier for this organization API key.
     - yes

.. _mcli-iam-organization-apikey-describe-options:

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type 
     - Description 
     - Required? 

   * - ``--orgId``
     - string
     - Unique identifier of the organization whose |api| keys you want
       to describe. 
       
       If specified, this value overrides any organization ID
       stored in your :ref:`profile <mcli-profiles>` or
       :ref:`environment variable <mcli-env-var>`.

       If omitted, your profile must contain the
       organization ID.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P`` 
     - string 
     - Name of the profile where your credentials are saved.

       If omitted, uses the {+default-profile+}. To learn more about 
       creating a profile, see :ref:`mcli-configure`.
     - no

.. _mcli-iam-organization-apikey-describe-output:

Output
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see :ref:`Troubleshooting
<mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   ID         DESCRIPTION          PUBLIC KEY     PRIVATE KEY
   <key-id>   <key-description>    <public-key>   <private-key>

.. _mcli-iam-organization-apikey-describe-examples:

Examples
--------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command describes an organization |api| key using
      the {+default-profile+}, which contains credentials and the
      organization ID. The output is returned in the default format.

      .. code-block:: sh

         mongocli iam organization apikey describe 5e3b8aa3c1234019903c9fce

      The previous command prints the following to the terminal.

      .. code-block:: json
         :copyable: false

         ID                         DESCRIPTION          PUBLIC KEY   PRIVATE KEY
         5e3b8aa3c1234019903c9fce   User1 Key            abcdefgh     ********-****-****-123468e0b5be

   .. tab:: JSON Output
      :tabid: json-output

      The following command describes an |api| key using the ``myOM``
      profile, which contains credentials. The output is returned in
      |json| format.
      
      .. code-block:: sh

         mongocli iam organization apikey describe 5e3b8aa3c1234019903c9fce --orgId 5a1b39eec902201990f12345 -P myOM --output json

      The previous command prints the following to the terminal in |json| 
      format.

      .. code-block:: json
         :copyable: false

         [
           {
             "id": "5e3b8aa3c1234019903c9fce",
             "desc": "User1 Key",
             "roles": [
               {
                 "orgId": "5e429f2e12345c6eac4d59c9",
                 "roleName": "ORG_OWNER"
               }
             ],
             "privateKey": "********-****-****-123468e0b5be",
             "publicKey": "abcdefgh"
           }
         ]
