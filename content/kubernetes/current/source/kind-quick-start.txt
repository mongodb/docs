The |k8s-op-full| uses the |k8s| API and tools to manage MongoDB
clusters running within your |k8s| cluster. |k8s-op-short| works together 
with |onprem|. This tutorial demonstrates how to deploy |onprem| on 
|k8s| and then deploy and connect to your first MongoDB replica set 
with |k8s-op-short|. You can use |kind| to quickly set up a cluster. 
To learn more, see `Kind <https://kind.sigs.k8s.io/>`_.

Prerequisites
-------------

This tutorial requires:

- A running |k8s| cluster.
- `Helm <https://helm.sh/docs/intro/install/>`_ installed on your 
  local machine.
- |k8s| nodes running on :ref:`supported hardware architectures
  <k8s-supported-hardware-arch-compatibility>`.
- The |mongo-shell| installed on your local machine.

If you use |kind| with Docker Desktop, allocate at least **8 GB of 
memory** and **4 CPUs** in the Docker Desktop resource settings. 
|onprem| requires a minimum of 5 GB of memory for the application 
and additional resources for the {+appdb+} and the |k8s-op-short|.
