.. _auto-embedding-enable-disable:

Enable or Disable Automated Embedding 
-------------------------------------

.. tabs:: 

   .. tab:: Atlas
      :tabid: atlas

      To enable Automated Embedding, you must create a {+avs+} index with 
      the ``autoEmbed`` type. To learn more, see :ref:`Create an Automated 
      Embedding Index <avs-create-index>`.

      To disable Automated Embedding at the organization level, use an
      :ref:`{+arp+} <atlas-resource-policies-overview>`. You can
      disable Automated Embedding entirely or disable it with project-level
      exceptions.

      To disable Automated Embedding entirely, use the following syntax in your {+arp+}:

      .. code-block:: text
         :copyable: true

         forbid (
           principal,
           action == ResourcePolicy::Action::"search.index.modify",
           resource
         )
         when {
           context.search.index.isAutoEmbed
         };

      To disable Automated Embedding with exceptions for specific projects,
      use the following syntax in your {+arp+}. Replace each ``<project-id>`` with
      the ID of the project to exempt. To find your project ID, see:
      :ref:`atlas-modify-project-settings`.

      .. code-block:: text
         :copyable: true

         forbid (
           principal,
           action == ResourcePolicy::Action::"search.index.modify",
           resource
         )
         when {
           context.search.index.isAutoEmbed
         }
         unless {
           resource in ResourcePolicy::Project::"<project-id-1>" ||
           resource in ResourcePolicy::Project::"<project-id-2>"
         };

      When active, these policies prevent new ``autoEmbed`` type indexes
      from being created. Indexes created before the resource policy was
      activated are not affected and must be deleted manually to maintain
      compliance.

      To apply this policy to your organization:

      .. procedure::
         :style: normal

         .. step:: In |service|, go to the :guilabel:`Organization Settings` page.

            a. If it's not already displayed, select your desired
               organization from the |ui-org-menu| in the navigation bar.

            #. In the sidebar, click :guilabel:`Organization Settings`.

         .. step:: Go to the :guilabel:`Resource Policies` page.

            In the sidebar, click :guilabel:`Resource Policies` under
            the :guilabel:`Configurations` header.

         .. step:: Click :guilabel:`Create policy`.

         .. step:: Name, describe, and define your policy.

            a. Give the policy a unique name.
            #. (Optional) Add a description.
            #. In the :guilabel:`Cedar Policy` field, paste the Cedar
               policy. Replace ``<project-id>`` with your project ID.

         .. step:: Click :guilabel:`Create policy`.

   .. tab:: Self-Managed
      :tabid: self

      To enable Automated Embedding, you must do the following: 

      .. procedure:: 
         :style: normal

         .. step:: Create a {+voyageai+} API key for indexing operations.

            To learn more, see :ref:`voyage-api-keys`.

         .. step:: Create a {+voyageai+} API key for query operations.

            To learn more, see :ref:`voyage-api-keys`.

         .. step:: Configure ``mongot`` to use the API keys.

            To learn more, see :ref:`voyage-api-key-for-auto-embed`.

         .. step:: Create an ``autoEmbed`` type {+avs+} index.

            To learn more, see :ref:`Create an Automated Embedding Index <avs-types-vector-search>`.

.. _auto-embedding-usage:

View Automated Embedding Usage
------------------------------

To view aggregated token usage and embedding model requests for 
Automated Embedding service: 

.. tabs:: 

   .. tab:: Atlas
      :tabid: atlas

      .. procedure:: 
         :style: normal

         .. step:: Log in to `{+service+} <https://cloud.mongodb.com>`__.

         .. include:: /includes/nav/steps-search-simple.rst

         .. step:: Click the :guilabel:`Automated Embedding` section if it's not already expanded.

         .. step:: Click :guilabel:`Usage` under :guilabel:`Automated Embedding`.

            The `Automated Embedding Usage <https://cloud.mongodb.com/go?l=https%3A%2F%2Fcloud.mongodb.com%2Fv2%2F%3Cproject%3E%23%2FautoEmbedding%4E%23%2Fusage%2F>`__
            page displays. The :guilabel:`Automated Embedding Usage` page
            displays the following information:

            .. list-table::
               :header-rows: 1
               :widths: 20 80

               * - Section
                 - Description

               * - Usage Summary
                 - Overview of your Automated Embedding usage, including total
                   tokens used and remaining free tokens.

               * - Usage by Model
                 - Breakdown of token usage by embedding model.

               * - Usage by Operation
                 - Breakdown of token usage by operation type (indexing and
                   querying).

         .. step:: Filter the usage data by date range and model.

   .. tab:: Self-Managed
      :tabid: self

      .. procedure:: 
         :style: normal

         .. step:: Log in to `{+service+} <https://cloud.mongodb.com>`__.

         .. step:: Go to the Usage page in the Atlas UI.

            a. If it's not already displayed, select your desired organization
               from the :icon-mms:`office` :guilabel:`Organizations` menu in
               the navigation bar.
               
            #. At the organization level, click :guilabel:`Usage` under the 
               :guilabel:`AI Models` header in the navigation bar.

         .. step:: Filter the usage data by date range and model.

         .. step:: View the token usage and embedding model requests for Automated Embedding.

            The :guilabel:`Usage` page displays usage metrics for all model API keys in your 
            organization. The page provides the following tabs:

            - :guilabel:`Usage Activity`: Shows detailed usage metrics and charts for your organization.

            - :guilabel:`Free Usage`: Shows free quota consumption by model.

            To learn more, see :ref:`Usage Page Features <voyage-usage-page-features>`.

.. _auto-embedding-view-rate-limits:

View Rate Limits 
----------------

To view rate limits for Automated Embedding:

.. tabs:: 

   .. tab:: Atlas
      :tabid: atlas

      .. procedure:: 
         :style: normal

         .. step:: Log in to `{+service+} <https://cloud.mongodb.com>`__.

         .. step:: Go to the :guilabel:`Search & Vector Search` page for your project.
               
            a. If it's not already displayed, select your desired organization
               from the :icon-mms:`office` :guilabel:`Organizations` menu in
               the navigation bar.

            #. If it's not already displayed, select your desired project
               from the :guilabel:`Projects` menu in the navigation bar.

            #. In the sidebar, click :guilabel:`Search & Vector Search` under the 
               :guilabel:`Database` header.

               The `Search & Vector Search <https://cloud.mongodb.com/go?l=https%3A%2F%2Fcloud.mongodb.com%2Fv2%2F%3Cproject%3E%23%2FautoEmbedding%2F>`__ 
               page displays.

         .. step:: Click the :guilabel:`Automated Embedding` section if it's not already expanded.

         .. step:: Click :guilabel:`Rate Limits` under :guilabel:`Automated Embedding`.

            The page displays the following information:

            .. list-table::
               :header-rows: 1 
               :widths: 30 70 

               * - Name 
                 - Description 
               
               * - :guilabel:`Model`
                 - List of {+voyageai+} embedding models.

               * - :guilabel:`Tokens Per Minute (TPM)`
                 - Number of tokens that you can process per minute.

               * - :guilabel:`Requests Per Min (RPM)`
                 - Number of API requests that you can send per minute.

   .. tab:: Self-Managed
      :tabid: self

      .. procedure:: 
         :style: normal

         .. step:: Log in to `{+service+} <https://cloud.mongodb.com>`__.

         .. step:: Go to the Usage page in the Atlas UI.


            a. If it's not already displayed, select your desired organization
               from the :icon-mms:`office` :guilabel:`Organizations` menu in
               the navigation bar.
               
            #. At the organization level, click :guilabel:`Rate Limits` under the 
               :guilabel:`AI Models` header in the navigation bar.

         .. step:: View the rate limits for Automated Embedding.

            The page displays the following information:

            .. list-table::
               :header-rows: 1 
               :widths: 30 70 

               * - Name 
                 - Description 
               
               * - :guilabel:`Model`
                 - List of {+voyageai+} embedding models.

               * - :guilabel:`Tokens Per Minute (TPM)`
                 - Number of tokens that you can process per minute.

               * - :guilabel:`Requests Per Min (RPM)`
                 - Number of API requests that you can send per minute.

.. _auto-embedding-increase-rate-limits:

Increase Rate Limits
--------------------

If you are on the Free Tier and need to increase rate limits for Automated 
Embedding, add a payment method to upgrade to a paid tier. To learn more, 
see :ref:`atlas-payment-method`. 

If you are on a Paid Tier, MongoDB automatically increases rate limits as 
your usage increases. If you still need a higher rate limit, contact 
`MongoDB Support <https://support.mongodb.com/>`__.
