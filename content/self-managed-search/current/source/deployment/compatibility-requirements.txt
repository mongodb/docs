=============================================
Compatibility and Requirements for ``mongot``
=============================================

.. meta::
   :description: Learn which operating systems, architectures, and deployment patterns are supported for self-managed {+fts+}, including cross-product compatibility with {+om+} and Kubernetes.

.. facet::
   :name: genre

This page describes version compatibility, minimum component
requirements, operating system support, and deployment compatibilities
for self-managed {+fts+} and {+avs+}.

.. note::

   Unless otherwise noted, Community Edition statements apply to
   ``mongot`` 1.70.1 and later.

How to Use This Page
--------------------

Use the following sections to verify your deployment target and
configuration:

- To verify minimum versions and component requirements for
  {+k8s-op+}-managed deployments, see
  :ref:`search-vector-search-compatibility-requirements`.
- To verify MongoDB Server and MongoDB Search version pairings,
  see :ref:`server-search-version-compatibility`.
- To verify MongoDB Search and {+k8s-op+} version pairings, see
  :ref:`search-mck-version-compatibility`.
- To verify which Community Edition packages are supported on
  which Linux platforms, see :ref:`search-package-os-support`.
- To rule out architectures and operating systems that are not
  valid deployment targets, see
  :ref:`unsupported-platforms-envs`.
- To determine whether a MongoDB Search deployment can be combined
  with MongoDB Server, {+k8s-op+}, {+om+}, and related components
  for your intended topology, see :ref:`compatibility-tables`.

.. _search-vector-search-compatibility-requirements:

Search and Vector Search Compatibility and Requirements
-------------------------------------------------------

This section describes the version compatibility and minimum
component requirements for running {+fts+} and {+avs+} with
the {+k8s-op+}.

Minimum Required Versions
~~~~~~~~~~~~~~~~~~~~~~~~~

To deploy {+fts+} and {+avs+} with the {+k8s-op+}, run the
following component versions or later:

.. list-table::
   :header-rows: 1
   :widths: 60 40

   * - Component
     - Minimum version
   * - MongoDB Server
     - 8.3.4
   * - ``mongot``
     - 1.70.1
   * - {+k8s-op+}
     - 1.9.1
   * - {+om+} or Cloud Manager
     - 8.0.25

You can run newer versions of any component if the combinations
remain within the supported pairings described in the following
sections.

Compatibility
~~~~~~~~~~~~~

``mongot`` and ``mongod``
^^^^^^^^^^^^^^^^^^^^^^^^^

For {+k8s-op+}-managed deployments, any supported ``mongot``
version is compatible with any supported ``mongod`` version of
MongoDB Server 8.3.4 or later.

The following compatibility behaviors apply:

- ``mongot`` 1.70.1 works with ``mongod`` 8.3.x.
- Features added in newer ``mongot`` might require a newer
  ``mongod`` to function.
- Existing features remain consistent when you upgrade``mongot``
  without upgrading ``mongod`` or when you upgrade ``mongod`` without upgrading
  ``mongot``. 

``mongot`` and the {+k8s-op-short+}
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The {+k8s-op-short+} declares the ``mongot`` versions it
supports. Use the {+k8s-op-short+} version that ships alongside
or after the ``mongot`` version you deploy. The
compatibility table later on this page starts with ``mongot``
1.70.1 and {+k8s-op+} 1.9.1, which are the minimum documented
versions for {+k8s-op+}-managed {+fts+} and {+avs+}
deployments. To learn more, see
`MongoDB Controllers for Kubernetes Operator Compatibility
<https://www.mongodb.com/docs/kubernetes/upcoming/tutorial/plan-k8s-op-compatibility#std-label-k8s-compatibility>`__.

Supported Platforms and Editions
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

{+fts+} and {+avs+} on {+k8s+} inherit the platform
support of the {+k8s-op+}. Confirm your cluster meets the
{+k8s-op+} requirements before you deploy ``mongot``.

- For supported MongoDB Server versions and base images, see
  `Supported Platforms and MongoDB Versions
  <https://www.mongodb.com/docs/kubernetes/upcoming/tutorial/plan-k8s-op-compatibility#std-label-k8s-mdb-version-compatibility>`__.
- For supported hardware architectures, see
  `Supported Hardware Architectures
  <https://www.mongodb.com/docs/kubernetes/upcoming/tutorial/plan-k8s-op-compatibility#std-label-k8s-supported-hardware-arch-compatibility>`__.
- For supported {+om+} versions, see
  `Cloud Manager and Ops Manager Versions
  <https://www.mongodb.com/docs/kubernetes/upcoming/tutorial/plan-k8s-op-compatibility#std-label-k8s-onprem-version-compatibility>`__.

You can run {+fts+} and {+avs+} against either edition of
MongoDB Server that the {+k8s-op-short+} manages:

- **Community Edition:** Manage the source ``mongod`` through
  the ``MongoDBCommunity`` resource. {+om+} and Cloud Manager
  do not apply in this configuration.
- **Enterprise Advanced:** Manage the source ``mongod`` through
  the ``MongoDB`` resource, paired with {+om+} or Cloud Manager.

Both editions use the same ``MongoDBSearch`` resource to deploy
``mongot``. For a deployment external to the {+k8s-op-short+},
reference it through ``spec.source.external`` on the
``MongoDBSearch`` resource. To learn more, see
`MongoDB Search and Vector Search Settings
<https://www.mongodb.com/docs/kubernetes/upcoming/reference/fts-vs-settings#std-label-k8s-fts-vs-settings>`__.

.. _unsupported-platforms-envs:

Unsupported Platforms and Environments
--------------------------------------

The following platforms and environments are not supported:

.. list-table::
   :header-rows: 1
   :widths: 50 50

   * - Platform or environment
     - Notes
   * - IBM Power (ppc64le)
     - ``mongot`` is built for x86_64 and aarch64 Linux only.
   * - IBM Z (s390x)
     - ``mongot`` is built for x86_64 and aarch64 Linux only.
   * - Native Windows deployment target
     - Native Windows ``mongot`` binaries are not available.
   * - Native macOS deployment target
     - Native macOS ``mongot`` binaries are not available.
   * - Windows Subsystem for Linux
     - Support assumes a native Linux deployment target.
   * - ``atlas-local`` Docker image in production
     - ``atlas-local`` is a single-container development image,
       not a production deployment target.

For local development on macOS or Windows, use a Docker-based
Linux runtime such as ``atlas-local`` on Docker Desktop rather
than native ``mongot`` binaries.

Compatibility Notes
-------------------

Load Balancing for Multi-mongot Topologies
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

If a replica set or shard uses more than one ``mongot`` instance,
use an L7 load balancer. An L4 load balancer is not sufficient
because ``mongod`` opens a long-lived connection to ``mongot``
and an L4 balancer cannot distribute traffic at the gRPC stream
level.

Sharded Deployments
~~~~~~~~~~~~~~~~~~~

For sharded deployments, each shard has its own ``mongot`` group. Shards do not
share ``mongot`` instances. In Kubernetes-based deployments, {+k8s-op+} can
automate this topology. If you manually set up deployments, you must create and
manage the per-shard topology.

Enterprise Production
~~~~~~~~~~~~~~~~~~~~~

For Enterprise Advanced, the intended production posture is MongoDB managed on
VMs, bare metal, or Kubernetes, with {+k8s-op+} managing the ``mongot``.

.. _mongot-om-monitoring-compatibility:

Ops Manager
~~~~~~~~~~~

{+om+} can manage the MongoDB deployment that {+fts+} uses and can
surface curated {+fts+} visibility, but it is not the management
plane for {+fts+} itself. To receive ``mongot`` metrics through
{+om+} monitoring, run {+om+} 8.0.25 or later.

Docker
~~~~~~

The following list details considerations for Docker deployments:

- ``atlas-local`` is for local development and continuous
  integration only. It does not pair with an external MongoDB
  deployment.
- Linux containerized {+fts+} Community Edition is a supported
  packaging and runtime mechanism on supported Linux platforms.
- Enterprise production support assumes {+k8s-op+}-managed
  {+fts+} for Enterprise Advanced rather than standalone
  container lifecycle management.

Packaging
~~~~~~~~~

Tarball and container installs are the packaging paths for
{+fts+} Community Edition. Native package-manager installation
with DEB or RPM is not yet available.
