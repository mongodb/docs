.. _mcli-atlas-quick-start-cmd:

=========================
mongocli atlas quickstart
=========================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``mongocli atlas quickstart`` command creates an ``M2`` cluster in 
the specified |service| project. You can either specify the options 
with the command or omit the options and run the command in interactive 
mode. That is, if you do not specify one or more required options with 
the command, the command prompts you for the settings for those 
options. If you do not yet have a default profile, the command provides 
the links to: 

- :atlas:`Create an Atlas account </tutorial/create-atlas-account/>`
- :ref:`Configure a default profile <mcli-configure>`

Use this command to get started with MongoDB :atlas:`Atlas </>`.

.. note:: 

   The command takes a few minutes to create the cluster; you can press 
   :kbd:`Ctrl+C` to return to the command line while the operation 
   completes in the background.

.. _mcli-atlas-quick-start-syntax:

Syntax 
------

.. code-block:: sh 

   mongocli atlas quickstart 
        --accessListIp <IP-addresses> 
        --clusterName <cluster-name>
        --password <user-password> 
        --profile|-P <profile-name> 
        --projectId <project-ID> 
        --provider AWS|GCP|AZURE
        --region|-r <region-name>
        --skipMongosh 
        --username <name-of-user>

.. _mcli-atlas-quick-start-options:

Options
-------

All of the following options are required to run the command in 
non-interactive mode. To run the command in interactive mode, you don't 
need to specify any of the following options with the command.

.. list-table::
   :header-rows: 1
   :widths: 20 10 70

   * - Option
     - Type
     - Description

   * - ``--accessListIp``
     - string 
     - List of IP addresses to add to the project access list. 
       |service| only allows connections to a cluster from addresses 
       listed in the IP access list. If you do not specify an IP 
       address, {+mcli+} detects and adds your public IP address to 
       the project access list. 

   * - ``--clusterName``
     - string
     - Name of the cluster. The cluster name cannot be changed after 
       you create the cluster. Cluster name can contain ASCII letters, 
       numbers, and hyphen. If you do not specify a name for the 
       cluster, defaults to ``Quickstart_{epoch}`` where ``{epoch}`` is 
       the date range represented as milliseconds elapsed since the 
       UNIX epoch.

   * - ``--password``
     - string 
     - Password for authenticating the user to MongoDB using 
       ``SCRAM-SHA``. In interactive mode, {+mcli+} auto-generates a 
       password in interactive mode if you do not specify a password.

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private keys for the 
       project are set. If omitted, uses the {+default-profile+}. To 
       learn more about creating a profile, see :ref:`mcli-configure`.

   * - ``--projectId``
     - string
     - .. include:: /includes/extracts/fact-basic-options-project-id.rst

   * - ``--provider``
     - string
     - Cloud service provider on which the servers are provisioned. 
       Value can be:

       - ``AWS`` - Amazon AWS
       - ``GCP`` - Google Cloud Platform
       - ``AZURE`` - Microsoft Azure

       If you do not specify the provider, defaults to ``AWS``.

   * - ``--region``, ``-r``
     - string
     - Physical location of the MongoDB cluster. Select your cloud 
       provider tab to view the available regions:

       .. tabs-cloud-providers::

          .. tab::
             :tabid: aws

             - ``US_EAST_1`` 
             - ``US_WEST_2``
             - ``EU_WEST_1``

             For a complete list of supported |aws| regions, see
             :atlas:`Amazon Web Services (AWS)
             </reference/amazon-aws/#amazon-aws>`. If you don't specify 
             a region, defaults to ``US_EAST_1``.

          .. tab::
             :tabid: gcp

             - ``CENTRAL_US``
             - ``WESTERN_EUROPE``
             - ``EASTERN_ASIA_PACIFIC``

             For a complete list of supported |gcp| regions, see
             :atlas:`Google Cloud Platform (GCP)
             </reference/google-gcp/#google-gcp>`. If you don't 
             specify a region, defaults to ``CENTRAL_US``.

          .. tab::
             :tabid: azure

             - ``US_EAST_2``
             - ``US_WEST``
             - ``EUROPE_NORTH``

             For a complete list of supported Azure regions, see
             :atlas:`Microsoft Azure 
             </reference/microsoft-azure/#microsoft-azure>`. If you 
             don't specify a region, defaults to ``US_EAST_2``.

   * - ``--skipMongosh`` 
     - boolean 
     - Flag that specifies whether or not to connect to the cluster 
       using the |mongo| shell. In interactive mode, you can specify 
       one of the following values: 

       - ``Y`` - to connect to the cluster using the |mongo| shell.
       - ``N`` - to not connect to the cluster using the |mongo| shell.
       
       If you do not specify a value, defaults to ``N``.

   * - ``--skipSampleData``
     - boolean 
     - Flag that indicates whether or not to load :atlas:`sample data 
       </sample-data/available-sample-datasets/>` on your |service| 
       cluster. In interactive mode, you can specify one of the 
       following values:

       - ``Y`` - to load sample data.
       - ``N`` - to not load the sample data.

       If you do not specify a value, defaults to ``N``.

   * - ``--username``
     - string
     - Username for authenticating the user to MongoDB. In interactive 
       mode, the default value is the same as the default cluster name.

.. _mcli-atlas-quick-start-output:

Output 
------

If the command succeeds, the command prints the following output to the 
terminal. 

.. code-block:: sh
   :copyable: false

   Now you can connect to your Atlas cluster with: mongosh -u <username> -p <user-password> mongodb+srv://<cluster-name>.7hgjn.mongodb.net

The command prints output similar to the following to the terminal if 
you run the command in the background by pressing :kbd:`Ctrl+C`.

.. code-block:: sh 
   :copyable: false 

   You can connect to your Atlas cluster with the following user: 
   username: <username> 
   password: <user-password>

If the command returns errors, see :ref:`Troubleshooting 
<troubleshooting>` for recommended solutions.

.. _mcli-atlas-quick-start-egs:

Examples 
--------

The following ``mongocli atlas quickstart`` commands create a cluster 
named ``QuickstartCluster`` in the |service| project. The command uses 
the {+default-profile+} for accessing the |service| project.

.. tabs:: 

   .. tab:: Non-Interactive Mode 
      :tabid: cli 

      .. code-block:: shell 

         mongocli atlas quickstart --clusterName QuickstartCluster --provider AWS --region US_EAST_1 --username testUser --password changeMe --accessListIp 192.0.2.15 --skipMongosh

   .. tab:: Interactive Mode 
      :tabid: interactive

      .. code-block:: shell 

         mongocli atlas quickstart 
         [Set up your Atlas cluster]
         Cluster Name [Press Enter to use the auto-generated name 'Quickstart-1616793812']: Quickstart-1616793812
         Cloud Provider: AWS
         Cloud Provider Region: US_EAST_1
         Do you want to load sample data into Quickstart-1619105361? No
         We are deploying your cluster...

        [Set up your database access details]
        Database User Username [Press Enter to use 'Quickstart-1616794178']: testUser
        Database User Password [Press Enter to use an auto-generated password 'YHB2THtJVE5a']: changeMe

        [Set up your network access list details]
        Access List Entry [Press Enter to use your public IP address '192.0.2.15']: 192.0.2.15

        [Connect to your new cluster]
        Do you want to access Quickstart-1616793812 with MongoDB Shell? No

        Creating your cluster... [It's safe to 'Ctrl + C']

The command prints the following output to the terminal after it 
completes: 

.. code-block:: sh 
   :copyable: false 

   Now you can connect to your Atlas cluster with: mongosh -u testUser -p changeMe mongodb+srv://quickstart-1616794174.7hgjn.mongodb.net

The command prints the following to the terminal if you press 
:kbd:`Ctrl+C`: 

.. code-block:: sh 
   :copyable: false 

   You can connect to your Atlas cluster with the following user: 
   username: testUser 
   password: changeMe
