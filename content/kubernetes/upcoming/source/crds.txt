.. _k8s-crds:

==========================
Custom Resource Definitions
==========================

.. meta::
   :description: Overview of the custom resource definitions (CRDs) available for the MongoDB Controllers for Kubernetes.

.. default-domain:: mongodb

|k8s-op-short| extends the |k8s| API with custom resource definitions
(CRDs) that allow you to define and manage MongoDB components
declaratively using YAML manifests. The following CRDs are available:

- ``MongoDB``: Deploys standalone, replica set, and sharded cluster
  database resources. To learn more, see :ref:`k8s-specification`.

- ``MongoDBOpsManager``: Deploys and configures Ops Manager and its
  backing Application Database. To learn more, see
  :ref:`k8s-om-specification`.

- ``MongoDBUser``: Defines database users and their authentication
  settings and roles. To learn more, see
  :ref:`mongodbuser-specification`.

- ``MongoDBMultiCluster``: Deploys MongoDB resources across multiple
  |k8s| clusters. To learn more, see
  :ref:`multi-cluster-specification`.

- ``MongoDBSearch``: Configures Atlas Search and Vector Search nodes
  for full-text and semantic search workloads. To learn more, see
  :ref:`k8s-fts-vs-settings`.

- ``ClusterMongoDBRole``: Defines reusable, cluster-scoped MongoDB
  roles that can be referenced by ``MongoDBUser`` resources. To learn
  more, see :ref:`mongodbrole-specification`.
