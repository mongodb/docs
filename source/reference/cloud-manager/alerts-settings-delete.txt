.. _mcli-cm-alerts-settings-delete-cmd:

=============================================
mongocli cloud-manager alerts settings delete
=============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``alerts settings delete`` command deletes the specified alert
configuration in a |cloud-short| project. You must confirm the operation
when you run this command without the ``--force`` option.

You can also delete a alert configuration using the |cloud-short| 
:cloudmgr:`UI
</tutorial/manage-alert-configurations/index.html#delete-an-alert-configuration>`
or :cloudmgr:`API </reference/api/alert-configurations-delete-config/>`.

.. _cm-alerts-settings-delete-command-syntax:

Syntax
------

.. code-block:: text
   :copyable: false

   mongocli cloud-manager|cm alerts settings|config delete|rm|Delete|Remove <alert-id>
        [ --force ]
        [ --output|-o <output-format> ]
        [ --profile|-p <profile-name> ]
        [ --projectId <project-ID> ]

.. _cm-alerts-settings-delete-command-options:

Options 
-------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``<alert-id>``
     - string
     - ID of the alert configuration to delete.
     - yes

   * - ``--force`` 
     - 
     - Flag that indicates that the alert configuration can 
       be deleted without requiring confirmation.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-p``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project containing the 
       alert configuration to delete. If omitted, uses the project ID in
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no


.. _cm-alerts-settings-delete-command-output:

Output 
------

The command prints a confirmation message similar to the 
following to the terminal if it succeeds. If the command 
prints an error, see :ref:`Troubleshooting <troubleshooting>` 
for recommended solutions.

.. code-block:: text
   :copyable: false

   Alert config '<alert-id>' deleted

If you choose not to delete the alert configuration, the command prints
a message similar to the following:

.. code-block:: text
   :copyable: false

   Alert config not deleted

.. _cm-alerts-settings-delete-command-examples:

Examples 
--------

Basic Example
~~~~~~~~~~~~~

The following ``alerts settings delete`` command deletes 
an alert configuration with the ID ``5ec4518ce4a7ba70e03dbf13`` after
prompting for a confirmation. The command uses the {+default-profile+}.

.. code-block:: text
   :copyable: false

   mongocli cm alerts settings delete 5ec4518ce4a7ba70e03dbf13 

The command prints the following to the terminal:

.. code-block:: text
   :copyable: false

   ? Are you sure you want to delete: 5ec4518ce4a7ba70e03dbf13 Yes
   Alert config '5ec4518ce4a7ba70e03dbf13' deleted

Force Example
~~~~~~~~~~~~~

The following ``alerts settings delete`` command deletes 
an alert configuration with the ID ``5ec4518ce4a7ba70e03dbf13`` without 
requiring confirmation. The command uses the {+default-profile+}.

.. code-block:: text
   :copyable: false

   mongocli cm alerts configs delete 5ec4518ce4a7ba70e03dbf13 --force

The command prints the following to the terminal:

.. code-block:: text
   :copyable: false

   Alert config '5ec4518ce4a7ba70e03dbf13' deleted
