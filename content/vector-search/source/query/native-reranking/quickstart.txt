=================================
Get Started With Native Reranking
=================================

.. important::

   Native Reranking is available as a Preview feature. The feature
   and the corresponding documentation might change at any time
   during the Preview period. Therefore, we don't recommend using
   this feature in production environments. We do not use any
   customer data from this feature to train our models at this time.
   To learn more, see `Preview Features
   <https://www.mongodb.com/docs/preview-features/>`__.

The :pipeline:`$rerank` aggregation stage reorders input documents
using :ref:`Voyage AI reranking models <voyage-rerankers>`
and returns the same documents sorted by relevance to the query. The 
``$rerank`` stage can appear anywhere in an aggregation pipeline. 

You don't need to create an index to run :pipeline:`$rerank` queries. 
However, MongoDB recommends using :pipeline:`$rerank` after a 
:pipeline:`$vectorSearch`, :pipeline:`$search`, :pipeline:`$rankFusion`, 
or :pipeline:`$scoreFusion` stage, which require creating a :ref:`{+avs+} 
<avs-types-vector-search>` or :ref:`{+fts+} <ref-create-index>` index on 
your collection.

This quickstart demonstrates how to enable and use Native Reranking in 
|service| by using the :pipeline:`$rerank` aggregation stage on queries 
against the :ref:`sample_mflix <sample-mflix>` dataset. 

.. _rerank-prereqs:

Prerequisites
-------------

Before you begin, complete the following tasks: 

- Deploy a |service| cluster with MongoDB 8.3 or later by selecting 
  :guilabel:`Latest Version with Auto-Upgrades` in the {+atlas-ui+} 
  :guilabel:`Cluster Builder` page. 

  To learn more about an |service| cluster, see :ref:`Create a Cluster 
  <create-new-cluster>`. Native reranking is not available for self-managed 
  deployments.

- Load the :ref:`sample_mflix <sample-mflix>` dataset into your cluster.

  To learn more, see :ref:`Load Sample Data <load-sample-data>`.

- Download and install {+mongosh+}.

  To learn more, see :mongosh:`Install the MongoDB Shell </install>`.

To learn more about the compatibility and requirements for using 
``$rerank``, see :ref:`rerank-considerations`.

---------

         :guilabel:`Enable Native Reranking` to :guilabel:`On`.

      #. Click :guilabel:`Confirm`.

      ten movies that are most relevant to the query for *martial arts*.

   .. step:: Create the indexes for running Native Reranking queries with ``$vectorSearch`` and ``$search``.

      MongoDB recommends using :pipeline:`$rerank` after a 
      :pipeline:`$vectorSearch`, :pipeline:`$search`, :pipeline:`$rankFusion`, 
      or :pipeline:`$scoreFusion` stage. To do so, create a {+avs+} and {+fts+} 
      index on your collection to run the :pipeline:`$vectorSearch` and 
      :pipeline:`$search` stages. 

      To try the sample queries that use the :pipeline:`$vectorSearch` and 
      :pipeline:`$search` stages in this quickstart, you must create a {+avs+} 
      and {+fts+} index on your collection.

      .. collapsible:: 
         :heading: Create a MongoDB Vector Search index
         :sub_heading: Create a MongoDB Vector Search index on the plot_embedding_voyage_4_large field.
         :expanded: false

         To create the index, complete the following steps:

         a. Click :guilabel:`Search & Vector Search` in the sidebar.

         #. Click :guilabel:`Create Index`.

         #. Start your index configuration: 
          
            - Select :guilabel:`Vector Search` as the index type.
            - Enter ``vector_index`` as the name of the index.
            - Select the ``sample_mflix.embedded_movies`` collection.
            - Select :guilabel:`JSON Editor` as the editor type.

         #. Click :guilabel:`Next` and paste the following index definition 
            in the JSON editor:

         #. Click :guilabel:`Next`, then click :guilabel:`Create Index`.

      .. collapsible::

         :expanded: false
