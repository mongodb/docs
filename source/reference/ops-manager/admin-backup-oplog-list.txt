.. _mcli-om-admin-backup-oplog-list-cmd:

============================================
mongocli ops-manager admin backup oplog list
============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/admonitions/admin-requires-global-owner.rst

The ``admin backup oplog list`` command retrieves the 
configurations of all oplog stores. You can also 
retrieve the configurations of all oplog stores using 
the :opsmgr:`API </reference/api/admin/backup/oplog/mongoConfigs/get-all-oplog-configurations/>`.

.. _mcli-om-admin-backup-oplog-list-syntax: 

Syntax 
------

.. code-block:: shell 

   mongocli ops-manager|om admin backup oplog(s) list|ls 
        [ --limit <items-per-page> ]
        [ --page <page-number> ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. include:: /includes/fact-command-line-help.rst

.. _mcli-om-admin-backup-oplog-list-options: 

Options 
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--limit``
     - integer
     - Number of items to return per page, up to a maximum of 500.
       Defaults to ``100`` if omitted.
     - no

   * - ``--page``
     - integer
     - Page number (1-based). Defaults to ``1`` if not specified.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for accessing |onprem| are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

.. _mcli-om-admin-backup-oplog-list-output: 

Output 
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh 
   :copyable: false 

   NAME         URI              SSL              LOAD FACTOR      MAX CAPACITY GB
   <oplog-ID>   <store-path>     <true|false>     <load-factor>    <capacity>

The default output contains a subset of the fields returned by this 
command. For the complete list of |json| fields returned by the 
command, see the :opsmgr:`API reference 
</reference/api/admin/backup/oplog/mongoConfigs/get-all-oplog-configurations/#response>`.

.. _mcli-om-admin-backup-oplog-list-egs: 

Example 
-------

The following ``mongocli om admin backup oplog list`` command 
retrieves the configurations of all oplog stores using the 
{+default-profile+}, which contains |onprem| credentials. 

.. tabs:: 

   .. tab:: Default Output 
      :tabid: default 

      .. code-block:: shell 

         mongocli om admin backup oplog list

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         NAME     URI                         SSL     LOAD FACTOR   MAX CAPACITY GB
         oplog1   mongodb://localhost:27017   false   1             <nil>

   .. tab:: JSON Output 
      :tabid: json 

      .. code-block:: shell 

         mongocli om admin backup oplog list --output json

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         {
           "links": [
             {
               "rel": "self",
               "href": "http://example.com:8080/api/public/v1.0/admin/backup/oplog/mongoConfigs?pageNum=1\u0026itemsPerPage=100"
             }
           ],
           "results": [
             {
               "id": "oplog1",
               "uri": "mongodb://localhost:27017",
               "ssl": false,
               "assignmentEnabled": true,
               "encryptedCredentials": false,
               "loadFactor": 1
             }
           ],
           "totalCount": 1
         }
