.. _mcli-cm-cluster-startup-command:

==================================
mcli cloud-manager cluster startup
==================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster startup`` command starts the replica set
:manual:`mongod </reference/program/mongod/#bin.mongod>` 
processes for the specified cluster in the specified 
|cloud-short| project. The command prompts you to confirm the 
operation when you run the command without the 
``--force`` option. You can also restart the :manual:`mongod 
</reference/program/mongod/#bin.mongod>` processes from the 
|cloud-short| :cloudmgr:`UI 
<tutorial/restart-deployment/>`.

.. _cm-cluster-startup-syntax:

Syntax 
------

.. code-block:: text 

   mcli cm|cloud-manager|cm cluster startup <cluster-name> 
        [ --force ]
        [ -p|--profile <profile-name> ] 
        [ --projectId <project-ID> ]

.. _cm-cluster-startup-options:

Options 
------- 

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``<cluster-name>``
     - string
     - The name of the cluster.
     - yes

   * - ``--force``
     - 
     - The flag that indicates that the replica set :manual:`mongod 
       </reference/program/mongod/#bin.mongod>` processes can be 
       started without prompting for confirmation.
     - no

   * - ``-p``, ``--profile``
     - string
     - The name of the profile where the public and private 
       keys for the project are set. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`configure-mcli`.
     - no

   * - ``--projectId``
     - string
     - The unique identifier of the project that contains the 
       cluster to startup. If omitted, uses the project ID in 
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no

.. _cm-cluster-startup-output:

Output 
------

The command prints output similar to the following to the terminal if 
the command succeeds. If the command prints errors, see 
:ref:`Troubleshooting <troubleshooting>` for recommended solutions.

.. code-block:: text 
   :copyable: false 

   Changes are being applied, please check https://cloud.mongodb.com/v2/<projectID>#deployment/topology for status


where ``projectID`` is the unique identifier of the project.

.. _cm-cluster-startup-examples: 

Examples 
--------

The following examples show the different ways to run the command.

.. _cm-cluster-startup-example1: 

Example 1
~~~~~~~~~

The following ``mcli cm cluster startup`` command 
prompts for confirmation before starting the the replica set 
``mongod`` processes for the cluster named ``CMsbx`` in the specified 
project. It uses the profile named ``egCMprofile`` for accessing the 
specified project.

.. code-block:: text 
   :copyable: false
   
   ~ mcli cm cluster startup CMsbx --projectId 5e2f06127a3e5a5700de8d8d --profile egCMprofile
   Are you sure you want to startup: CMsbx Yes

.. _cm-cluster-startup-example2:

Example 2
~~~~~~~~~

The following ``mcli cloud-manager cluster startup`` command 
*does not* prompt for confirmation before starting the replica set 
``mongod`` processes for the cluster named ``CMsbx`` in the specified project. 
It uses the ``default`` profile for accessing the specified project.

.. code-block:: text 
   :copyable: false
   
   ~ mcli cloud-manager cluster startup CMsbx --projectId 5e2f06127a3e5a5700de8d8d --force 

Output
~~~~~~

The :ref:`cm-cluster-startup-example1` and :ref:`cm-cluster-startup-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false 

   Changes are being applied, please check https://cloud.mongodb.com/v2/5e2f06127a3e5a5700de8d8d#deployment/topology for status
