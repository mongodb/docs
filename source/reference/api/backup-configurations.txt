=====================
Backup Configurations
=====================

.. default-domain:: mongodb

.. include:: /includes/fact-groups-projects-synonyms.rst

Overview
--------

A backup configuration determines the settings used to back up a sharded
cluster or replica set. The ``backupConfigs`` resource lets you view and
update backup configurations. In certain cases, you can also create backup
configurations, as described in :doc:`/reference/api/backup/update-backup-config`. 
The ``backupConfigs`` resource supports only the ``GET`` and ``PATCH``
methods.

To update or create a backup configuration, you must issue the request from an
IP address on your user account's :ref:`whitelist
<whitelist-for-api-operations>`. The ``backupConfigs`` resource accepts
``PATCH`` requests only from whitelisted IP addresses.

.. include:: /includes/api/base-api-uri.rst

Endpoints
---------

The following lists the endpoints available for :doc:`Backup Configuration
</tutorial/enable-backup>`.

.. list-table::
   :header-rows: 1
   :widths: 10 50 40

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/api/public/v1.0/groups/{GROUP-ID}/backupConfigs </reference/api/backup/get-all-backup-configs-for-group>`

     - Get all backup configurations for a group.

   * - ``GET``

     - :doc:`/api/public/v1.0/groups/{GROUP-ID}/backupConfigs/{CLUSTER-ID} </reference/api/backup/get-one-backup-config-by-cluster-id>`

     - Get a single backup configuration by cluster ID.

   * - ``PATCH``

     - :doc:`/api/public/v1.0/groups/{GROUP-ID}/backupConfigs/{CLUSTER-ID} </reference/api/backup/update-backup-config>`

     - Update a backup configuration.
