Use the following resource to retrieve and update your
:ref:`API whitelist <whitelist-for-api-operations>`.

Users with the :authrole:`Project Owner` role in the project perform
whitelisted |api| operations from the IP addresses you have added to
this list.

|service| whitelists this resource's ``POST`` and ``DELETE``
operations. These endpoints must originate from an address already on
your |api| whitelist.

You can access the |api| whitelist only for the current user. The
username and |api| key you use to authenticate must be those of the
user identified in the endpoint. The endpoint identifies the user by
user ID. To retrieve user IDs, see :ref:`api-get-all-users-in-group`.

Endpoints
---------

.. list-table::
   :header-rows: 1
   :widths: 10 40 50

   * - Method
     - Endpoint
     - Description

   * - ``GET``
     - :doc:`/users/{USER-ID}/whitelist </reference/api/api-whitelist/personal/get-whitelist-for-current-user>`
     - Get the API Whitelist for the Current User

   * - ``GET``
     - :doc:`/users/{USER-ID}/whitelist/({IP-ADDRESS} | {CIDR-BLOCK}) </reference/api/api-whitelist/personal/get-whitelist-for-ip-or-cidr>`
     - Get the API Whitelist Entry for a Specified Address or CIDR Block

   * - ``POST``
     - :doc:`/users/{USER-ID}/whitelist </reference/api/api-whitelist/personal/add-entry-to-whitelist>`
     - Add Entries to the User's API Whitelist

   * - ``DELETE``
     - :doc:`/users/{USER-ID}/whitelist/({IP-ADDRESS} | {CIDR-BLOCK}) </reference/api/api-whitelist/personal/remove-entry-from-whitelist>`
     - Delete an Entry from the User's API Whitelist
