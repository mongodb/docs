====================
Project IP Whitelist
====================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: twocols

.. include:: /includes/fact-projects-groups-synonyms.rst

The ``whitelist`` endpoint manages a |service| project's
:ref:`IP whitelist <access-list>`.

The ``whitelist`` endpoint supports creating temporary whitelist
entries that automatically expire within a user-configurable 7-day
period.

   .. include:: /includes/fact-project-user-whitelist.rst

.. include:: /includes/api-base-url-new.rst

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/whitelist </reference/api/whitelist-get-all>`

     - Get all whitelist entries in the project associated to
       ``{GROUP-ID}``.

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/whitelist/{WHITELIST-ENTRY} </reference/api/whitelist-get-one-entry>`

     - Get the whitelist entry specified to ``{WHITELIST-ENTRY}`` from
       the project associated to ``{GROUP-ID}``.

   * - ``POST``

     - :doc:`/groups/{GROUP-ID}/whitelist </reference/api/whitelist-add-one>`

     - Add one or more whitelist entries to the project
       associated to ``{GROUP-ID}``.

   * - ``DELETE``

     - :doc:`/groups/{GROUP-ID}/whitelist/{WHITELIST-ENTRY} </reference/api/whitelist-delete-one>`

     - Delete the whitelist entry specified to ``{WHITELIST-ENTRY}``
       from the project associated to ``{GROUP-ID}``.
