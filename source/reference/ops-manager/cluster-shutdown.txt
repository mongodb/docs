.. _mcli-om-cluster-shutdown-command:

=====================================
mongocli ops-manager cluster shutdown
=====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster shutdown`` command shuts down the replica set 
:manual:`mongod </reference/program/mongod/#bin.mongod>` processes 
for the specified cluster in the specified |mms| project. The command 
prompts you to confirm the operation when you run the command 
without the ``--force`` option. You can also shut down the 
:manual:`mongod </reference/program/mongod/#bin.mongod>` processes 
from the |mms| :opsmgr:`UI </tutorial/shut-down-deployment/>`.

.. note:: 

   When you shutdown a process, replica set, or cluster, |mms| 
   continues to manage it, even though it is not running. Your data 
   is not removed and you can :ref:`restart your process, cluster, or 
   replica set <mcli-cm-cluster-startup-command>` and access your data.

.. _om-cluster-shutdown-syntax:

Syntax 
------

.. code-block:: text 

   mongocli ops-manager|om cluster shutdown <cluster-name>
        [ --force ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _om-cluster-shutdown-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``<cluster-name>``
     - string
     - Name of the cluster that contains the replica 
       set to shut down.
     - yes

   * - ``--force``
     - 
     - Flag that indicates that the replica set :manual:`mongod 
       </reference/program/mongod/#bin.mongod>` processes can be 
       shut down without prompting for confirmation.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are set. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to shutdown. If omitted, uses the project ID in 
       the profile or :ref:`environment variable <mcli-env-var>`.
     - no

.. _om-cluster-shutdown-output: 

Output 
------

The command output to the terminal looks similar to the following if the 
command succeeds. If the command prints errors, see :ref:`Troubleshooting 
<mcli-troubleshooting>` for recommended solutions.

.. code-block:: text
   :copyable: false

   Changes are being applied, please check http://<hostname>:<port>/v2/<projectID>#deployment/topology for status

where: 

- ``<hostname>:<port>`` is the |mms| base URL.
- ``<projectID>`` is the unique identifier of the project that contains 
  the cluster being shut down.

.. _om-cluster-shutdown-examples: 

Examples 
--------

The following examples show the different ways to run the command.

.. _om-cluster-shutdown-example1:

Example 1
~~~~~~~~~

The following ``mongocli om cluster shutdown`` command prompts 
for confirmation before shutting down the the replica set :manual:`mongod 
</reference/program/mongod/#bin.mongod>` processes for the cluster named 
``myCluster`` in the specified project. It uses the {+default-profile+} to access the specified project.

.. code-block:: text 
   :copyable: false 

   ~  mongocli om cluster shutdown myCluster --projectId 5e2dec2131cd7c007880b05e
   Are you sure you want to shutdown: myCluster Yes

.. _om-cluster-shutdown-example2:

Example 2
~~~~~~~~~

The following ``mongocli ops-manager cluster shutdown`` command 
*does not* prompt for confirmation before shutting down the replica set 
:manual:`mongod </reference/program/mongod/#bin.mongod>` processes for the 
cluster named ``myCluster`` in the specified project. It uses the
profile ``OMprofile`` to access the specified project.

.. code-block:: text 
   :copyable: false 

   mongocli ops-manager cluster shutdown myCluster --projectId 5e2dec2131cd7c007880b05e --profile OMprofile --force

Output
~~~~~~

The :ref:`om-cluster-shutdown-example1` and :ref:`om-cluster-shutdown-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false 

   Changes are being applied, please check http://localhost:30700/v2/5e2dec2131cd7c007880b05e#deployment/topology for status
