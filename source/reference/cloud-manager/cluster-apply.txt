.. _mcli-cm-cluster-apply-command:

====================================
mongocli cloud-manager cluster apply
====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster apply`` command creates the cluster in the 
specified |cloud| project if the cluster does not already exist or 
updates the cluster in the specified |cloud| project if 
the cluster already exists. You can also: 

- Create a cluster through the |cloud-short| :cloudmgr:`UI 
  </tutorial/deploy-replica-set/>`, :ref:`CLI <mcli-cm-cluster-create-command>`, 
  and :cloudmgr:`Automation Configuration API </reference/api/automation-config/>`.

- Update a cluster through the the |cloud-short| :cloudmgr:`UI 
  </tutorial/edit-replica-set/>`, :ref:`CLI <mcli-cm-cluster-update-command>`, 
  and :cloudmgr:`Automation Configuration API </reference/api/automation-config/>`, 

.. _cm-cluster-apply-syntax:

Syntax 
------

.. code-block:: text 
   :copyable: false 

   mongocli cloud-manager|cm cluster apply 
        --file|-f <cm-settings-file> 
        [ --profile|-P <profile-name> ] 
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _cm-cluster-apply-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--file``, ``-f`` 
     - string 
     - Path to the file that contains the settings for 
       creating the cluster. To learn more about the 
       configuration properties to specify in the file, see 
       :ref:`mms-cluster-settings-file`.
     - yes

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable <mcli-env-var>`.
     - no

.. _cm-cluster-apply-command-output:

Output 
------

The command prints output similar to the following to the 
terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` for 
recommended solutions.

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://cloud.mongodb.com/v2/<projectId>#deployment/topology for status

where ``projectId`` is the unique identifier of the project.

.. _cm-cluster-apply-examples:

Examples 
--------

The following examples show two ways to run the command.

.. _cm-cluster-apply-example1:

Example 1 
~~~~~~~~~

The following ``mongocli cm cluster apply`` command deploys the settings
in the file named ``cm-cluster.yaml`` to the specified project. The
command uses the default profile for the |api| access keys and project
ID.

.. code-block:: sh 
   :copyable: false

   mongocli cm cluster apply --file cm-cluster.yaml

.. _cm-cluster-apply-example2:

Example 2 
~~~~~~~~~

The following ``mongocli cloud-manager cluster apply`` command deploys
the settings in the file named ``cm-cluster.json`` to the cluster named
``myCluster``. It uses the profile ``CMprofile`` for the |api| access 
keys. 

.. code-block:: sh 
   :copyable: false

   mongocli cloud-manager cluster apply --file cm-cluster.json --projectId 5e33a1e0f10fab0a7846aa1d --profile CMprofile

Output 
~~~~~~

The :ref:`cm-cluster-apply-example1` and :ref:`cm-cluster-apply-example2` 
commands print the following to the terminal:

.. code-block:: json
   :copyable: false 

   Changes are being applied, please check https://cloud.mongodb.com/v2/5e33a1e0f10fab0a7846aa1d#deployment/topology for status
