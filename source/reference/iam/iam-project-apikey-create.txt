.. _mcli-iam-project-apikey-create:

==================================
mongocli iam project apikey create
==================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``iam project apikey create`` command creates a new 
|api| key for the specified project.

You must have ``GROUP_USER_ADMIN`` privileges to create an 
|api| key for the project.

.. _mcli-iam-project-apikey-create-syntax:

Syntax 
------

.. code-block:: text 

   mongocli iam project|projects apiKey|apiKeys|apikeys create
        --desc "<description-of-key>"
        [ --projectId <project-id> ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        --role <list-of-roles>

.. include:: /includes/fact-command-line-help.rst

.. _mcli-iam-project-apikey-create-options:

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type 
     - Description 
     - Required? 

   * - ``--desc``
     - string
     - Description of the |api| key.
     - yes

   * - ``--projectId``
     - string
     - Unique identifier of the project for which you want to
       create an |api| key.
       
       If specified, this value overrides any project ID
       stored in your :ref:`profile <mcli-profiles>` or
       :ref:`environment variable <mcli-env-var>`.

       If omitted, your profile must contain the
       project ID.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P`` 
     - string 
     - Name of the profile where your credentials are saved.
       
       You must have credentials with the ``Project Owner`` role to
       create an |api| key for the project. 

       If omitted, uses the {+default-profile+}. To learn more about 
       creating a profile, see :ref:`mcli-configure`.
     - no

   * - ``--role`` 
     - string 
     - Role or roles to assign to the |api| key. To assign more than
       one role, you can specify each role with a ``--role`` flag or
       specify the roles in a comma-separated list with one ``--role``
       flag. 
     - yes

.. _mcli-iam-project-apikey-create-output:

Output
------

If the command succeeds, it prints the following output to the terminal.
If the command returns errors, see :ref:`Troubleshooting
<troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   API Key '<api-key-id>' created.
   Public API Key <public-key>
   Private API Key <private-key>

.. warning:: Copy and Save Public and Private Keys

   {+mcli+} returns the Private |api| Key only once. After running this
   command, immediately copy, save, and secure both the Public and
   Private |api| Keys.

.. _mcli-iam-project-apikey-create-examples:

Examples
--------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command creates an |api| key using the
      {+default-profile+}, which contains ``Project Owner``
      credentials, the project ID, and specifies the |service|
      service. The output is returned in the default format.

      .. code-block:: sh

         mongocli iam project apikey create --desc "My API key" --role GROUP_OWNER

      The previous command creates the |api| key and 
      prints the following to the terminal.

      .. code-block:: json
         :copyable: false

         API Key '5f46abb2d58b421fe3edbb3a' created.
         Public API Key cofodjld
         Private API Key c5730d9a-7b85-41be-88c3-16225b08c795

   .. tab:: JSON Output
      :tabid: json-output

      The following command creates an |api| key using the ``myOM``
      profile, which contains ``Project Owner`` credentials and
      specifies the |onprem| service. The output is returned in |json|
      format.

      .. code-block:: sh

         mongocli iam project apikey create --desc "My API key" --projectId 5a1b39eec902201990f12345 --role GROUP_USER_ADMIN --profile myOM --output json

      The previous command creates the |api| key and 
      prints the following to the terminal in |json| format.

      .. code-block:: json
         :copyable: false
