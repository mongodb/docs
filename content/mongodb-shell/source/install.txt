Supported Operating Systems
~~~~~~~~~~~~~~~~~~~~~~~~~~~

You can install MongoDB Shell 2.0.0 on these operating systems:

.. list-table::
   :widths: 30,70
   :header-rows: 1

   * - Operating System
     - Supported Versions

   * - macOS
     - 11+ (x64 and ARM64)

   * - Microsoft Windows
     - Microsoft Windows Server 2016+

       | Microsoft Windows 10+

   * - Linux
     - Red Hat Enterprise Linux (RHEL) 8+ (x64, ARM64, ppc64le, and s390x)

       | Ubuntu 20.04+ (x64 and ARM64)

       | Amazon Linux 2023 (x64 and ARM64)

       | Amazon Linux 2 (x64 and ARM64)

       | Debian 11+

       | SLES 15

       | Oracle Linux 8+ running the Red Hat Compatible Kernel (RHCK). MongoDB Shell does **not** support the Unbreakable Enterprise Kernel (UEK).

      Install with Homebrew
      ~~~~~~~~~~~~~~~~~~~~~

      This section describes how to install ``mongosh`` on macOS through
      the Homebrew package manager. To learn how to manually install
      ``mongosh`` from an archive instead, see
      :ref:`macos-install-archive`.

      .. important::

         In versions of ``mongosh`` older than 2.5.8, if you're using
         ``Node.js`` versions ^20.19.5, ^22.18.0, or ^24.3.0, the tab
         autocomplete may execute code up to and including potentially
         destructive operations. For example,
         ``db.collection.deleteMany({}).<tab>``.

         To prevent this issue, if you already have ``mongosh``
         installed, run the following command to ensure you have the
         latest versions of ``mongosh`` and Node.js:

         .. code-block:: sh

            brew upgrade mongosh

         To check what versions of ``mongosh`` and Node.js you have
         installed, run:

         .. io-code-block::
            :copyable: true

            .. input::
               :language: sh

               <path-to-mongosh-binary> --build-info

            .. output::
               :language: javascript
               :emphasize-lines: 2,9

               {
                 "version": "2.5.8",
                 "distributionKind": "packaged",
                 "buildArch": "x64",
                 "buildPlatform": "linux",
                 "buildTarget": "unknown",
                 "buildTime": "2025-09-09T10:18:30.614Z",
                 "gitVersion": "7b6bd25a97ce617f8d5bfa26c36db6d99dc92419",
                 "nodeVersion": "v24.9.0",
                 "opensslVersion": "3.6.0",
                 "sharedOpenssl": true,
                 "runtimeArch": "x64",
                 "runtimePlatform": "darwin",
                 "runtimeGlibcVersion": "N/A",
                 "deps": {
                   "nodeDriverVersion": "6.19.0",
                   "kerberosVersion": "2.2.2"
                 }
               }

         To view the complete list of system requirements for Homebrew,
         see the
         `Homebrew Website <https://docs.brew.sh/Installation>`__.

      Considerations
      ``````````````

      ``mongosh`` installed with Homebrew does not provide the same
      stability and feature completeness guarantees as the official
      builds. For example, Homebrew ``mongosh`` installations don't
      support :manual:`automatic client-side field level encryption
      </core/security-automatic-client-side-encryption/>`.

            Supported Platforms
            ~~~~~~~~~~~~~~~~~~~

            :binary:`mongosh` is supported on Debian versions 11 and
            greater. This page shows instructions for Debian 12
            (Bookworm) and Debian 11 (Bullseye).

            Procedure
            ~~~~~~~~~

            .. procedure::
               :style: normal

               .. step:: Import the public key used by the package management system

                  From a terminal, run the following command to import
                  the MongoDB public GPG key from
                  `<https://www.mongodb.org/static/pgp/server-{+pgp-version+}.asc>`_:

                  .. code-block:: sh

                     wget -qO- https://www.mongodb.org/static/pgp/server-{+pgp-version+}.asc | sudo tee /etc/apt/trusted.gpg.d/server-{+pgp-version+}.asc

                  The previous command writes the GPG key to your
                  system's ``/etc/apt/trusted.gpg.d`` folder and
                  displays the key in your terminal. You do not need to
                  copy or save the key.
                  
                  If you receive an error indicating that ``gnupg`` is
                  not installed, perform the following steps:
                     
                  #. Install ``gnupg`` and its required libraries using
                     the following command:

                     .. code-block:: sh

                        sudo apt-get install gnupg
                     
                  #. Retry importing the key:

                     .. code-block:: sh

                        wget -qO- https://www.mongodb.org/static/pgp/server-{+pgp-version+}.asc | sudo tee /etc/apt/trusted.gpg.d/server-{+pgp-version+}.asc
                        
                     
               .. step:: Create a list file for MongoDB

                  Create the list file
                  ``/etc/apt/sources.list.d/mongodb-org-{+mdb-version+}.list``
                  for your version of Debian.

                  Select the appropriate tab for your version of Debian.

                  .. tabs::
                           
                     .. tab:: 12 (Bookworm)
                        :tabid: bookworm

                        .. code-block:: sh

                           echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/debian bookworm/mongodb-org/8.0 main" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
                     
                     .. tab:: 11 (Bullseye)
                        :tabid: bullseye

                        .. code-block:: sh

                           echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/debian bullseye/mongodb-org/8.0 main" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
                        
               .. step:: Reload the local package database

                  .. code-block:: sh

                     sudo apt-get update

               .. step:: Install the mongosh package

                  .. include:: /includes/intro-openssl-installs.rst

                  - Starting in Debian 12 (Bookworm), the default
                    OpenSSL version is 3.0.

                  - In Debian 11 (Bullseye) and older versions, the
                    default OpenSSL version is 1.1.

                  To install the latest stable version of ``mongosh`` with the included
                  OpenSSL libraries:

                  .. code-block:: sh

                     sudo apt-get install -y mongodb-mongosh

                  To install ``mongosh`` with OpenSSL 3.0 libraries:

                  .. code-block:: sh 

                     sudo apt-get install -y mongodb-mongosh-shared-openssl3

                  To install ``mongosh`` with OpenSSL 1.1 libraries:

                  .. code-block:: sh 

                     sudo apt-get install -y mongodb-mongosh-shared-openssl11

               .. step:: Confirm that mongosh installed successfully

                  To confirm that ``mongosh`` installed successfully,
                  run the following command:

                  .. code-block:: sh

                     mongosh --version

                  The command returns the version of ``mongosh`` you
                  installed.
