Considerations
--------------

- .. include:: /includes/facts/cross-region-limits.rst

- .. include:: /includes/fact-cluster-project-limit.rst

- If your |service| project contains a
  :ref:`custom role <mongodb-roles>` that uses actions introduced
  in a specific MongoDB version, you must delete that role before
  creating clusters with an earlier MongoDB version.

- |service| clusters created after July 2020 use |tls| version 1.2 by
  default.

- When you create a cluster, |service| creates a :doc:`network
  container </reference/api/vpc-get-containers-list-all/>` in the
  project for the cloud provider to which you deploy the cluster if one
  does not already exist.

- You can't modify a tenant (**M2**/**M5**) cluster, even to upgrade it
  to a dedicated cluster.

- You can't modify a paused cluster (**"paused" : true**). You must
  call this endpoint to set **"paused" : false**. After this endpoint
  responds with **"paused" : false**, you can call it again with the
  changes you want to make to the cluster.

Required Roles
--------------

To use this resource, your API Key must have either the
:authrole:`Organization Owner` or :authrole:`Project Owner` roles.

Resource
--------

.. include:: /includes/api/base-api-uri-1-5.rst

.. code-block:: http

   PATCH /groups/{GROUP-ID}/clusters

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1
   :stub-columns: 1
   :widths: 20 14 11 55

   * - Path Element
     - Type
     - Necessity
     - Description

   * - GROUP-ID
     - string
     - Required
     - Unique identifier for the :ref:`project <group-id>` in which to
       create the cluster.

   * - CLUSTER-NAME
     - string
     - Required
     - Name of the cluster to modify.
