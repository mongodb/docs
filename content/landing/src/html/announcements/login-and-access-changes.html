Login and Access Changes to MongoDB Services

Publish Date: July 09, 2019

On July 10th, 2019 we are launching a new unified login experience. The new login gives you access to all MongoDB services (MongoDB Atlas, MongoDB Cloud Manager, Support, and JIRA) with a single identity that allows you to move between services seamlessly without needing to reauthenticate.

What does this mean?

You will continue to authenticate with MongoDB services using the same credentials you used in the past. If you switch between MongoDB services after authenticating, you will remain signed in. You should not notice a difference in any service after this change. If you notice any differences or have separate credentials for different MongoDB services, please contact Support-Operations at support-operations@mongodb.com to resolve the issue. In the future, you will be able to log in using different authentication providers such as Google Sign-In or SSO providers like Okta, Ping, and others. Customers using supported SSO providers will be able to federate their identity to MongoDB.
