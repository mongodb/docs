Access ``All Clusters`` View
----------------------------

Access the :guilabel:`All Clusters` view by clicking on the
:guilabel:`Clusters` link at the top left corner of |service|.

In the search box, you can enter the :ref:`organization name
<organizations>`, :ref:`project name <projects>`, or |cluster| name to
search for specific organizations, projects, or |clusters|.

You can also use the dropdowns to filter the displayed list by
:guilabel:`Availability`, |cluster| :guilabel:`Type`, MongoDB
:guilabel:`Version`, and :guilabel:`Configuration` settings. If no
projects within an Organization match the filter criteria, then that
Organization will not appear in the results as well.

.. note::

   By default, |clusters| without any contact with a Monitoring
   Agent for the last 6 months are not displayed. To display these
   |clusters|, click :guilabel:`Show Inactive Clusters`.


|cluster| Details
-----------------

The :guilabel:`All Clusters` view displays the |clusters| by
:doc:`Organization/Projects </organizations-projects>`.

The view displays the following information for the |cluster|:

.. list-table::
   :header-rows: 1

   * - Field
     - Description

   * - :guilabel:`Name`
     - The name of the |cluster|.

   * - :guilabel:`Version`
     - MongoDB Version of the |cluster|.

   * - :guilabel:`Data Size`

     - Size of the |cluster| data.

   * - :guilabel:`Nodes`

     - Number of nodes in the |cluster|.

   * - :guilabel:`Backup`

     - Backup enabled status.

   * - :guilabel:`SSL`

     - SSL enabled status.

   * - :guilabel:`Auth`

     - Authentication required status.

   * - :guilabel:`Alerts`

     - Number of open alerts.
