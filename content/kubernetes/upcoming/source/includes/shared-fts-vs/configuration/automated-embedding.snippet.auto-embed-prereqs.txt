
- ``mongot`` 1.70.0 or later on a self-managed deployment
  (Community tarball or container, or Local Development).

- MongoDB 8.2 or later.

- Two |voyage| |api| keys.

  Use two separate keys, one for index-time embedding generation
  and one for query-time. Separate keys isolate query workload
  from indexing workload and allow independent rate-limit
  accounting. To learn more, see :ref:`auto-embed-rate-limits`.

- Outbound network access from ``mongot`` to the embedding
  endpoint.

  The default endpoint is ``https://ai.mongodb.com/v1/embeddings``,
  which proxies |voyage| with MongoDB-managed billing, for keys
  generated using the {+atlas-ui+}. You can also use
  ``https://api.voyageai.com/v1/embeddings`` for direct |voyage|
  access if you generated the |api| key directly from |voyage|.
