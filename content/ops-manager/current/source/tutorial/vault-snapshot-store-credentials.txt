==============================================
HashiCorp Vault for Snapshot Store Credentials
==============================================

.. default-domain:: mongodb

.. meta::
   :keywords: s3, AppDB, secret manager, backup
   :description: Learn how Ops Manager reads S3 snapshot store credentials from HashiCorp Vault on demand, including caching, security, prerequisites, and migration steps.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

You can store the |aws| credentials for an |s3ss| in the
|onprem| application database or in
`HashiCorp Vault <https://www.hashicorp.com/products/vault>`__.
When you store credentials in a key vault, |onprem| fetches
them from HashiCorp Vault on demand. On-demand retrieval
helps you centralize, rotate, and audit sensitive
credentials.

|onprem| supports HashiCorp Vault and local credential storage.
You choose which method to use for each |s3ss|.

Credential Storage Options
--------------------------

|onprem| supports two credential storage locations for an |s3ss|:

.. list-table::
   :header-rows: 1
   :stub-columns: 1
   :widths: 25 75

   * - Storage Location
     - Description

   * - AppDB
     - |onprem| stores the |aws| access key and secret key in the
       |onprem| application database. AppDB is the default method and
       doesn't require HashiCorp Vault.

   * - Key Vault
     - |onprem| reads the |aws| access key and secret key from
       HashiCorp Vault on demand. |onprem| stores only the path to each
       secret, not the secret value.

Use a key vault when your security or audit requirements call for a
centralized secret manager. Use AppDB when you do not use HashiCorp
Vault.

How Ops Manager Uses HashiCorp Vault
------------------------------------

To use HashiCorp Vault for |s3ss| credentials, you must complete
two configuration tasks:

#. Create a key vault configuration that stores the connection and
   authentication details for your HashiCorp Vault server. To learn
   how, see :ref:`om-configure-key-vault`.

#. Configure an |s3ss| to use the key vault. You specify the key
   vault, the path to the |aws| access key, and the path to the
   |aws| secret key. |onprem| reads the credentials from the key
   vault and passes them to |aws| S3. To learn how, see
   :ref:`add-s3-blockstore`.

|onprem| authenticates to HashiCorp Vault with either a token or a
|jwt|. |onprem| then reads the |aws| access key and secret key from the
secret paths that you specify for the snapshot store. You provide each
path in the form ``<secret-path>/<key>``.

Credential Retrieval and Caching
--------------------------------

|onprem| reads the |aws| access key and secret key from HashiCorp
Vault and caches them in memory. |onprem| doesn't store these
credentials in the |onprem| application database.

By default, |onprem| refreshes a cached credential every five
minutes. To change this interval, set the
``mms.keyVault.secret.cacheSoftTtlMs`` property.

If HashiCorp Vault is unavailable when |onprem| refreshes a cached
credential, |onprem| keeps using the last known credential and logs
the event, so existing backups continue. If no cached credential
exists and |onprem| can't reach HashiCorp Vault, the operation that
needs the credential fails.

.. important::

   |onprem| doesn't have an alert specifically for HashiCorp Vault
   availability. If Vault becomes unreachable and no cached
   credential remains, the resulting backup failures trigger the
   same backup alerts that apply to any failed backup job. Configure
   backup alerts to receive them. To learn how, see
   :ref:`configure-alerts`.

When |onprem| validates an |s3ss| that uses a key vault and S3
returns a ``401`` or ``403`` error, |onprem| refreshes the credentials
from HashiCorp Vault once, then retries. If the retry also fails,
|onprem| returns an error and the validation fails.

Security
--------

|onprem| protects your HashiCorp Vault authentication details as
follows:

- |onprem| encrypts the Vault authentication secret at rest with the
  ``gen.key`` file, the same method that |onprem| uses for other
  integrations.

- |onprem| redacts the Vault authentication secret and any custom
  certificate in API responses and in the |onprem| console. |onprem|
  returns ``[REDACTED]`` instead of the stored value.

Requirements
------------

To use HashiCorp Vault for |s3ss| credentials, you need:

- A reachable HashiCorp Vault deployment.

- Vault policies, roles, and secret paths that grant |onprem|
  read-only access to the |aws| credentials.

- Secure, reliable network connectivity between |onprem|, the
  :term:`Backup Daemon`, and HashiCorp Vault.

Migrate Existing Snapshot Stores to HashiCorp Vault
---------------------------------------------------

To switch an existing |s3ss| from local credentials to HashiCorp
Vault:

#. Create a key vault configuration, as described in
   :ref:`om-configure-key-vault`.

#. Edit the |s3ss| and set its credential storage location to the key
   vault, as described in :ref:`edit-s3-blockstore`.
