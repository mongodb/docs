====================
Supported Topologies
====================

.. facet::
   :name: genre
   :values: reference

.. meta:: 
   :description: Learn about the supported topologies for deploying MongoDB Search and Vector Search on Kubernetes, including single-cluster replica sets and sharded clusters.
   :keywords: vector search, compatibility, requirements, mongot, mongod, version

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

You can deploy |text-search| and |vector-search| with the
|k8s-op-short| against a range of MongoDB deployment topologies. The
topology that you choose determines how the |k8s-op-short| deploys
and connects the ``mongot`` process to your MongoDB deployment.

Use this page to confirm that your planned topology is supported
before you deploy. To confirm the MongoDB Server, ``mongot``, and
|k8s-op-short| versions that you need, see
:ref:`k8s-fts-vs-compatibility-requirements`.

Deployment Dimensions
---------------------

A supported topology is a combination of the following dimensions:

- **MongoDB Cluster topology.** Whether the ``mongot`` process serves a
  MongoDB replica set or a sharded cluster. For sharded clusters, the
  |k8s-op-short| deploys a separate ``mongot`` group for each shard.

- **Load balancer mode.** When you run multiple ``mongot`` instances,
  you must place an L7 load balancer between ``mongod`` and the
  ``mongot`` pods. You can use one of the following options: 
  
  - *Managed* load balancer. The |k8s-op-short| deploys and configures 
    an Envoy proxy.
  - *Unmanaged* load balancer. You provide and configure your
    own L7 load balancer. 
    
  A single ``mongot`` instance connects directly to ``mongod`` and does 
  not require a load balancer.

- **MongoDB deployment.** Whether the |k8s-op-short| manages the MongoDB
  deployment inside |k8s| (managed) or the ``mongot`` process
  connects to a MongoDB deployment that runs outside |k8s|
  (external).

Supported Topology Matrix
-------------------------

The following table lists the support status for each combination of
deployment dimensions:

.. important::

   External MongoDB deployments require MongoDB Enterprise. MongoDB
   Community supports only replica set deployments that the
   |k8s-op-short| manages inside |k8s|.

.. list-table::
   :header-rows: 1
   :widths: 25 25 25 25

   * - Cluster Topology
     - Load Balancer Mode
     - MongoDB Deployment
     - Support Status

   * - Replica set
     - Managed
     - Managed
     - Supported

   * - Replica set
     - Managed
     - External
     - Supported

   * - Replica set
     - Unmanaged
     - Managed
     - Supported

   * - Replica set
     - Unmanaged
     - External
     - Supported

   * - Sharded cluster
     - Managed
     - Managed
     - Supported

   * - Sharded cluster
     - Managed
     - External
     - Supported

   * - Sharded cluster
     - Unmanaged
     - Managed
     - Supported

   * - Sharded cluster
     - Unmanaged
     - External
     - Supported

Topology Considerations
-----------------------

Review the following considerations when you choose a topology:

- External MongoDB deployments are available only for MongoDB
  Enterprise. MongoDB Community supports replica set deployments
  that the |k8s-op-short| manages inside |k8s|.

For other deployment limitations, see :ref:`k8s-fts-vs-limitations`.
