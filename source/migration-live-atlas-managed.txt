=========================================================================
Live Migrate: Pull Your Data into Atlas (MongoDB Before {+c2c-version+})
=========================================================================

.. default-domain:: mongodb

.. facet::
   :name: genre
   :values: tutorial

.. meta::
   :keywords: live migration (pull) to MongoDB Atlas pre-6.0, legacy pull live migration

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can pull data from existing MongoDB deployments, ``JSON``, or ``CSV``
files into |service| using one of the following tools managed by |service|
within the {+atlas-ui+}.

To locate live migration (pull) in the {+atlas-ui+}, in the left-side
panel of your organization's page, go to :guilabel:`Live Migration to Atlas`
and select :guilabel:`General Live Migration`.

.. include:: /includes/fact-c2c.rst

.. list-table::
   :header-rows: 1
   :widths: 35 65

   * - Method
     - Description

   * - :ref:`live-import-replica`

     - Use the |service| user interface to pull a MongoDB replica set
       into an |service| replica set {+cluster+}.

   * - :ref:`live-import-sharded`

     - Use the |service| user interface to pull a MongoDB sharded {+cluster+}
       into an |service| sharded {+cluster+}.

   * - Live migrate a standalone MongoDB node

     - :manual:`Convert </tutorial/convert-standalone-to-replica-set>`
       the standalone MongoDB node to a single-node replica set and then
       :ref:`live migrate (pull) <live-migration-pre60>` a {+cluster+}
       into |service|. If you can't convert the standalone to a replica set,
       use :ref:`mongorestore <mongorestore>` to migrate to |service|.
