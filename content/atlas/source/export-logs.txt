.. _export-logs:

================================
Export Logs to External Services
================================

.. meta::
   :description: Configure your M10+ Atlas clusters to export system logs every minute to external services, including AWS S3, Azure Blob Storage, Datadog, Google Cloud Storage, and Splunk.

.. default-domain:: mongodb

.. facet::
   :name: genre
   :values: reference

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can configure your ``M10+`` {+service+} clusters to export
system logs every minute to an external service. Exporting logs to
an external service enables centralized log management, long-term
retention, and integration with your existing monitoring and
observability tools.

Supported Log Types
-------------------

{+service+} supports exporting the following log types:

- ``mongod`` logs
- ``mongos`` logs
- ``mongod-audit`` logs
- ``mongos-audit`` logs

You can specify which log types you want to export and configure
up to 10 export paths to send logs to multiple destinations
simultaneously.

.. important::

   Logs can contain sensitive information (including PII). You are
   responsible for the storage and treatment of your logs in your
   external service. To have {+service+}
   :ref:`redact certain information <database-log-redaction>` before
   exporting logs, contact |mdb-support|.

Considerations
--------------

- You must have an ``M10+`` cluster to export logs. Log export is
  not available for free or shared-tier clusters.
- You must have :authrole:`Project Owner` or
  :authrole:`Organization Owner` access to configure log export.
- Each {+service+} host typically produces 1 GB of logs per day.
  Exporting logs incurs a data transfer cost. The exact data
  transfer cost varies depending on the destination, region, and
  cloud provider.
- Network issues or retries can cause duplicate log entries in
  your external service.

Supported Export Destinations
-----------------------------

The following table summarizes the external services that
{+service+} supports for log export:

.. list-table::
   :header-rows: 1
   :widths: 30 70

   * - Destination
     - Description

   * - :ref:`AWS S3 <export-logs-external-sinks>`
     - Export ``mongod``, ``mongos``, and audit logs to an |aws|
       |s3| bucket. Supports Multi-Region Access Points (MRAPs).

   * - :ref:`Azure Blob Storage <export-logs-azure>`
     - Export ``mongod``, ``mongos``, and audit logs to an
       {+az-bs+} container.

   * - :ref:`Datadog <export-logs-datadog>`
     - Export ``mongod``, ``mongos``, and audit logs to Datadog
       for real-time log analysis and dashboards.

   * - :ref:`Google Cloud Storage <export-logs-gcs>`
     - Export ``mongod``, ``mongos``, and audit logs to a
       {+gcs+} bucket.

   * - :ref:`OpenTelemetry <export-logs-otel>`
     - Export ``mongod``, ``mongos``, and audit logs to an
       OpenTelemetry (OTel) endpoint for vendor-neutral
       observability.

   * - :ref:`Splunk <export-logs-splunk>`
     - Export ``mongod``, ``mongos``, and audit logs to Splunk
       Cloud or Splunk Enterprise for centralized log search
       and analysis.
