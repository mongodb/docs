====================================
Set Up Monitoring for New Deployment
====================================

.. meta::
   :description: Set up monitoring for a new mongot deployment by enabling metrics, integrating Prometheus, configuring alerts, centralizing logs, and capturing an FTDC baseline.

.. facet::
   :name: genre
   :values: tutorial

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 3
   :class: singlecol

Follow these steps to set up monitoring for a new ``mongot`` deployment. 
Monitoring gives you visibility into the overall health of your ``mongot`` 
deployment and allows you to troubleshoot any issues that might come up.

Steps
-----

.. procedure::

   .. step:: Enable the metrics endpoint.

      The ``mongot`` configuration file included in the package download, 
      ``config.default.yml``, enables the metrics endpoint on 
      ``localhost:9946``. For production deployments, bind to an interface
      reachable from your Prometheus. For example, if you front Prometheus with
      a network policy or a specific NIC, you can use ``0.0.0.0:9946``. If your
      endpoint is exposed beyond loopback, protect it with TLS or a reverse
      proxy. For details, see :ref:`<mongot-configuration-options>`.

   .. step:: Set up your monitoring toolkit to gather metrics.

      For details, see :ref:`<mongot-tool-integrations>`.

   .. step:: Configure the recommended alerts.

      For details, see :ref:`<mongot-recommended-alerts>`.

   .. step:: Forward the logs.
     
      Centralize ``mongot`` logs into your existing log platform. Most failures
      are visible in logs before they appear in metrics. For details, see 
      :ref:`<mongot-logs-ftdc>`.
       
   .. step:: Capture an FTDC baseline.

      Capture an FTDC baseline so that you can provide MongoDB support with 
      a comparison point if something goes wrong later. For details, 
      see :ref:`<mongot-logs-ftdc>`.
