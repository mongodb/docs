====================================================
Get Status of a Request to Verify LDAP Configuration
====================================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

Returns the status of the most recent request for verification of an
|ldap| over |tls| configuration. |service| returns an |http| 404 error
if you request the status of any request other than the most recent.

.. note::

   - .. include:: /includes/facts/explain-ldap.rst

   - .. include:: /includes/facts/projects-groups-synonyms.rst

.. include:: /includes/api-base-url.rst

Syntax
------

.. code-block:: http

   GET /groups/{GROUP-ID}/userSecurity/ldap/verify/{REQUEST-ID}

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 15 10 10 65
   :header-rows: 1

   * - Parameter
     - Type
     - Necessity
     - Description

   * - ``GROUP-ID``
     - string
     - Required
     - Unique identifier for the |service| project associated with the
       verification request.

   * - ``REQUEST-ID``
     - string
     - Required
     - Unique identifier of a request to verify an |ldap|
       configuration. Returned in the response document to the
       :doc:`Verify LDAP Configuration </reference/api/ldaps-configuration-request-verification>` endpoint.

Example Request
---------------

The following example requests the status of a request to verify an
|ldap| configuration.

.. code-block:: shell

   curl --include --user "{PUBLIC-KEY}:{PRIVATE-KEY}" \
        --header "Accept: application/json" \
        --header "Content-Type: application/json" \
        --digest \
        --request GET "https://cloud.mongodb.com/api/atlas/v1.0/groups/{GROUP-ID}/userSecurity/ldap/verify/{REQUEST-ID}?pretty=true"

Example Response
----------------

The following example returns a status of ``SUCCESS``.
