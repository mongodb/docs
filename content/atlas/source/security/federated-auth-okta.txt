============================================
Configure Federated Authentication from Okta
============================================

.. meta::
   :description: Configure federated authentication with Okta for MongoDB Atlas to allow users to log in with company credentials and manage SAML identity provider settings.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. |idp-provider| replace:: Okta

This guide shows you how to configure federated authentication
using |idp-provider| as your |idp|.

After integrating |idp-provider| and |service|, you can use your
company's credentials to log in to |service| and other MongoDB cloud
services.

.. note::

   |service| now supports |sso| for database users through
   :ref:`oidc-authentication-authorization`.

   If you are using Okta's built-in MongoDB Cloud app, you can use
   `Okta's documentation
   <https://saml-doc.okta.com/SAML_Docs/How-to-Configure-SAML-2.0-for-MongoDB-Cloud.html>`__.

   If you are creating your own SAML app, use the procedures described here.

.. include:: /includes/roles-required-fed-auth.rst

Prerequisites
-------------

To use |idp-provider| as an |idp| for |service|, you must have:

- An `Okta account <https://www.okta.com/>`__.
- A custom, routable domain name.

Procedures
----------

Throughout the following procedure, it is helpful to have one browser
tab open to your :ref:`Atlas Federation Management Console
<atlas-federated-authentication>` and one tab open to your Okta
account.

Sign in to |service| Using Your Login URL
-----------------------------------------

All users that you assign to the |idp-provider| application can log
in to |service| using their |idp-provider| credentials on the
:guilabel:`Login URL`. Users have access to the organizations you
mapped to your |idp|.

.. include:: /includes/admonitions/use-alternate-idp-multiple-mapped-domains.rst
