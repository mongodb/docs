=============
Monitor Usage
=============

.. default-domain:: mongodb

.. meta::
   :description: Monitor API key usage for Voyage AI models in the Atlas UI.
   :keywords: Atlas, Atlas UI, Voyage AI, API key usage, usage monitoring, billing, usage metrics

.. facet::
   :name: genre
   :values: tutorial

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can monitor the usage of your {+model-api-keys+} in the {+atlas-ui+}
by tracking API requests, token consumption, and costs. Usage monitoring is
available at both the organization and project levels.

View Usage
----------

You can view your API key usage at the organization or project level.

.. tabs::

   .. tab:: Organization Level
      :tabid: org-level

      .. _voyage-view-usage-org-level:

      At the organization level, you can view comprehensive usage metrics
      across all projects, including free and paid usage, and access billing
      information.

      You must have :authrole:`Organization Read Only` or higher access to {+service+}.

      .. include:: /includes/usage/steps-view-usage-org-level.rst

   .. tab:: Project Level
      :tabid: project-level

      .. _voyage-view-usage-project-level:

      At the project level, you can view usage metrics for {+model-api-keys+}
      within a specific project. This view provides a focused look at usage
      for a single project.

      You must have :authrole:`Project Read Only` or higher access to {+service+}.

      .. include:: /includes/usage/steps-view-usage-project-level.rst

.. _voyage-usage-page-features:

Usage Page Features
-------------------

The :guilabel:`Usage` page provides several views that you can use to 
monitor and analyze your {+voyageai+} usage. The available features vary 
depending on whether you are viewing usage at the organization or project level.

Total Usage
~~~~~~~~~~~

In the :guilabel:`Total Usage` chart, you can view the total :term:`tokens` 
used across your organization or project over time. You can filter usage data by 
using the following options:

.. list-table::
   :header-rows: 1
   :widths: 30 40 15 15

   * - Filter
     - Description
     - Organization Level
     - Project Level

   * - :guilabel:`Usage Period`
     - Select the time period for which to view usage data.
     - ✓
     - ✓

   * - :guilabel:`Projects`
     - Filter by specific projects in your organization.
     - ✓
     -

   * - :guilabel:`Models`
     - Filter by specific {+voyageai+} models.
     - ✓
     - ✓

You can also toggle between :guilabel:`Tokens` and :guilabel:`Pixels` to view usage measured in tokens processed or pixels processed (applies to multimodal models).

Additional Views
~~~~~~~~~~~~~~~~

The :guilabel:`Usage` page also provides the following views to help
you monitor and analyze your {+voyageai+} usage:

.. list-table::
   :header-rows: 1
   :widths: 30 40 15 15

   * - Feature
     - Description
     - Organization Level
     - Project Level

   * - :guilabel:`Free Usage` tab
     - Click the :guilabel:`Free Usage` tab to view remaining free quota by model.
     - ✓
     -

   * - :guilabel:`Top Keys by API Requests`
     - Shows which {+model-api-keys+} are generating the most API requests.
     - ✓
     - ✓

   * - :guilabel:`Top Projects by API Requests`
     - Displays which projects are making the most API requests.
     - ✓
     -
