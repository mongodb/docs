Configure Online Archive for Standard Collections
=================================================

.. meta::
   :description: Configure an Online Archive for a standard collection in Atlas by creating an archiving rule using date match or custom criteria and specifying partition fields for efficient data retrieval.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/flex-dont-use.rst

Overview
--------

You can configure data in a standard collection to be archived by
specifying an archiving rule. The archiving rule for a standard
collection can be one of the following:

- A date field paired with a retention period in days, which
  determines when |service| moves data from the cluster to the
  archive (:guilabel:`Date Match`).
- A custom |json| query that selects the documents to archive
  (:guilabel:`Custom Criteria`).

To configure your |service| cluster for online archive:

1. Create an archiving rule by providing the collection namespace
   and the criteria for selecting data to archive.

2. (Optional) Specify commonly queried fields to partition archived
   data.

When you configure an {+Online-Archive+} on your {+cluster+},
|service| creates 2 {+fdi+}\s on your {+cluster+} for your archive
only and for your {+cluster+} and archive.

For limitations that apply to all online archives, see
:ref:`config-online-archive-limitations`.

Prerequisites
-------------

Before configuring an {+Online-Archive+}, ensure that your |service|
cluster is healthy. If one node is down, the archive remains in
``PENDING`` status until the cluster is fully operational.

Required Access
---------------

To create an {+Online-Archive+}, you must have
:authrole:`Project Data Access Admin` access or higher to the
project.

To watch for an archive to be available, you must have
:authrole:`Project Read Only` access or higher to the project.

Configure Through the {+atlas-admin-api+}
----------------------------------------------

To configure an online archive from the API, send a ``POST`` request
to the :oas-bump-atlas-tag:`onlineArchives <online-archive>`
endpoint.

.. note::

   If you use the ``DATE`` criteria, you must specify the ``date``
   field as part of the partition keys.

If the cluster already has an ``Active`` online archive with the
same archiving rule for the same database and collection, the
operation fails. However, if the existing online archive is in
``Paused`` or ``Deleted`` state, the new online archive is created
and its status is set to ``Active``. To learn more about the syntax
and options, see
:oas-bump-atlas-op:`API <creategroupclusteronlinearchive>`.
