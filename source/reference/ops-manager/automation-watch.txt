=====================================
mongocli ops-manager automation watch
=====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``automation watch`` command monitors the specified 
|onprem| project's automation configuration status until it reaches
a goal state, at which time it displays a confirmation message.

.. _om-automation-watch-syntax:

Syntax 
------

.. code-block:: text 
   
   mongocli ops-manager|om automation watch 
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --projectId projectId ]

.. include:: /includes/fact-command-line-help.rst

.. _om-automation-watch-options:

Options
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - .. include:: /includes/extracts/fact-basic-options-project-id.rst
     - no

.. _om-automation-watch-output: 

Output 
------

The ``automation watch`` command continuously polls an |onprem| 
project's automation configuration status until it reaches a goal 
state. While the command is polling, it periodically prints dots 
(``.``) to the terminal.

When the automation configuration reaches a goal state, or if it is 
already in an stable state, the command prints a confirmation message.

If the command prints an error, see 
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. _om-automation-watch-examples:

Example
-------

The following ``mongocli ops-manager automation watch`` command 
monitors the |onprem| project associated with the default profile. 

.. code-block:: text
   :copyable: false

   mongocli ops-manager automation watch

In this example project, the |onprem| replica set is restarted just 
before running the command. The command prints dots while the replica 
set is restarting, then prints a status message:

.. code-block:: text
   :copyable: false

   .......................
   Changes deployed successfully
