=======================
|data-lake| Limitations
=======================

.. default-domain:: mongodb

.. meta::
   :keywords: |data-lake| limitations

.. include:: /includes/fact-data-lake-beta.rst

|service| |data-lake| only supports the following `Amazon S3 storage classes 
<https://aws.amazon.com/s3/storage-classes/?nc=sn&loc=3>`__: 

* `S3 Standard <https://aws.amazon.com/s3/storage-classes/?nc=sn&loc=3#General_purpose>`__ 
* `S3 Intelligent-Tiering <https://aws.amazon.com/s3/storage-classes/?nc=sn&loc=3#Unknown_or_changing_access>`__

|data-lake| does not support the following features:

* Creating Indexes
* Monitoring |data-lakes| with |service| monitoring tools
* Creating a |data-lake| with |s3| buckets from more than one |aws| account
* Assigning |service| temporary users permission to query a |data-lake|
* Querying documents larger than 16MB
* Adding IP address associated with your |data-lakes| to your |service| project whitelist
* Assigning |service| read only access to your |aws| account with |aws| security groups
* Returning more than 1,000 collections for wildcard collections
