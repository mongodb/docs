=============
Test Failover
=============

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. include:: /includes/fact-projects-groups-synonyms.rst

.. include:: /includes/api-requirements.rst

To learn more about failover testing and the replica set election 
process, see :doc:`Test Failover </tutorial/test-failover>`.

Permissions Required 
--------------------

You must have :authrole:`Project Cluster Manager` or higher role to 
test failover.

Resource 
--------

.. include:: /includes/api-base-url.rst

Syntax
------

.. code-block:: none

   POST /groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/restartPrimaries

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1
   :widths: 15 10 10 65

   * - Path Element
     - Type
     - Necessity
     - Description

   * - ``GROUP-ID``

     - String

     - Required

     - The unique identifier of the :ref:`project <group-id>`
       containing the cluster you want to test.

   * - ``CLUSTER-NAME``

     - String

     - Required

     - The name of the cluster to test.
