=========================
sh.isConfigShardEnabled()
=========================

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

Definition
----------

.. method:: sh.isConfigShardEnabled()

The ``sh.isConfigShardEnabled()`` method returns whether
a cluster has a :ref:`config shard <config-shard-concept>`.
If it does, then ``sh.isConfigShardEnabled()`` also
returns the config shard's host and :ref:`tag <sharding-add-shard-tag>`
information.

Output
------

``sh.isConfigShardEnabled`` returns a document
that contains the following fields:

.. code-block:: shell
   
   {
      enabled: <boolean>,
      host?: <string>,
      tags?: [<string>]
   }

.. list-table
   :header-rows: 1

   * - Field
     - Type
     - Description

   * - ``enabled``
     - boolean
     - ``true`` if the cluster has a config
       shard, ``false`` if it does not.

   * - ``host``
     - string
     - Hostname of config shard.
       Only present if ``enabled`` is ``true``.
       

   * - ``tags``
     - array of strings
     - Tags that have been added to
       shard using :ref:`sharding-add-shard-tag`.
       Only present if ``enabled`` is ``true`` and
       the shard has associated tags.

Example
-------

The following example shows the method call and output 
when you call ``sh.isConfigShardEnabled()`` on
a cluster that has a config shard, but this shard
does not have any associated tags:

.. code-block:: shell

   $ sh.isConfigShardEnabled()

   > { enabled: true, host: 'shard3/localhost:27017' }
