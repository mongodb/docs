Overview
--------

.. include:: /includes/extracts/enable-public-api-overview.rst

Once you have enabled the Public API, users who want to use the API must
generate their own API keys and optionally configure access to whitelisted
operations. See :doc:`/tutorial/configure-public-api-access`. A user's
:doc:`roles </reference/user-roles>` determine which :doc:`API resources
</reference/api>` the user can access.

For a description of the Public API, see the :doc:`/core/api`.
