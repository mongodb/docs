.. _mcli-om-admin-backup-s3-describe-command:

=============================================
mongocli ops-manager admin backup s3 describe
=============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/admonitions/admin-requires-global-owner.rst

The ``admin backup s3 describe`` command retrieves 
configuration information for the specified backup
:opsmgr:`S3 blockstore </reference/glossary/#term-s3-snapshot-store>` 
configuration. You can also retrieve an |s3| configuration using 
the :opsmgr:`API </reference/api/admin/backup/snapshot/s3Configs/get-one-s3-blockstore-configuration-by-id/>`, or view |s3| 
configurations in the 
:opsmgr:`Administration Console </admin-console/>`.

.. _om-admin-backup-s3-describe-syntax: 

Syntax 
------

.. code-block:: text 

   mongocli ops-manager|om admin backup s3 describe|get <s3-name> 
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. include:: /includes/fact-command-line-help.rst

Arguments
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument
     - Type 
     - Description 
     - Required?

   * - ``<s3-name>``
     - string
     - Unique identifier of the |s3| blockstore configuration that you
       want to describe. You can retrieve a list of |s3| names for 
       a deployment with the 
       :ref:`List Backup S3 Configurations <mcli-om-admin-backup-s3-list-cmd>` command.
     - yes

.. _om-admin-backup-s3-describe-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for accessing |onprem| are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

.. _om-admin-backup-s3-describe-examples:

Examples
--------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command retrieves configuration information for an 
      |s3| blockstore using the {+default-profile+}, which contains 
      :authrole:`Global Owner` |mms| credentials. The output is 
      returned in the default format.

      .. code-block:: sh

         mongocli ops-manager admin backup s3 describe s3BackupExample

      The previous command prints the following to the terminal.

      .. code-block:: sh
         :copyable: false

         NAME               URI                        SSL    LOAD FACTOR   AUTH METHOD
         s3BackupExample    mongodb://127.0.0.1:27017  false  1            
         
   .. tab:: JSON Output
      :tabid: json-output

      The following command retrieves configuration information for an 
      |s3| blockstore using the {+default-profile+}, which contains 
      :authrole:`Global Owner` |mms| credentials. The output is 
      returned in |json| format.

      .. code-block:: sh

         mongocli ops-manager admin backup s3 describe s3BackupExample -o json

      The previous command prints the following to the terminal in
      |json| format.

      .. code-block:: json 
         :copyable: false 
      
         {
           "id": "s3BackupExample",
           "uri": "mongodb://127.0.0.1:27017",
           "ssl": false,
           "assignmentEnabled": true,
           "encryptedCredentials": false,
           "loadFactor": 1,
           "awsAccessKey": "access",
           "awsSecretKey": "secret",
           "s3BucketEndpoint": "https://s3.us-east-1.amazonaws.com/bucketExample/",
           "s3BucketName": "bucketExample",
           "s3MaxConnections": 50,
           "disableProxyS3": false,
           "acceptedTos": true,
           "sseEnabled": false,
           "pathStyleAccessEnabled": false
         }
