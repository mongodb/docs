.. _mcli-om-admin-backup-oplog-describe-cmd:

================================================
mongocli ops-manager admin backup oplog describe
================================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/admonitions/admin-requires-global-owner.rst

The ``admin backup oplog describe`` command retrieves the 
configuration of one oplog store. You can also 
retrieve the configuration of one oplog store using 
the :opsmgr:`API </reference/api/admin/backup/oplog/mongoConfigs/get-one-oplog-configuration-by-id/>`.

.. _mcli-om-admin-backup-oplog-describe-syntax: 

Syntax 
------

.. code-block:: shell 

   mongocli ops-manager|om admin backup oplog(s) describe|get <oplog-id>
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. include:: /includes/fact-command-line-help.rst

Arguments
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument
     - Type 
     - Description 
     - Required?

   * - ``<oplog-id>``
     - string
     - Unique identifier of the oplog store that you
       want to describe. You can retrieve a list of oplog store IDs for 
       a project with the 
       :ref:`List Backup Oplog Configurations <mcli-om-admin-backup-oplog-list-cmd>` command.
     - yes

.. _mcli-om-admin-backup-oplog-describe-options: 

Options 
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for accessing |onprem| are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

.. _mcli-om-admin-backup-oplog-list-output: 

Output 
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh 
   :copyable: false 

   NAME         URI              SSL              LOAD FACTOR      MAX CAPACITY GB
   <oplog-ID>   <store-path>     <true|false>     <load-factor>    <capacity>

The default output contains a subset of the fields returned by this 
command. For the complete list of |json| fields returned by the 
command, see the :opsmgr:`API reference 
</reference/api/admin/backup/oplog/mongoConfigs/get-one-oplog-configuration-by-id/#response>`.

.. _mcli-om-admin-backup-oplog-describe-egs: 

Example 
-------

The following ``mongocli om admin backup oplog describe`` command 
retrieves the configuration of the oplog store whose ID is ``oplog1``
using the {+default-profile+}, which contains |onprem| credentials. 

.. tabs:: 

   .. tab:: Default Output 
      :tabid: default 

      .. code-block:: shell 

         mongocli om admin backup oplog describe oplog1

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         NAME     URI                         SSL     LOAD FACTOR   MAX CAPACITY GB
         oplog1   mongodb://localhost:27017   false   1             <nil>

   .. tab:: JSON Output 
      :tabid: json 

      .. code-block:: shell 

         mongocli om admin backup oplog describe oplog1 --output json

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         {
           "id": "oplog1",
           "uri": "mongodb://localhost:27017",
           "ssl": false,
           "assignmentEnabled": true,
           "encryptedCredentials": false,
           "loadFactor": 1
         }
