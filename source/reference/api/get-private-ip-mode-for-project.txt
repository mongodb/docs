==================================================
Verify Connect via Peering Only Mode for a Project
==================================================

.. default-domain:: mongodb

.. meta::
   :canonical: https://www.mongodb.com/docs/atlas/reference/api-resources-spec/#tag/Network-Peering

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. note:: Deprecated Feature

   This feature has been deprecated. Use
   :dochub:`Split Horizon connection strings <atlas-horizon-faq>` to
   connect to your cluster. These connection strings allow you to
   connect using both |vpc|/VNet Peering and allowed public IP
   addresses. To learn more about support for Split Horizon, see
   :dochub:`this FAQ <atlas-horizon-faq>`. Use the
   :doc:`Disable Connect via Peering Only Mode for a Project endpoint </reference/api/set-private-ip-mode-for-project>`
   to :ref:`disable Peering Only <disable-peering-mode>`.

Verify whether an |service| project has enabled
:guilabel:`Connect via Peering Only` mode. You must have the
:authrole:`Project Read Only` role to successfully call this endpoint.

.. include:: /includes/api/base-api-uri.rst

Resource
--------

.. code-block:: http

   GET /groups/{GROUP-ID}/privateIpMode

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1
   :widths: 15 10 10 65

   * - Body Parameter
     - Type
     - Necessity
     - Description
   
   * - ``GROUP-ID``
     - string
     - Required
     - Unique identifier of N |service| project for which you want to  enable :guilabel:`Connect via Peering Only`
       mode.

Request Query Parameters
~~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/no-query-single-parameters.rst

Request Body Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/no-body-parameters.rst

Response
--------

.. list-table::
   :header-rows: 1
   :widths: 15 10 75

   * - Response Element
     - Type
     - Description

   * - enabled
     - boolean
     - Indicates whether :guilabel:`Connect via Peering Only` mode is enabled or disabled
       for an |service| project.

Example Request
---------------

.. code-block:: sh
   :linenos:

   curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
        --header "Accept: application/json" \
        --include \
        --request GET "https://cloud.mongodb.com/api/atlas/v1.0/groups/{GROUP-ID}/privateIpMode?pretty=true"
