==========================
Display Legacy UUID Values
==========================

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

You can specify how Compass displays legacy UUID values of binary subtype 3
through the Settings panel.

About This Task
---------------

You can choose one of the following encoding options:

.. list-table::
   :header-rows: 1
   :stub-columns: 1
   :widths: 20 80

   * - **Encoding Option**
     - **Description**

   * - Raw data (no encoding)
     - Displays legacy UUIDs as raw binary data without encoding.

   * - Legacy Java UUID
     - Displays legacy UUIDs using Java's UUID encoding:
       ``LegacyJavaUUID("UUID_STRING")``.

   * - Legacy C# UUID
     - Displays legacy UUIDs using C#'s UUID encoding:
       ``LegacyCSharpUUID("UUID_STRING")``.

   * - Legacy Python UUID
     - Displays legacy UUIDs using Python's UUID encoding:
       ``LegacyPythonUUID("UUID_STRING")``.

Procedure
---------

.. procedure::
   :style: normal

   .. include:: includes/fact-navigate-to-settings.rst

   .. step:: View encoding options
      
      Click the dropdown to display encoding options for displaying legacy UUID
      values.

   .. step:: Select encoding format
      
      Click on the desired encoding format to select how |compass| displays legacy UUIDs.

   .. step:: Click :guilabel:`Save`

.. note::
   Changing this setting only affects how Compass displays legacy UUIDs. It does
   not modify how MongoDB stores the UUIDs themselves.
