.. _mcli-iam-organization-create:

================================
mongocli iam organization create
================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``mongocli iam organization create`` command creates a new 
|cloud-short| or |onprem| organization.

You must have the ``Global Owner`` role to create an organization.

.. _mcli-iam-organization-create-syntax:

Syntax 
------

.. code-block:: text 

   mongocli iam organization|organizations|org|orgs create <organization-name>
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. important::
   
   The service for which the organization is created is determined by 
   your profile.

.. include:: /includes/fact-command-line-help.rst

.. _mcli-iam-organization-create-options:

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type 
     - Description 
     - Required? 

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P`` 
     - string 
     - Name of the profile where your credentials are saved.
       
       You must have credentials with the ``Global Owner`` role to 
       create an organization. You can use either a 
       :opsmgr:`Global API key pair
       </reference/api/global-api-keys/index.html#global-programmatic-api-keys>` or
       a :opsmgr:`Personal API key
       </tutorial/configure-public-api-access/#personal-api-keys-deprecated>`:

       .. list-table::
          :header-rows: 1
          :widths: 50 40 40

          * - Credential Type
            - ``public_api_key`` Value
            - ``private_api_key`` Value

          * - Global API Keys
            - Public Global API key
            - Private Global API key

          * - Personal API Key
            - |cloud-short| or |onprem| username
            - Personal API key

       The profile must include a ``service`` value of either:

       - ``"ops-manager"`` for an |onprem| organization, or
       - ``"cloud-manager"`` for a |cloud-short| organization.

       If omitted, uses the {+default-profile+}. To learn more about 
       creating a profile, see :ref:`mcli-configure`.

     - no

.. _mcli-iam-organization-create-output:

Output
------

The command prints following fields in the specified format to the 
terminal if the command succeeds. If the command returns errors, 
see :ref:`Troubleshooting <mcli-troubleshooting>` for recommended 
solutions.

.. list-table::
   :header-rows: 1
   :widths: 30 70

   * - Field 
     - Description 

   * - ``id``
     - Unique identifier of the organization.

   * - ``links``
     - One or more links to sub-resources and/or related resources.
       The relations between URLs are explained in the `Web Linking
       Specification <http://tools.ietf.org/html/rfc5988>`_.

   * - ``name``
     - Name of the organization.

.. _mcli-iam-organization-create-examples:

Examples
--------

The following command creates an |onprem| organization called 
``myOrg`` using the {+default-profile+}, which contains the 
``Global Owner`` credentials and specifies the |onprem| service:

.. code-block:: text

   mongocli iam organization create myOrg --output json

The previous command creates the |onprem| organization ``myOrg`` and 
prints the following in the specified format to the terminal. For more 
information on these fields, see 
:ref:`Output <mcli-iam-organization-create-output>`.
