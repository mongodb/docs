.. _mcli-om-cluster-describe-command:

=====================================
mongocli ops-manager cluster describe
=====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster describe`` command retrieves details about 
the specified cluster in the specified |mms| project. You 
can also retrieve details about a cluster through the |mms| 
:opsmgr:`API </reference/api/clusters/>`.

.. _om-cluster-describe-syntax: 

Syntax 
------

.. code-block:: text 

   mongocli ops-manager|om cluster describe <cluster-name> 
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _om-cluster-describe-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``<cluster-name>``
     - string 
     - Name of the cluster to retrieve.
     - yes

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable 
       <mcli-env-var>`.
     - no

.. _om-cluster-describe-output:

Output 
------

The command prints the following fields in |json| format to 
the terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` for 
recommended solutions.

.. include:: /includes/mms-config-props.rst

.. _om-cluster-describe-examples:

Examples 
--------

The following examples show two ways to run the command.

.. _om-cluster-describe-example1:

Example 1
~~~~~~~~~

The following ``mongocli om cluster describe`` command 
retrieves details about the cluster named ``myCluster``. It 
uses the {+default-profile+} where the |api| access keys 
and the project ID are stored.

.. code-block:: sh
   :copyable: false

   mongocli ops-manager cluster describe myCluster

.. _om-cluster-describe-example2:

Example 2
~~~~~~~~~

The following ``mongocli om cluster describe`` command 
retrieves the details about the cluster named ``myCluster`` in the 
specified project. It uses the profile ``OMprofile`` to access the 
specified project.

.. code-block:: sh
   :copyable: false

   mongocli om cluster describe myCluster --projectId 5e3392a531cd7c007883d3cb --profile OMprofile
