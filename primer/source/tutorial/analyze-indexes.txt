=================
Analyze Index Use
=================

.. default-domain:: mongodb

Overview
--------

Analyze Query Plan
~~~~~~~~~~~~~~~~~~

MongoDB provides the :method:`~cursor.explain()` method to return
information on the query plan of a particular query. The query plan is
the plan the server uses to find the matches for a query. This
information may be useful when optimizing a query.

.. include:: /includes/steps/getting-started-index-explain.rst

See the :manual:`Explain Output Reference
</reference/method/cursor.explain/#explain-output-fields>` for complete
documentation of all explain fields and the :manual:`Analyze Query
Performance </tutorial/analyze-query-plan>` tutorial for more
information.

Specify Index to Use
~~~~~~~~~~~~~~~~~~~~

The :method:`~cursor.hint()` method allows you to force MongoDB to use
a specific index for a query, bypassing MongoDB's normal query
optimization and index selection process. Pass either the index name or
the index specification document to identify the index to use.

.. include:: /includes/steps/getting-started-index-hint.rst

For more information on MongoDB's query plan selection, see
:manual:`Query Plans </core/query-plans>`. See also
:method:`~cursor.hint()` for more information and examples.
