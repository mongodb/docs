====================================
Cloud Provider Snapshot Restore Jobs
====================================

.. default-domain:: mongodb

.. include:: /includes/fact-projects-groups-synonyms.rst

.. admonition:: Cloud Provider Snapshots Only
   :class: important

   This resource only applies to clusters using
   :ref:`backup-cloud-provider`. 

The ``/backup/restoreJobs`` resource enables you to create or view
restore jobs for an |service| cluster with :ref:`backup-cloud-provider` 
enabled.

Endpoints
---------

.. include:: /includes/api-base-url.rst

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description

   * - ``GET``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/restoreJobs </reference/api/cloud-provider-snapshot-restore-jobs-get-all>`
     - Get all cloud provider snapshot restore jobs for the specified cluster.

   * - ``GET``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/restoreJobs/{JOB-ID} </reference/api/cloud-provider-snapshot-restore-jobs-get-one>`
     - Get one cloud provider snapshot restore jobs for the specified cluster.

   * - ``POST``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/restoreJobs </reference/api/cloud-provider-snapshot-restore-jobs-create-one>`
     - Create a new restore job from a cloud provider snapshot associated to the specified cluster.

       .. include:: /includes/fact-api-whitelist-required.rst

   * - ``DELETE``
     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/backup/restoreJobs/{JOB-ID} </reference/api/cloud-provider-snapshot-restore-jobs-delete-one>`
     - Cancels the cloud provider snapshot manual download restore job associated to ``{JOB-ID}``.

       .. include:: /includes/fact-api-whitelist-required.rst
