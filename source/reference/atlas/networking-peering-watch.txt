=======================================
mongocli atlas networking peering watch
=======================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/styles/corrections.rst

The ``networking peering watch`` command monitors one network peering
connection in the specified project until it becomes available, at which
time it displays a confirmation message.

Syntax 
------

.. code-block:: text 

   mongocli atlas networking peering watch <peering-connection-id>
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _atlas-networking-peering-watch-arguments:

Arguments  
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - ``<peering-connection-id>``
     - string
     - Unique identifier of the network peering connection that you
       want to watch.
     - yes

.. _atlas-networking-peering-watch-options:

Options 
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the network
       peering connection that you want to watch. If omitted, uses the
       project ID in the profile or :ref:`environment variable
       <mcli-env-var>`.
     - no

.. _atlas-networking-peering-watch-output:

Output
------

The ``networking peering watch`` command continuously polls |service|
for the status of the network peering connection until it reaches an
available status.

.. note::

   The {+mcli+} considers ``WAITING_FOR_USER`` and ``FAILED`` to be
   available statuses.

   For more information on network peering connection statuses, see the
   :atlas:`Atlas documentation </reference/api/vpc-get-connection/#response-elements>`.

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   Network peering changes completed.

Example
-------

The following command watches one network peering connection for a
project using the {+default-profile+}, which contains credentials and
the project ID. The output is returned in the default format.

.. code-block:: sh

    mongocli atlas networking peering watch 5f621dc701240c5b7c3a888e

The previous command prints the following to the terminal.

.. code-block:: sh
   :copyable: false

   Network peering changes completed.
