===========================
Create a Log Collection Job
===========================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

.. include:: /includes/fact-groups-projects-synonyms.rst

When you create a log collection job, |mms| starts a background job to
download the logs from the specified |mms| deployment. Use this
endpoint to create a new log collection job.

.. include:: /includes/api/base-api-uri.rst

Resource
--------

.. code-block:: http
   :copyable: false

   POST /groups/{PROJECT-ID}/logCollectionJobs

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 15 10 75
   :header-rows: 1
   :stub-columns: 1

   * - Name
     - Type
     - Description

   * - ``PROJECT-ID``
     - string
     - Unique identifier of the :term:`project` for which to create a
       log collection job.

Request Query Parameters
~~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/no-query-single-parameters.rst

Request Body Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 15 10 10 65
   :header-rows: 1
   :stub-columns: 1

   * - Name
     - Type
     - Necessity
     - Description

   * - ``resourceType``
     - string
     - Required
     - Type of resource from which to collect logs. Must be one of the
       following values:

       - ``CLUSTER``, for a sharded cluster.
       - ``PROCESS``, for a node in the replica set.
       - ``REPLICA_SET``, for a replica set.`

   * - ``resourceName``
     - string
     - Required
     - Name of the resource from which to collect logs.
       The value is based on the resource type:
       
       - For the ``CLUSTER`` ``resourceType``, the value is the name of
         the |service| deployment, or the ``CLUSTER-ID``.
         For example, ``atlas-my-deployment``.
         To obtain it, use the
         :doc:`/groups/{PROJECT-ID}/clusters/{CLUSTER-ID} endpoint</reference/api/clusters/clusters-get-one>`.
         
       - For the ``PROCESS`` ``resourceType``, the value is
         the name of the replica set followed by the node name.
         For example, ``Cluster0-shard-1-node-0``, or
         ``atlas-123abc-shard-0-node-1``.
         
         .. note::

            To obtain the name of the replica set, the list of
            nodes, and other information, access the cluster and run
            :method:`rs.conf()._id <rs.conf>`
            and :method:`rs.status() <rs.status>`.

       - For the ``REPLICA_SET`` ``resourceType``, the value is
         the name of the replica set in the cluster followed
         by the shard name. For example, ``atlas-123abc-shard-0``.

   * - ``sizeRequestedPerFileBytes``
     - number
     - Required
     - Size for each log file in bytes.

   * - ``logTypes``
     - array
     - Required
     - Array of strings specifying the types of logs to collect. Each
       array element must be one of the following values:

       - ``AUTOMATION_AGENT``
       - ``BACKUP_AGENT``
       - ``MONITORING_AGENT``
       - ``MONGODB``
       - ``FTDC`` 
         
   * - ``redacted``
     - boolean
     - Optional
     - If set to ``true``, emails, hostnames, IP addresses, and
       namespaces in API responses involving this job are replaced with
       random string values.

Response
--------

.. list-table::
   :widths: 10 10 60
   :header-rows: 1
   :stub-columns: 1

   * - Name
     - Type
     - Description

   * - ``id``
     - string
     - Unique identifier of the created log collection job.

Example Request
---------------

.. cond:: cloud

   .. code-block:: sh
      :linenos:

      curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
       --header "Accept: application/json" \
       --header "Content-Type: application/json" \
       --include \
       --request POST "https://{+cloudmgr-url+}/api/public/v1.0/groups/{PROJECT-ID}/logCollectionJobs?pretty=true"
       --data '
         {
           "resourceType": "PROCESS",
           "resourceName": "my_deployment_1",
           "redacted": true,
           "sizeRequestedPerFileBytes": 10000000,
           "logTypes": [
               "FTDC",
               "MONGODB",
               "AUTOMATION_AGENT"
           ]
         }'

.. cond:: onprem

   .. code-block:: sh
      :linenos:

      curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
       --header "Accept: application/json" \
       --header "Content-Type: application/json" \
       --include \
       --request POST "https://{+opsmgr-url+}/api/public/v1.0/groups/{PROJECT-ID}/logCollectionJobs?pretty=true"
       --data '
         {
           "resourceType": "PROCESS",
           "resourceName": "my_deployment_1",
           "redacted": true,
           "sizeRequestedPerFileBytes": 10000000,
           "logTypes": [
               "FTDC",
               "MONGODB",
               "AUTOMATION_AGENT"
           ]
         }'
