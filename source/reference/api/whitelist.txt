=========
Whitelist
=========

.. default-domain:: mongodb

The ``whitelist`` resource lets you retrieve and update a user's
:ref:`whitelist <whitelist-for-api-operations>`, which controls the user's
access to whitelisted API operations. Users can issue whitelisted API
operations only from addresses on their own whitelists. To use the 
``whitelist`` resource you must know the ID of the user who owns
the whitelist. To retrieve a user's ID, see :ref:`api-get-all-users-in-group`.

The ``whitelist`` resource's ``POST`` and ``DELETE`` operations are themselves
whitelisted. They must originate from an address already on the user's whitelist.

You can access whitelist information only for the user issuing the request.
The ``USER-ID`` in the URL *must* match the ID of the user associated with the
:term:`Public API key` used to send the request.

Endpoints
---------

The following endpoints are available for ``hosts``.

.. list-table::
   :header-rows: 1
   :widths: 10 60 30

   * - Method
     - Endpoint
     - Description
   
   * - ``GET``
     - :doc:`/api/public/v1.0/users/{USER-ID}/whitelist </reference/api/whitelist-get-for-current-user>`
     - Retrieves the whitelist for the current user.

   * - ``GET``
     - :doc:`/api/public/v1.0/users/{USER-ID}/whitelist/{IP-ADDRESS} </reference/api/whitelist-get-for-ip-address>`
     - Retrieves the whitelist for the specified IP address.

   * - ``POST``
     - :doc:`/api/public/v1.0/users/{USER-ID}/whitelist/{IP-ADDRESS} </reference/api/whitelist-add-entries>`
     - Adds entries to the whitelist for the current user.

   * - ``DELETE``
     - :doc:`/api/public/v1.0/users/{USER-ID}/whitelist/{IP-ADDRESS} </reference/api/whitelist-delete-entry>`
     - Deletes an entry from the whitelist for the current user.
