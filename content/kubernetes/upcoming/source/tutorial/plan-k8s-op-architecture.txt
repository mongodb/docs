=============================================================
|k8s-op-full| Architecture Overview
=============================================================

.. meta::
   :description: Architecture guide for the MongoDB Controllers for Kubernetes Operator, covering the operator itself, MongoDB database resources, Ops Manager resources, reconciliation, multi-cluster deployments, and networking.

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

The |k8s-op-full| is a |k8s| controller that extends the
|k8s| API with custom resource definitions (CRDs) for MongoDB
database deployments and |onprem|. Understanding how these custom
resources map to |k8s| primitives, how the |k8s-op-short| reconciles
changes, and how components connect to each other is essential
before you plan your first deployment.

This section covers the architecture of every resource the
|k8s-op-short| manages, from a single-cluster standalone deployment
through a multi-cluster |onprem| topology with backup infrastructure. 

.. _arch-operator-overview:

Operator Overview
-----------------

The |k8s-op-short| runs as a Deployment in a dedicated namespace. It
watches for custom resources of type ``MongoDB``, ``MongoDBOpsManager``,
``MongoDBUser``, and ``MongoDBMultiCluster``. When you create or update
one of these resources, the |k8s-op-short| reads the specification,
validates it, and reconciles the |k8s| objects (StatefulSets, Services,
Secrets, ConfigMaps) needed to bring the actual state in line with your
declared state.

The |k8s-op-short| provides a container image for the {+mdbagent+} in
|onprem|, which allows you to manage and deploy MongoDB database
clusters with full monitoring, backups, and automation. The |k8s|
container serves as a host on which |onprem| orchestrates the
installation of ``mongod`` processes and deploys the cluster
configuration.

As part of deployment, the |k8s-op-short| creates PersistentVolumes
for the StatefulSets. The containers use these PersistentVolumes to
maintain cluster state between restarts.

The following diagram illustrates the flow of configuration for a
|k8s| instance using the |k8s-op-short|:

.. figure:: /images/kubernetes-operator.bakedsvg.svg
   :alt: Diagram showing the flow of configuration for a Kubernetes
         instance using the Kubernetes Operator.
   :figwidth: 700px
   :lightbox:

The |k8s-op-short|'s architecture consists of two primary resource
families:

- **The** ``MongoDBOpsManager`` **custom resource**, through which the
  |k8s-op-short| deploys |onprem| components: the {+appdb+}, the
  |onprem| application, and the Backup Daemon. After the deployment is
  operational, the |onprem| components reconcile updates that you make
  to the MongoDB cluster configuration. To learn more, see
  :ref:`meko-om-arch`.

- **The** ``MongoDB`` **custom resource**, through which the
  |k8s-op-short| deploys database instances (standalone, replica set,
  or sharded cluster) and the ``MongoDBUser`` resources. After
  deployment, these resources reconcile updates you make to the user
  or MongoDB cluster configuration. To learn more, see
  :ref:`mdb-resources-arch`.
