====================================
mongocli atlas privateendpoint watch
====================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/styles/corrections.rst

The ``privateendpoint watch`` command monitors one private endpoint
connection in the specified project until it becomes available, at which
time it displays a confirmation message.

Syntax 
------

.. code-block:: text 

   mongocli atlas privateendpoint(s)|privateEndpoint(s) watch <private-endpoint-id>
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

.. include:: /includes/fact-command-line-help.rst

.. _atlas-privateendpoint-watch-arguments:

Arguments  
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - ``<private-endpoint-id>``
     - string
     - Unique identifier of the private endpoint connection that you
       want to watch.
     - yes

.. _atlas-privateendpoint-watch-options:

Options 
-------

.. list-table::
   :widths: 20 10 60 10

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the private
       endpoint connection that you want to watch. If omitted, uses the
       project ID in the profile or :ref:`environment variable
       <mcli-env-var>`.
     - no

.. _atlas-privateendpoint-watch-output:

Output
------

The ``privateendpoint watch`` command continuously polls |service|
for the status of one private endpoint connection until it reaches an
available status.

.. note::

   The {+mcli+} considers ``WAITING_FOR_USER`` to be an available
   status.

   For more information on private endpoint connection statuses, see the
   :atlas:`Atlas documentation </reference/api/private-endpoint-get-one-private-endpoint-connection/#response-elements>`.

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   Private endpoint changes completed.

Example
-------

The following command watches one private endpoint connection for a
project using the {+default-profile+}, which contains credentials and
the project ID. The output is returned in the default format.

.. code-block:: sh

    mongocli atlas privateendpoint watch 5f4fc14da2b47835a58c63a2

The previous command prints the following to the terminal.

.. code-block:: sh
   :copyable: false

   Private endpoint changes completed.
