.. _mcli-om-cluster-apply-command:

==============================
mcli ops-manager cluster apply
==============================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``cluster apply`` command creates the cluster in the 
specified |mms| project if the cluster does not already exist or 
updates the cluster in the specified |mms| project if 
the cluster already exists. You can also: 

- Create a cluster through the |mms| :opsmgr:`UI 
  </tutorial/deploy-replica-set/>`,  :ref:`CLI 
  <mcli-om-cluster-create-command>`, and :opsmgr:`Automation 
  Configuration API </reference/api/automation-config/>`,

- Update a cluster through the the |mms| :opsmgr:`UI 
  </tutorial/edit-replica-set/>`,  :ref:`CLI 
  <mcli-cm-cluster-update-command>`, and :opsmgr:`Automation 
  Configuration API </reference/api/automation-config/>`.

.. _om-cluster-apply-syntax:

Syntax 
------

.. code-block:: text 
   :copyable: false 

   mcli ops-manager|om cluster apply 
        -f|--file <om-settings-file> 
        [ -p|--profile <profile-name> ] 
        [ --projectId <project-ID> ]

.. _om-cluster-apply-options: 

Options 
-------

.. list-table:: 
   :header-rows: 1 
   :widths: 20 10 60 10 

   * - Option 
     - Type 
     - Description 
     - Required?

   * - ``-f``, ``--file`` 
     - string 
     - The path to the file that contains the settings for 
       creating the cluster. To learn more about the 
       configuration properties to specify in the file, see 
       :ref:`mms-cluster-settings-file`.
     - yes

   * - ``-p``, ``--profile``
     - string
     - The name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       ``default`` profile. To learn more about creating a 
       profile, see :ref:`configure-mcli`.
     - no

   * - ``--projectId``
     - string
     - The unique identifier of the project that contains the 
       cluster to update. If omitted, uses the project ID in 
       the profile or the :ref:`environment variable <mcli-env-var>`.
     - no

.. _om-cluster-apply-output:

Output 
------

The command prints output similar to the following to the 
terminal if the command succeeds. If the command returns 
errors, see :ref:`Troubleshooting <troubleshooting>` 
for recommended solutions.

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://<hostname>:<port>/v2/<projectId>#deployment/topology for status

where: 

- ``<hostname>:<port>`` is the |mms| base URL
- ``projectId`` is the unique identifier of the project 

.. _om-cluster-apply-examples:

Examples 
--------

The following examples show two ways to run the command.

.. _om-cluster-apply-example1:

Example 1
~~~~~~~~~

The following ``mcli om cluster apply`` command 
deploys the settings in the file named ``om-cluster.yaml`` to the specified 
project. The command uses a custom profile named ``egOMprofile`` for the 
|api| access keys for the project.

.. code-block:: sh 
   :copyable: false

   mcli ops-manager cluster apply --file om-cluster.yaml --projectId 5e33a17031cd7c007883e20d --profile egOMprofile

.. _om-cluster-apply-example2:

Example 2
~~~~~~~~~

The following ``mcli ops-manager cluster apply`` command 
deploys the settings in the file named ``om-cluster.json`` to the cluster 
named ``myCluster``. It uses the ``default`` profile for the |api| access 
keys and project ID. 

.. code-block:: sh 
   :copyable: false

   mcli ops-manager cluster apply --file om-cluster.json 

Output
~~~~~~

The :ref:`om-cluster-apply-example1` and :ref:`om-cluster-apply-example2` 
commands print the following to the terminal:

.. code-block:: text 
   :copyable: false

   Changes are being applied, please check http://localhost:30700/v2/5e33a17031cd7c007883e20d#deployment/topology for status
