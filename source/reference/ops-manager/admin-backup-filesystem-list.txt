.. _mcli-om-admin-backup-filesystem-list-cmd:

=================================================
mongocli ops-manager admin backup filesystem list
=================================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. include:: /includes/styles/corrections.rst

The ``admin backup filesystem list`` command retrieves the 
configurations of all file system stores. You can also 
retrieve the configurations of all file system stores using 
the :opsmgr:`API </reference/api/admin/backup/snapshot/fileSystemConfigs/get-all-file-system-store-configurations/>`.

.. _mcli-om-admin-backup-filesystem-list-syntax: 

Syntax 
------

.. code-block:: shell 

   mongocli ops-manager admin backup |om filesystem|fileSystem list 
        [ --limit <items-per-page> ]
        [ --page <page-number> ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. _mcli-om-admin-backup-filesystem-list-options: 

Options 
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--limit``
     - integer
     - Number of items to return per page, up to a maximum of 500.
       Defaults to ``100`` if not specified.
     - no

   * - ``--page``
     - integer
     - Page number (1-based). Defaults to ``1`` if not specified.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for accessing |onprem| are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

.. _mcli-om-admin-backup-filesystem-list-output: 

Output 
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <mcli-troubleshooting>` for recommended solutions.

.. code-block:: sh 
   :copyable: false 

   ID                 PATH             WIREDTIGER   MMAPV1
   <file-system-ID>   <store-path>     NONE|GZIP    NONE|GZIP

The default output contains a subset of the fields returned by this 
command. For the complete list of |json| fields returned by the 
command, see the :opsmgr:`API reference 
</reference/api/admin/backup/snapshot/fileSystemConfigs/get-all-file-system-store-configurations/#response>`.

.. _mcli-om-admin-backup-filesystem-list-egs: 

Example 
-------

The following ``mongocli om admin backup filesystem list`` command 
retrieves the configurations of all file systems using the 
{+default-profile+}, which contains |onprem| credentials. 

.. tabs:: 

   .. tab:: Default Output 
      :tabid: default 

      .. code-block:: shell 

         mongocli om admin backup filesystem list

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         ID     PATH   WIREDTIGER   MMAPV1
         test   /      NONE         GZIP

   .. tab:: JSON Output 
      :tabid: json 

      .. code-block:: shell 

         mongocli om admin backup filesystem list --output json

      The previous command returns the following: 

      .. code-block:: shell 
         :copyable: false 

         {
           "links": [
             {
               "rel": "self",
               "href": "http://localhost:8080/api/public/v1.0/admin/backup/snapshot/fileSystemConfigs?pageNum=1\u0026itemsPerPage=100"
             }
           ],
           "results": [
             {
               "id": "test",
               "loadFactor": 1,
               "mmapv1CompressionSetting": "GZIP",
               "storePath": "/",
               "wtCompressionSetting": "NONE",
               "assignmentEnabled": true
             }
           ],
           "totalCount": 1
         }
