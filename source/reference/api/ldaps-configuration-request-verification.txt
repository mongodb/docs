============================
Verify an LDAP Configuration
============================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

Requests a verification of an |ldap| configuration over |tls| for an
|service| project. Pass the ``requestId`` in the response object to the
:doc:`Verify LDAP Configuration </reference/api/ldaps-configuration-verification-status>`
endpoint to get the status of a verification request. |service|
retains only the most recent request for each project.

.. note::

   - .. include:: /includes/facts/explain-ldap.rst

   - .. include:: /includes/facts/projects-groups-synonyms.rst

   - This endpoint does not verify the ``ldap.userToDNMapping``
     document array. To verify that users can authenticate with this
     parameter, use the
     :manual:`mongoldap </reference/program/mongoldap>` package
     component bundled with MongoDB Enterprise 3.6 or later with a
     config file that includes the same |ldap| parameters that you
     specify for |service|.

.. include:: /includes/api-base-url.rst

Syntax
------

.. code-block:: none

   POST /groups/{GROUP-ID}/userSecurity/ldap/verify

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 15 10 10 65
   :header-rows: 1

   * - Parameter
     - Type
     - Necessity
     - Description

   * - ``GROUP-ID``
     - string
     - Required
     - Unique identifier for the |service| project associated with the
       |ldap| over |tls| configuration.

Example Request
---------------

The following example requests verification of an LDAP configuration.

.. code-block:: shell
   :linenos:
   :emphasize-lines: 6-13

   curl --include --user "{PUBLIC-KEY}:{PRIVATE-KEY}" \
        --header "Accept: application/json" \
        --header "Content-Type: application/json" \
        --digest \
        --request POST "{+service-api-v1-base-uri+}/groups/{PROJECT-ID}/userSecurity/ldap/verify?pretty=true" \
        --data '
          {
            "hostname":"atlas-ldaps-01.ldap.myteam.com",
            "port": 636,
            "bindUsername":"N=Administrator,CN=Users,DC=atlas-ldaps-01,DC=myteam,DC=com",
            "bindPassword":"MyldapPassWord",
            "authzQueryTemplate": "{USER}?memberOfGroup?base"
          }'

Example Response
----------------

The following response indicates that the request for verification for
the LDAP configuration is pending.

.. code-block:: json
   :copyable: false
   :linenos:
   :emphasize-lines: 8-10

   {
     "groupId" : "{PROJECT-ID}",
     "links" : [ {
        "href" : "{+service-api-v1-base-uri+}/groups/{PROJECT-ID}/userSecurity/ldap/verify/{REQUEST-ID}",
        "rel" : "self"
     } ],
     "request" : {
       "bindUsername" : "CN=Administrator,CN=Users,DC=atlas-ldaps-01,DC=myteam,DC=com",
       "hostname" : "atlas-ldaps-01.ldap.myteam.com",
       "port" : 636
     },
     "requestId" : "{REQUEST-ID}",
     "status" : "PENDING",
     "validations" : [ ],
   }
