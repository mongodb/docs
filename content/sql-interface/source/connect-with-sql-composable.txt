==============================
{+sql-interface+} Server Setup
==============================

.. meta::
   :keywords: connect, sql, mongodb
   :description: Configure the MongoDB SQL Interface for Atlas or self-managed EA. Includes steps for driver selection and installing the SQL Schema Builder.
.. default-domain:: mongodb

.. facet::
   :name: genre
   :values: tutorial

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can use the MongoDB {+sql-interface+} with {+service+} {+fdi+}s or self-managed 
{+ea+} databases.
Choose your deployment type to get started: 

.. composable-tutorial::
   :options: deployment-type
   :defaults: atlas

   Prerequisites
   -------------

   - A :ref:`MongoDB database user <mongodb-users>` with which to connect.

   Procedure
   ---------

   .. selected-content::
      :selections: atlas

      .. procedure::
         :style: normal

         .. include:: /includes/nav/steps-db-deployments-page.rst

         .. step:: On the cluster card, click :guilabel:`Connect`.   
            
            A connection dialog box opens.

         .. step:: In the connection dialog box, click :guilabel:`Atlas SQL`.

            If this is your first time, leave
            the :guilabel:`Quick Start` option selected. {+service+} automatically
            creates and manages a {+fdi+} in the background to
            support the SQL connection. The {+fdi+} uses {+aws+} as the cloud provider.
            You cannot modify this {+fdi+}.

            The :guilabel:`Advanced` option lets you specify the :ref:`data store <config-adf>` 
            and cloud provider, but you'll need to create and manage the {+fdi+} yourself.
            To learn more, see :ref:`gst-deploy-adf`.
             
         .. step:: Select your driver.

            From the :guilabel:`Select your driver` dropdown, select the
            driver that matches your |bi| tool.

            .. include:: /includes/table-driver-bi-tool.rst

         .. step:: Copy connection string.

            a. Select a database from the :guilabel:`Database` dropdown.

            #. Click :icon-lg:`Copy` to copy the connection string. You'll need 
               this when you connect your |bi| tool.

            #. Click :guilabel:`Done`. 

   .. selected-content::
      :selections: self

      For local or self-managed deployments, you must generate a schema by 
      downloading and running the {+schema-builder+}. To learn more, see `the {+schema-builder+} README <https://translators-connectors-releases.s3.amazonaws.com/mongodb-schema-manager/docs/MongoDB_Schema_Manager-overview.pdf>`__.

      .. procedure::
         :style: normal

         .. step:: Download the {+schema-builder+}.

            Download the {+schema-builder+} for your operating system from the `MongoDB download center <https://www.mongodb.com/try/download/sql-schema-builder>`__.

         .. step:: Run the {+schema-builder+}.
            
            .. tabs::

               .. tab:: Linux/MacOS
                  :tabid: linux-macos

                  a. Move the downloaded file to a directory of your choice and rename to ``mongodb-schema-manager``.

                  #. Go to the directory where you placed the file and run the following command to make the 
                     {+schema-builder+} executable:

                     .. code-block:: bash

                        chmod +x mongodb-schema-manager

                  #. Run the {+schema-builder+} using a terminal command with your 
                     connection string like the following example. This command generates 
                     or regenerates the schema for your collections within the database.

                     .. code-block:: bash

                        ./mongodb-schema-manager --uri 'mongodb://<db_username>:<db_password>@<host>:<port>/<database>?authSource=admin'

                  #. Look for the ``__sql_schemas`` collection in your database 
                     to verify the created SQL schema. Inside this collection, there will be a document for each
                     collection with its schema map and data type assignments.

               .. tab:: Windows
                  :tabid: windows

                  a. Move the downloaded file to a directory of your choice and 
                     rename to ``mongodb-schema-manager.exe``.

                  #. Run the {+schema-builder+} using a terminal command with your 
                     connection string like the example below. This command generates 
                     or regenerates the schema for your collections within the database.

                     .. code-block:: bash

                        ./mongodb-schema-manager.exe --uri 'mongodb://<db_username>:<db_password>@<host>:<port>/<database>?authSource=admin'

                  #. Verify the created SQL schema by looking for the ``__sql_schemas`` collection
                     in your database. Inside this collection, there will be a document for each
                     collection with its schema map and data type assignments.
