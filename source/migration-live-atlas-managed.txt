.. _live-migration:

=======================================
Live Migrate: Pull Your Data into Atlas
=======================================

.. default-domain:: mongodb

.. meta::
   :keywords: live migration (pull) to MongoDB Atlas

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

You can pull data from existing MongoDB deployments, ``JSON``, or ``CSV``
files into |service| using one of the following tools managed by |service|
within the |service| user interface.

To locate live migration (pull) in the {+atlas-ui+}, in the left-side
panel of your organization's page, go to :guilabel:`Live Migration to Atlas`
and select :guilabel:`General Live Migration`.

.. list-table::
   :header-rows: 1
   :widths: 35 65

   * - Method
     - Description

   * - :doc:`Live migrate (pull) a replica set </import/live-import>`

     - Use the |service| user interface to pull a MongoDB replica set
       into an |service| replica set cluster.

   * - :doc:`Live migrate (pull) a sharded cluster </import/live-import-sharded>`

     - Use the |service| user interface to pull a MongoDB sharded cluster
       into an |service| sharded cluster.

   * - Live migrate a standalone MongoDB node

     - :manual:`Convert </tutorial/convert-standalone-to-replica-set>`
       the standalone MongoDB node to a single-node replica set and
       then :doc:`live migrate (pull) </import/live-import>` it into
       |service|. If you can't convert the standalone to a replica set,
       use :doc:`mongorestore </import/mongorestore>` to migrate to |service|.
