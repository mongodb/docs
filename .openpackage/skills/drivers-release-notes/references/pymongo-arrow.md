# PyMongoArrow

**Add for:** major, minor, and patch releases

| Variable | Value |
|---|---|
| `GITHUB_REPO` | `mongodb-labs/mongo-arrow` |
| `RELEASE_NOTES_FILE` | `content/pymongo-arrow-driver/upcoming/source/reference/release-notes.txt` |
| `CHANGELOG_LINK` | `mongodb-labs/mongo-arrow/releases/tag/{FULL_VERSION}` |
| `UPGRADE_REF` | n/a — no upgrade guide |

- Section name: `Changes in Version {VERSION}` (not `What's New in {VERSION}`)
- Breaking changes: no upgrade guide. Use the **PyMongoArrow: Breaking Changes
  Admonition** template from `references/docs-templates.md` immediately
  following the `Changes in Version {VERSION}` header.
