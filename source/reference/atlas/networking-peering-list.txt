.. _mcli-atlas-networking-peering-list-cmd:

======================================
mongocli atlas networking peering list
======================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``networking peering list`` command retrieves information about all
network peering connections for a given |service| project.

Syntax 
------

.. code-block:: text 

   mongocli atlas networking peering list|ls
        [ --output|-o <output-format> ]
        [ --page <page-number> ]
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]
        [ --providerName <cloud-provider-name> ]

.. include:: /includes/fact-command-line-help.rst

.. _atlas-networking-peering-list-options:

Options 
-------

.. list-table::
   :widths: 20 10 60 10
   :header-rows: 1

   * - Option 
     - Type
     - Description
     - Required?

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--page``
     - integer
     - Page number.
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the network
       peering connections that you want to retrieve. If omitted, uses
       the project ID in the profile or :ref:`environment variable
       <mcli-env-var>`.
     - no

   * - ``--providerName``
     - string
     - Cloud provider for this VPC peering connection. Accepted values
       are:

       - ``AWS``
       - ``Azure``
       - ``GCP``.

       If omitted, defaults to ``AWS``.

     - no

.. _atlas-networking-peering-list-output:

Output
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see
:ref:`Troubleshooting <troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   ID                          STATUS                          CONTAINER ID
   <peering-connection-1-id>   <peering-connection-1-status>   <network-container-1-id>
   <peering-connection-2-id>   <peering-connection-2-status>   <network-container-2-id>

.. include:: /includes/fact-default-output.rst

- :atlas:`Atlas API </reference/api/vpc-get-connections-list/#response-elements>`

Example
-------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command retrieves network peering connections for a
      project using the {+default-profile+}, which contains credentials
      and the project ID. The output is returned in the default format.

      .. code-block:: sh

         mongocli atlas networking peering list

      The previous command prints the following to the terminal.

      .. code-block:: sh
         :copyable: false

         ID                         STATUS      CONTAINER ID
         5f60c6040948295c09356649   AVAILABLE   5f4eb2178c9ff67e809a3619

   .. tab:: JSON Output
      :tabid: json-output

      The following command retrieves network peering connections for a
      project using the {+default-profile+}, which contains credentials
      and the project ID. The output is returned in |json| format.

      .. code-block:: sh

         mongocli atlas networking peering list --output json

      The previous command prints the following to the terminal in
      |json| format.

      .. code-block:: json
         :copyable: false

         [
           {
             "awsAccountId": "<aws-account-id>",
             "connectionId": "<aws-peering-connection-id>",
             "containerId": "5f4eb2178c9ff67e809a3619",
             "id": "5f60c6040948295c09356649",
             "routeTableCidrBlock": "10.0.0.0/24",
             "statusName": "AVAILABLE",
             "vpcId": "<aws-vpc-id>"
           }
         ]
