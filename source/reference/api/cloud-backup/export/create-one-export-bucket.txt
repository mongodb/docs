.. _create-backup-export-bucket:

=================================
Create One Snapshot Export Bucket 
=================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 2
   :class: singlecol

The ``exportBuckets`` resource allows you to grant |service| access 
to the specified bucket for exporting backup snapshots.

.. include:: /includes/api-requirements.rst

.. include:: /includes/fact-projects-groups-synonyms.rst

.. _create-backup-export-bucket-resource:

Resource
--------

.. include:: /includes/api-base-url-new.rst

.. code-block:: none

   POST /groups/{GROUP-ID}/backup/exportBuckets

.. _create-backup-export-bucket-path-params:

Request Path Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 20 20 60
   :header-rows: 1
   :stub-columns: 1

   * - Parameter
     - Type
     - Description

   * - ``GROUP-ID``
     - string
     - Unique 24-hexadecimal digit string that identifies the 
       :ref:`project <group-id>` for which to grant access to the 
       |s3| bucket.

.. _create-backup-export-bucket-query-params:

Request Query Parameters
~~~~~~~~~~~~~~~~~~~~~~~~

.. include:: /includes/api/no-query-single-parameters.rst

.. _create-backup-export-bucket-body-params:

Request Body Parameters
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :widths: 20 20 10 50
   :header-rows: 1
   :stub-columns: 1

   * - Parameter
     - Type
     - Necessity
     - Description

   * - ``bucketName``
     - string
     - Required
     - Name of the bucket that the provided role ID is authorized to 
       access. You must also specify the ``iamRoleId``.

   * - ``cloudProvider``
     - string
     - Required
     - Name of the provider of the cloud service where |service| can 
       access the |s3| bucket. |service| only supports ``AWS``.

   * - ``iamRoleId``
     - string
     - Required
     - Unique identifier of the role that |service| can use to access 
       the bucket. If necessary, use the :ref:`UI <manage-iam-roles>` 
       or :doc:`API </reference/api/cloud-provider-access-get-roles/>` 
       to retrieve the role ID. You must also specify the 
       ``bucketName``.

.. _create-backup-export-bucket-response:

Response
--------

The HTTP document contains the following elements:

.. list-table::
   :widths: 20 20 60
   :header-rows: 1
   :stub-columns: 1

   * - Parameter
     - Type
     - Description

   * - ``id``
     - string 
     - Unique identifier of the |s3| bucket.

   * - ``bucketName``
     - string 
     - Name of the bucket that the role ID is authorized to access. 

   * - ``cloudProvider``
     - string
     - Name of the provider of the cloud service where |service| can 
       access the |s3| bucket. |service| only supports ``AWS``.

   * - ``iamRoleId``
     - string
     - Unique identifier of the role that |service| uses to access the 
       bucket.

   * - ``projectId``
     - string 
     - Unique 24-hexadecimal digit string identifying the project.

.. _create-backup-export-bucket-eg:

Example Request
---------------

.. code-block:: shell

   curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" -X POST --digest \
        --header "Accept: application/json" \
        --header "Content-Type: application/json" \
        "https://cloud.mongodb.com/api/atlas/v1.0/groups/{GROUP-ID}/backup/exportBuckets?pretty=true" \
        --data '{
          "iamRoleId": "12345678f901a234dbdb00ca",
          "bucketName": "example-bucket",
          "cloudProvider": "AWS"
        }'

Example Response
----------------

.. code-block:: json
   :copyable: false

   {
     "_id": "{BUCKET-ID}",
     "bucketName": "example-bucket",
     "cloudProvider": "AWS",
     "iamRoleId": "12345678f901a234dbdb00ca",
     "projectId": "{PROJECT-ID}"
   }
