========
Clusters
========

.. default-domain:: mongodb


.. include:: /includes/fact-projects-groups-synonyms.rst

The ``clusters`` resource provides access to your cluster
configurations. The resource lets you create, edit and delete clusters.
The resource requires your
:ref:`project-id`.

.. important::

   Changes to cluster configurations can affect costs. Before making
   changes, please see :doc:`/billing`.

Endpoints
---------

.. list-table::
   :header-rows: 1
   :widths: 10 40 50

   * - Method
     - Endpoint
     - Description

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/clusters </reference/api/clusters-get-all>`

     - Get all clusters for the specified project.

       For multi-cloud clusters, use
       :doc:`/reference/api/cluster-advanced/get-all-cluster-advanced`.

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME} </reference/api/clusters-get-one>`

     - Get a cluster in the specified project.

       For multi-cloud clusters, use
       :doc:`/reference/api/cluster-advanced/get-one-cluster-advanced`.

   * - ``GET``

     - :doc:`/clusters </reference/api/clusters-get-all-key>`

     - Get details for all clusters in all projects available to the
       programmatic or personal API key making the request.

       .. include:: /includes/api/facts/no-multi-cloud-support-list.rst

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/processArgs </reference/api/clusters-get-advanced-configuration-options>`

     - Get the advanced configuration options for the cluster in the
       specified project.

   * - ``POST``

     - :doc:`/groups/{GROUP-ID}/clusters </reference/api/clusters-create-one>`

     - Create a cluster in the specified project.

       For multi-cloud clusters, use
       :doc:`/reference/api/cluster-advanced/create-one-cluster-advanced`.

   * - ``PATCH``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME} </reference/api/clusters-modify-one>`

     - Update a cluster in the specified project.

       For multi-cloud clusters, use
       :doc:`/reference/api/cluster-advanced/modify-one-cluster-advanced`.

   * - ``PATCH``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/processArgs </reference/api/clusters-modify-advanced-configuration-options>`

     - Set one or more advanced configuration options for the cluster
       in the specified project.

   * - ``DELETE``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME} </reference/api/clusters-delete-one>`

     - Delete a cluster in the specified project.

   * - ``POST``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/restartPrimaries </reference/api/clusters-test-failover>`

     - Test failure of a :term:`primary` replica set member to verify
       your application handles a replica set failover.

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/clusters/{CLUSTER-NAME}/status </reference/api/clusters-check-operation-status>`

     - Check if |service| has completed adding a user to, or 
       removing a user from, your cluster.

   * - ``POST``
     - :doc:`/groups/{GROUP-ID}/sampleDatasetLoad/{CLUSTER-NAME} </reference/api/cluster/load-dataset>`
     - Load the sample dataset into your cluster.

   * - ``GET``
     - :doc:`/groups/{GROUP-ID}/sampleDatasetLoad/{SAMPLE-DATASET-ID} </reference/api/cluster/check-dataset-status>`
     - Check the status of the dataset loading.

   * - ``GET``

     - :doc:`/groups/{GROUP-ID}/clusters/provider/regions </reference/api/cluster-get-regions>`

     - Get the cloud providers and regions available for a cluster in
       the specified project.
