.. _mcli-iam-global-apikey-describe:

==================================
mongocli iam globalApiKey describe
==================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``iam globalApiKey describe`` command provides details about
the specified Global |api| Key. You can also retrieve details about 
the key using the |onprem| :opsmgr:`API 
</reference/api/api-keys/global/get-one-global-api-key/>`.

.. _mcli-iam-global-apikey-describe-syntax:

Syntax 
------

.. code-block:: text 

   mongocli iam globalApiKey|globalApiKeys describe <api-key-id>
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]

.. include:: /includes/fact-command-line-help.rst

.. _mcli-iam-global-apikey-describe-arguments:

Arguments  
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - ``<api-key-id>``
     - string
     - Unique identifier of the Global API Key to describe.
     - yes

.. _mcli-iam-global-apikey-describe-options:

Options 
-------

.. list-table:: 
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type 
     - Description 
     - Required? 

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P`` 
     - string 
     - Name of the profile where your credentials are saved. If omitted, 
       uses the {+default-profile+}. To learn more about 
       creating a profile, see :ref:`mcli-configure`. 

       You must have credentials with the :authrole:`Global Read Only` role 
       to retrieve details about a Global |api| Key. 
     - no

.. _mcli-iam-global-apikey-describe-output:

Output
------

If the command succeeds, it returns the following output in the default
format. If the command returns errors, see :ref:`Troubleshooting
<troubleshooting>` for recommended solutions.

.. code-block:: sh
   :copyable: false

   ID         DESCRIPTION          PUBLIC KEY     PRIVATE KEY
   <key-id>   <key-description>    <public-key>   <private-key>

For the complete list of |json| fields returned by the command, see the 
:opsmgr:`API reference 
</reference/api/api-keys/global/get-one-global-api-key/#response>`. The 
default output only includes a subset of the fields returned by this 
command.

.. _mcli-iam-global-apikey-describe-examples:

Examples
--------

.. tabs::

   .. tab:: Default Output
      :tabid: default-output

      The following command describes a Global |api| Key using
      the {+default-profile+}, which contains credentials for accessing 
      the |onprem| instance. The output is returned in the default format.

      .. code-block:: sh

         mongocli iam globalApiKey describe 5f5aa08257aef32b04ec588c

      The previous command prints the following to the terminal.

      .. code-block:: json
         :copyable: false

         ID                         DESCRIPTION         PUBLIC KEY   PRIVATE KEY
         5f5aa08257aef32b04ec588c   My Global API key   abcdefgh     ********-****-****-f4d64595457e

   .. tab:: JSON Output
      :tabid: json-output

      The following command describes a Global |api| Key using the ``myOM``
      profile, which contains credentials. The output is returned in
      |json| format.
      
      .. code-block:: sh

         mongocli iam globalApiKey describe 5f5aa08257aef32b04ec588c  -P myOM --output json

      The previous command prints the following to the terminal in |json| 
      format.

      .. code-block:: json
         :copyable: false

         {
           "id": "5f5aa08257aef32b04ec588c",
           "desc": "My Global API key",
           "roles": [
             {
               "roleName": "GLOBAL_READ_ONLY"
             },
             {
               "roleName": "GLOBAL_USER_ADMIN"
             }
           ],
           "privateKey": "********-****-****-f4d64595457e",
           "publicKey": "abcdefgh"
         }
