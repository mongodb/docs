.. _mcli-cm-maintenanceWindow-update-command:

===============================================
mongocli cloud-manager maintenanceWindow update
===============================================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The ``maintenanceWindow update`` command updates the configuration of
one |cloud-short| :cloudmgr:`maintenance window </reference/api/maintenance-windows>`
for the specified project. You can also update the configuration of a
maintenance window through the :cloudmgr:`API 
</reference/api/maintenance-windows-update-one/>`.

.. _mcli-cm-maintenanceWindow-update-syntax:

Syntax 
------

.. code-block:: text 

   mongocli cloud-manager|cm maintenanceWindow update <maintenance-window-id>
        --startDate <start-date>
        --endDate <end-date>
        --alertType <alert-type>
        [ --desc <description> ]
        [ --output|-o <output-format> ]
        [ --profile|-P <profile-name> ]
        [ --projectId <project-ID> ]

Arguments  
---------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Argument 
     - Type 
     - Description 
     - Required?

   * - ``<maintenance-window-id>``
     - string
     - Unique identifier of the maintenance window that you
       want to update.
     - yes

.. _mcli-cm-maintenanceWindow-update-options: 

Options 
-------

.. list-table::
   :header-rows: 1
   :widths: 20 10 60 10

   * - Option
     - Type
     - Description
     - Required?

   * - ``--startDate``
     - string
     - Timestamp in ``ISO 8601`` date and time format in UTC when the
       maintenance window starts.
     - yes

   * - ``--endDate``
     - string
     - Timestamp in ``ISO 8601`` date and time format in UTC when the
       maintenance window ends.
     - yes

   * - ``--alertType``
     - string
     - Comma-separated list of alert types to silence during maintenance
       window. For example: ``HOST,REPLICA_SET,CLUSTER,AGENT,BACKUP``
     - yes

   * - ``--desc``
     - string
     - Description of the maintenance window.
     - no

   * - ``--output``, ``-o``
     - string 
     - .. include:: /includes/extracts/fact-basic-options-output.rst
     - no

   * - ``--profile``, ``-P``
     - string
     - Name of the profile where the public and private 
       keys for the project are saved. If omitted, uses the 
       {+default-profile+}. To learn more about creating a 
       profile, see :ref:`mcli-configure`.
     - no

   * - ``--projectId``
     - string
     - Unique identifier of the project that contains the 
       cluster. If omitted, uses the project ID in the profile or 
       :ref:`environment variable <mcli-env-var>`.
     - no

Example 
-------

The following example uses the ``mongocli cm maintenanceWindow
update`` command to update the maintenance window with ID
``5f7cb0eec902201990cb1506``. It uses a profile called ``myCm`` and
specifies JSON output.

.. code-block:: none
   
   mongocli cm maintenanceWindow update 5f7cb0eec902201990cb1506 \
     --startDate 2020-10-23T22:00:00Z \
     --endDate 2020-10-24T22:00:00Z \
     --alertType HOST,BACKUP \
     --desc "One-day maintenance window" \
     -P myCm -o json

The above command produces the following output. For more information about
these fields, see :ref:`Output <mcli-cm-maintenanceWindow-update-output>`.

.. code-block:: json
   :copyable: false

   {
     "id": "5f7cb0eec902201990cb1506",
     "groupId": "5f1f39ffc902201990f53873",
     "created": "2020-10-06T18:01:18Z",
     "startDate": "2020-10-23T22:00:00Z",
     "endDate": "2020-10-24T22:00:00Z",
     "updated": "2020-10-06T18:05:29Z",
     "alertTypeNames": [
       "BACKUP",
       "HOST"
     ],
     "description": "One-day maintenance window"
   }
