.. _server-side-javascript:

======================
Server-side JavaScript
======================

.. meta::
   :description: Explore server-side JavaScript functions in MongoDB, including how to enable or disable execution and which commands, methods, and operators run server-side code.

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

.. important::

   :ref:`Map-reduce <map-reduce>` is deprecated starting in MongoDB 5.0.

MongoDB provides the following operators that perform server-side
execution of JavaScript code:

- :query:`$where` operator that evaluates a JavaScript expression or a
  function in order to query for documents.

- :group:`$accumulator` and :expression:`$function` aggregation
  operations that allows users to define custom aggregation expressions.

.. _running-js-scripts-in-mongo-on-mongod-host:

Run .js files with the MongoDB Shell
------------------------------------

You can specify a JavaScript (``.js``) file to :binary:`~bin.mongosh`
to execute the file on the server. This is a good technique
for performing batch administrative work. When you run
:binary:`~bin.mongosh` on the server, connecting via the localhost
interface, the connection is fast with low latency.

Behavior
--------

JavaScript in MongoDB
~~~~~~~~~~~~~~~~~~~~~

Although server-side methods use JavaScript, most interactions with
MongoDB do not use JavaScript but instead use an :driver:`idiomatic
driver </>` in the language of the interacting application.

Concurrency
~~~~~~~~~~~

Refer to the individual method or operator documentation for any
concurrency information. See also the :ref:`concurrency table
<faq-concurrency-operations-locks>`.

SELinux Considerations
~~~~~~~~~~~~~~~~~~~~~~

If you use SELinux, any MongoDB operation that requires server-side
JavaScript results in a segfault error. To disable execution of
server-side JavaScript, see :ref:`disable-server-side-js`.
