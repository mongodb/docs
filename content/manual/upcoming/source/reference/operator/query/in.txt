===
$in
===

.. default-domain:: mongodb

.. meta::
   :description: Use the $in operator to select documents where a field's value equals any value in a specified array, and also query for multiple possible values.

.. contents:: On this page

.. query:: $in

   The :query:`$in` operator selects the documents where the value
   of a field equals any value in the specified array.

Compatibility
-------------

.. |operator-method| replace:: ``$in``

.. include:: /includes/fact-compatibility.rst

Syntax
------

The :query:`$in` operator has the following form:

.. code-block:: javascript

   { field: { $in: [ <value1>, <value2>, ... <valueN> ] } }

.. include:: /includes/fact-comparison-order.rst

If ``field`` has an array, the :query:`$in` operator
selects the documents whose ``field`` has an array that contains
at least one element that matches a value in the specified array.
For example, ``<value1>``, ``<value2>``, and so on.

:query:`$in` compares each parameter to each document in the
collection, which can cause performance issues. To improve
performance:

-  Limit the number of parameters passed to :query:`$in` to tens of
   values. Using hundreds of parameters can negatively impact query
   performance.
-  Create an index on the ``field`` you want to query.

.. note::

   This document describes the :query:`$in` query operator.
   For the ``$in`` aggregation operator, see
   :doc:`/reference/operator/aggregation/in`.

Examples
--------

To create the ``inventory`` collection used in the examples, run:

.. code-block:: javascript

   db.inventory.insertMany( [
      { item: "Pens", quantity: 350, tags: [ "school", "office" ] },
      { item: "Erasers", quantity: 15, tags: [ "school", "home" ] },
      { item: "Maps", tags: [ "office", "storage" ] },
      { item: "Books", quantity: 5, tags: [ "school", "storage", "home" ] }
   ] )

Match Values
~~~~~~~~~~~~

This query selects documents in the ``inventory`` collection where the
value of the ``quantity`` field is ``5`` or ``15``:

.. code-block:: javascript

   db.inventory.find( { quantity: { $in: [ 5, 15 ] } }, { _id: 0 } )

The output:

.. code-block:: javascript

   { item: 'Erasers', quantity: 15, tags: [ 'school', 'home' ] },
   { item: 'Books', quantity: 5, tags: [ 'school', 'storage', 'home' ] }

Although you can write the query using the :query:`$or` operator,
use the :query:`$in` operator rather than the :query:`$or` operator
when performing equality checks on the same field.

Match Values in an Array
~~~~~~~~~~~~~~~~~~~~~~~~

The following :method:`~db.collection.updateMany()` operation sets the
``exclude`` field to ``false`` when the ``tags`` array has at least one
element that matches either ``"home"`` or ``"school"``:

.. code-block:: javascript

   db.inventory.updateMany(
      { tags: { $in: [ "home", "school" ] } },
      { $set: { exclude: false } }
   )

Example output:

.. code-block:: javascript

   {
      item: 'Pens',
      quantity: 350,
      tags: [ 'school', 'office' ],
      exclude: false
    },
    {
      item: 'Erasers',
      quantity: 15,
      tags: [ 'school', 'home' ],
      exclude: false
    },
    {
      item: 'Books',
      quantity: 5,
      tags: [ 'school', 'storage', 'home' ],
      exclude: false
    }

Use ``$in`` with a Regular Expression
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The :query:`$in` operator can select documents using regular expressions
of the form ``/pattern/``. You cannot use :query:`$regex` expressions
inside :query:`$in`.

This query selects documents in the ``inventory`` collection where the
``tags`` field starts with ``be`` or ``st``:

.. code-block:: javascript

   db.inventory.find( { tags: { $in: [ /^be/, /^st/ ] } } )

The query selects all documents in the ``inventory`` collection where
the ``tags`` field has either a string that starts with ``be`` or
``st`` or an array with at least one element that starts with ``be`` or
``st``.
